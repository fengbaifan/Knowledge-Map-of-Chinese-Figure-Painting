# 变更日志格式规范

## 文件组织

```
99-日志文档/
├── 2026-04/
│   ├── 变更日志_2026-04-19.md
│   └── 变更日志_2026-04-20.md
└── 2026-05/
    └── 变更日志_2026-05-01.md
```

**规则**：
- 按年月创建子目录（格式：`YYYY-MM`）
- 每天一个日志文件（格式：`变更日志_YYYY-MM-DD.md`）
- 同一天的多次修订追加到同一文件

## 字段说明

### 记录头部

| 字段 | 必填 | 说明 |
|------|------|------|
| 时间戳 | 是 | 格式 `[HH:MM:SS]`，24小时制 |
| 记录编号 | 是 | 当天的第几次修订，从 #1 开始 |
| 修改人 | 是 | Git 用户名 + 邮箱 |
| 修改时间 | 是 | 完整时间戳 `YYYY-MM-DD HH:MM:SS` |
| 修改类型 | 是 | 数据修订（自动修复）/ 手动修订 / 配置变更 |
| 触发来源 | 是 | extraction-reviewer / 用户手动操作 / 其他技能 |

### 影响范围

| 字段 | 必填 | 说明 |
|------|------|------|
| 文件路径 | 是 | 被修改文件的相对路径 |
| 原始文件 | 否 | 如果是修订版，指向原始文件 |
| 修改行数 | 是 | 受影响的数据行数 |
| 修改字段数 | 是 | 受影响的字段数 |

### 修改内容详情

**字段名修改**：
- 使用表格列出所有字段名变更
- 包含原字段名、新字段名、影响行数

**数据值修改**：
- 按行号列出每个修改
- 包含字段名、修改前值、修改后值、原因

### 修改原因

简要说明为什么进行此次修订：
- 自动修复：列出修复的问题类型
- 手动修订：说明修订的依据和目的

### 相关文件

列出与此次修订相关的所有文件：
- 审查报告
- 问题清单
- Git commit hash（如果已提交）

## Git Diff 映射规则

从 `git diff` 输出提取信息：

```diff
diff --git a/03-初步抽取/古画品录/M2.1_历史人物.csv b/03-初步抽取/古画品录/M2.1_历史人物.csv
index abc123..def456 100644
--- a/03-初步抽取/古画品录/M2.1_历史人物.csv
+++ b/03-初步抽取/古画品录/M2.1_历史人物.csv
@@ -4,1 +4,1 @@
-meso_001,Painter,顾恺之,,,,,"{""birth"": ""580"", ""death"": ""640""}",period_tang,,《续画品》,卷一·顾恺之,"顾恺之，字长康，晋陵无锡人。",
+meso_001,Painter,顾恺之,,,,,"{""birth"": ""585"", ""death"": ""643""}",period_tang,,《续画品》,卷一·顾恺之,"顾恺之，字长康，晋陵无锡人。",
```

**提取规则**：
1. 文件路径：从 `diff --git` 行提取
2. 行号：从 `@@` 行提取（格式：`@@ -旧行号,行数 +新行号,行数 @@`）
3. 修改内容：
   - `-` 开头的行：修改前的值
   - `+` 开头的行：修改后的值
4. 字段识别：解析 CSV 行，对比差异字段

## 特殊情况处理

### 新增文件

```markdown
### 影响范围

- **文件路径**：`03-初步抽取/新文献/M2.1_历史人物.csv`
- **操作类型**：新增文件
- **记录行数**：50 行

### 修改内容详情

新增文件，包含 50 条历史人物记录。
```

### 删除文件

```markdown
### 影响范围

- **文件路径**：`03-初步抽取/旧文献/M2.1_历史人物.csv`
- **操作类型**：删除文件
- **原因**：文献已合并到其他文件

### 修改内容详情

删除文件，原有 30 条记录已迁移到 `03-初步抽取/合并文献/M2.1_历史人物.csv`。
```

### 批量修改

如果修改行数超过 20 行，使用汇总格式：

```markdown
### 修改内容详情

**批量修改**：共修改 150 行

- 字段：`source_book`
- 修改类型：添加书名号
- 影响行数：150 行
- 修改模式：`文献名` → `《文献名》`

详细变更列表见：`99-日志文档/2026-04/详细变更_20260419_173045.csv`
```
