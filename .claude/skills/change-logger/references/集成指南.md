# 集成指南

## 与 extraction-reviewer 集成

### 集成目标

在 extraction-reviewer 完成数据修订后，自动调用 change-logger 记录所有变更。

### 集成方式

#### 方式一：在 reviewer.py 中直接调用

修改 `extraction-reviewer/scripts/reviewer.py`：

```python
# 在文件开头添加导入
import sys
from pathlib import Path

# 添加 change-logger 到路径
change_logger_path = Path(__file__).parent.parent.parent / 'change-logger' / 'scripts'
sys.path.insert(0, str(change_logger_path))

from logger import log_changes

# 在生成修订数据后调用
def main():
    # ... 审查逻辑 ...
    
    # 生成修订数据
    revised_file = save_revised_data(df, output_dir)
    
    # 记录变更
    if auto_fixed_count > 0:
        log_changes(
            modified_files=[revised_file],
            original_file=input_file,
            reason=f"extraction-reviewer 自动修复：{', '.join(issue_types)}",
            report_file=report_path,
            issue_list_file=issue_list_path
        )
    
    # ... 生成报告 ...
```

#### 方式二：使用 subprocess 调用

```python
import subprocess

def call_change_logger(modified_files, reason, **kwargs):
    """调用 change-logger 记录变更"""
    cmd = [
        'python',
        '.claude/skills/change-logger/scripts/logger.py',
        '--files', *modified_files,
        '--reason', reason
    ]
    
    # 添加可选参数
    if 'original_file' in kwargs:
        cmd.extend(['--original-file', kwargs['original_file']])
    if 'report_file' in kwargs:
        cmd.extend(['--report-file', kwargs['report_file']])
    
    result = subprocess.run(cmd, capture_output=True, text=True)
    
    if result.returncode == 0:
        print(f"✅ 变更日志已记录：{result.stdout.strip()}")
    else:
        print(f"⚠️ 变更日志记录失败：{result.stderr}")

# 在 reviewer.py 中调用
call_change_logger(
    modified_files=[revised_file],
    reason=f"extraction-reviewer 自动修复：{auto_fixed_count}个问题",
    original_file=input_file,
    report_file=report_path
)
```

### 传递的信息

从 extraction-reviewer 传递给 change-logger 的信息：

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `modified_files` | list | 修订后的文件路径列表 | `['04-审查与修订/古画品录/修订数据/M2.1_修订版.csv']` |
| `original_file` | str | 原始文件路径 | `'03-初步抽取/古画品录/M2.1_历史人物.csv'` |
| `reason` | str | 修订原因 | `'extraction-reviewer 自动修复：字段名错误、书名格式'` |
| `report_file` | str | 审查报告路径 | `'04-审查与修订/古画品录/报告/审查报告_20260419.md'` |
| `issue_list_file` | str | 问题清单路径 | `'04-审查与修订/古画品录/报告/问题清单_20260419.csv'` |
| `auto_fixed_count` | int | 自动修复的问题数 | `19` |
| `issue_types` | list | 问题类型列表 | `['字段名错误', '书名格式不规范']` |

### 修改 extraction-reviewer 的输出

在审查报告末尾添加变更日志链接：

## 与其他技能集成

### 通用集成模式

任何需要记录变更的技能都可以调用 change-logger：

```python
from pathlib import Path
import sys

# 添加 change-logger 到路径
project_root = Path(__file__).parent.parent.parent
change_logger_path = project_root / '.claude' / 'skills' / 'change-logger' / 'scripts'
sys.path.insert(0, str(change_logger_path))

from logger import log_changes

# 在修改文件后调用
log_changes(
    modified_files=['path/to/modified/file.csv'],
    reason='技能名称：修改原因',
    metadata={
        'skill_name': '技能名称',
        'operation': '操作类型',
        'custom_field': '自定义信息'
    }
)
```

### 集成检查清单

在集成 change-logger 时，确保：

- [ ] 在修改文件**之后**调用 change-logger
- [ ] 传递准确的文件路径（相对于项目根目录）
- [ ] 提供清晰的修改原因
- [ ] 如果有原始文件，传递 `original_file` 参数
- [ ] 如果有相关报告，传递 `report_file` 参数
- [ ] 处理 change-logger 可能的错误（如不在 Git 仓库中）

## 自动触发机制

### 文件监听（可选）

如果需要监听文件变更自动触发，可以使用 watchdog：

```python
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
import time

class ChangeHandler(FileSystemEventHandler):
    def on_modified(self, event):
        if event.is_directory:
            return
        
        # 只监听特定目录
        if '03-初步抽取' in event.src_path or '04-审查与修订' in event.src_path:
            if event.src_path.endswith('.csv'):
                print(f"检测到文件变更：{event.src_path}")
                
                # 调用 change-logger
                log_changes(
                    modified_files=[event.src_path],
                    reason='文件监听：自动检测到变更'
                )

# 启动监听
observer = Observer()
observer.schedule(ChangeHandler(), path='03-初步抽取', recursive=True)
observer.schedule(ChangeHandler(), path='04-审查与修订', recursive=True)
observer.start()

try:
    while True:
        time.sleep(1)
except KeyboardInterrupt:
    observer.stop()

observer.join()
```

### Git Hook 集成

在 `.git/hooks/post-commit` 中添加：

```bash
#!/bin/bash

# 获取本次提交修改的 CSV 文件
changed_files=$(git diff-tree --no-commit-id --name-only -r HEAD | grep '\.csv$')

if [ -n "$changed_files" ]; then
    echo "检测到 CSV 文件变更，记录到变更日志..."
    
    # 获取提交信息
    commit_msg=$(git log -1 --pretty=%B)
    
    # 调用 change-logger
    python .claude/skills/change-logger/scripts/logger.py \
        --files $changed_files \
        --reason "Git commit: $commit_msg" \
        --commit-hash $(git rev-parse HEAD)
fi
```

## 错误处理

### 处理 change-logger 不可用的情况

```python
def safe_log_changes(**kwargs):
    """安全地调用 change-logger，处理各种错误"""
    try:
        from logger import log_changes
        log_changes(**kwargs)
        return True
    except ImportError:
        print("⚠️ change-logger 未安装，跳过变更日志记录")
        return False
    except Exception as e:
        print(f"⚠️ 变更日志记录失败：{e}")
        return False

# 在技能中使用
if safe_log_changes(modified_files=[...], reason='...'):
    print("✅ 变更日志已记录")
else:
    print("⚠️ 变更日志记录失败，但不影响主流程")
```

### 处理非 Git 仓库

```python
import subprocess

def is_git_repo():
    """检查是否在 Git 仓库中"""
    try:
        result = subprocess.run(
            ['git', 'rev-parse', '--git-dir'],
            capture_output=True,
            text=True
        )
        return result.returncode == 0
    except:
        return False

# 在调用前检查
if is_git_repo():
    log_changes(...)
else:
    print("⚠️ 不在 Git 仓库中，无法使用 git diff，跳过变更日志")
```

## 最佳实践

1. **在事务边界调用**：在完成一组相关修改后调用一次，而不是每修改一个文件调用一次
2. **提供详细原因**：reason 参数应该清楚说明为什么修改，而不只是"修改了文件"
3. **传递上下文**：尽可能传递 original_file、report_file 等上下文信息
4. **错误不阻塞**：change-logger 失败不应该阻塞主流程
5. **定期清理**：旧的日志文件可以归档或压缩
