# 修复策略

本文档说明自动修复的实现逻辑和人工处理的判断标准。

---

## 自动修复原则

1. **确定性修复**：只修复100%确定的问题
2. **可逆性**：保留原始数据，修订数据保存到新文件
3. **可追溯**：记录所有修改操作
4. **保守策略**：有疑问时标记为需人工处理

---

## 可自动修复的问题

### 1. 书名格式修复

**问题**：source_book 缺少书名号或有多余符号

**修复逻辑**：
```python
def fix_book_name(name):
    if pd.isna(name) or name == '':
        return name
    
    # 清理多余符号
    name = name.strip()
    name = name.strip('"').strip("'")
    
    # 检查是否已有书名号，避免重复添加
    if not (name.startswith('《') and name.endswith('》')):
        if not name.startswith('《'):
            name = '《' + name
        if not name.endswith('》'):
            name = name + '》'
    
    return name
```

**示例**：
- `历代名画记` → `《历代名画记》`
- `"《续画品》"` → `《续画品》`
- `<古画品录>` → `《古画品录》`

---

### 2. 章节格式修复

**问题**：source_chapter 使用空格而非间隔号

**修复逻辑**：
```python
def fix_chapter(chapter):
    if pd.isna(chapter) or chapter == '':
        return chapter
    
    # 将空格替换为间隔号
    chapter = chapter.replace(' ', '·')
    
    return chapter
```

**示例**：
- `卷一 顾野王` → `卷一·顾野王`
- `卷三 西京寺观等画壁 兴善寺` → `卷三·西京寺观等画壁·兴善寺`

### 3. ID前缀修复

**问题**：ID缺少规范前缀

**修复逻辑**：
```python
def fix_id_prefix(value, table_type):
    if pd.isna(value) or value == '':
        return value
    
    # 根据表类型确定前缀
    prefix_map = {
        'M1.1_时序本体': 'period_',
        'M1.2_空间地理本体': 'loc_',
        'M1.3_图像志题材分类': 'icon_',
        'M2.1_历史人物': 'meso_',
        'M2.2_履历与时空轨迹': 'cv_',
        'M2.3_社会关系实例': 'rel_',
        'M3.1_作品实体表': 'micro_',
        'M3.2_文献著录与品评': 'lit_'
    }
    
    prefix = prefix_map.get(table_type, '')
    if prefix and not value.startswith(prefix):
        value = prefix + value
    
    return value
```

**示例**：
- `gu_kaizhi` → `meso_gu_kaizhi`（人物表）
- `luoshen_fu` → `micro_luoshen_fu`（作品表）
- `001` → `cv_001`（履历表）

### 4. 字段名修复

**问题**：字段名不符合规范（如使用点号而非下划线）

**修复逻辑**：
```python
def fix_field_names(df):
    rename_map = {
        'source.chapter': 'source_chapter',
        'source.text': 'source_text',
        'source.page': 'source_page',
        'originalsource_sentence': 'source_text',
        'original_source': 'source_text'
    }
    
    df.rename(columns=rename_map, inplace=True)
    return df
```

**示例**：
- `source.chapter` → `source_chapter`
- `originalsource_sentence` → `source_text`

---

### 5. 空白字符清理

**问题**：字段值包含多余的空白字符

**修复逻辑**：
```python
def clean_whitespace(value):
    if pd.isna(value) or value == '':
        return value
    
    # 清理首尾空白
    value = str(value).strip()
    
    # 清理多余的内部空白（保留单个空格）
    value = ' '.join(value.split())
    
    return value
```

**示例**：
- `  顾恺之  ` → `顾恺之`
- `卷一  ·  顾野王` → `卷一 · 顾野王`

---

### 6. 引号清理

**问题**：字段值包含多余的引号

**修复逻辑**：
```python
def clean_quotes(value):
    if pd.isna(value) or value == '':
        return value
    
    value = str(value).strip()
    
    # 移除首尾的引号（单引号或双引号）
    if (value.startswith('"') and value.endswith('"')) or \
       (value.startswith("'") and value.endswith("'")):
        value = value[1:-1]
    
    return value
```

**示例**：
- `"《历代名画记》"` → `《历代名画记》`
- `'顾恺之'` → `顾恺之`

## 需人工处理的问题

### 1. 必填字段为空

**问题**：必填字段缺失或为空

**原因**：无法自动推断正确的值

**处理方式**：
- 在问题清单中标记为"严重"
- 提供原文引用，帮助人工填充
- 建议查阅原始文献

**示例**：
```
文件：M2.1_历史人物_古画品录.csv
行号：15
字段：source_book
问题：必填字段为空
建议：查阅原始文献，填写来源典籍名称
```

---

### 2. 外键引用不存在

**问题**：外键字段引用的ID在目标表中不存在

**原因**：可能是ID拼写错误或目标记录缺失

**处理方式**：
- 检查是否是拼写错误（如meso_wu_daozi vs meso_wudaozi）
- 检查目标表是否缺少该记录
- 提供相似ID建议

**示例**：
```
文件：M3.1_作品实体表_古画品录.csv
行号：23
字段：creator_ref
当前值：meso_wu_daozi
问题：引用的person_id在人物表中不存在
建议：检查是否应为 meso_wudaozi（相似度90%）
```

---

### 3. 枚举值错误

**问题**：枚举字段的值不在允许范围内

**原因**：使用了中文或其他非标准值

**处理方式**：
- 提供允许的枚举值列表
- 尝试映射（如"画家" → "Painter"）
- 标记为需人工确认

**示例**：
```
文件：M2.1_历史人物_古画品录.csv
行号：8
字段：primary_role
当前值：画家
问题：枚举值不在允许范围内
允许值：Painter, Critic, Collector, Calligrapher, Scholar, Patron, Other
建议：应为 Painter
```

**可能的自动映射**：
```python
role_mapping = {
    '画家': 'Painter',
    '评论家': 'Critic',
    '收藏家': 'Collector',
    '书法家': 'Calligrapher',
    '学者': 'Scholar',
    '赞助人': 'Patron'
}
```

---

### 4. 数据逻辑矛盾

**问题**：相关字段的值不符合逻辑

**原因**：数据录入错误或理解偏差

**处理方式**：
- 标记矛盾的字段组合
- 提供逻辑检查结果
- 建议查阅原始文献核实

**示例**：
```
文件：M2.1_历史人物_古画品录.csv
行号：12
问题：period_ref与birth_death不匹配
period_ref：period_tang（618-907）
birth_death：{"birth": 1200, "death": 1280}
建议：检查period_ref是否应为period_song
```

---

### 5. JSON格式错误

**问题**：JSON字段格式不正确

**原因**：缺少引号、括号不匹配等

**处理方式**：
- 尝试简单修复（如添加引号）
- 无法修复时标记为需人工处理
- 提供正确格式示例

**可尝试自动修复的情况**：
```python
# 缺少引号的键名
{birth: 535, death: 602} → {"birth": 535, "death": 602}

# 字符串类型的数字
{"birth": "535"} → {"birth": 535}
```

**需人工处理的情况**：
```python
# 格式严重错误
{birth: 535, death: → 无法自动修复
```

## 修复执行流程

### 1. 读取原始数据

```python
df = pd.read_csv(file_path, encoding='utf-8-sig', index_col=False)
```

### 2. 执行检查，收集问题

```python
issues = []
for check_func in [check_completeness, check_format, check_consistency]:
    issues.extend(check_func(df))
```

### 3. 分类问题

```python
auto_fixable = [issue for issue in issues if issue['auto_fixable']]
manual_required = [issue for issue in issues if not issue['auto_fixable']]
```

### 4. 执行自动修复

```python
fixed_df = df.copy()
for issue in auto_fixable:
    fixed_df = apply_fix(fixed_df, issue)
```

### 5. 保存修订数据

```python
output_path = f"{output_dir}/修订数据/{filename}_修订版_{timestamp}.csv"
fixed_df.to_csv(output_path, encoding='utf-8-sig', index=False)
```

### 6. 生成报告

```python
generate_report(issues, auto_fixable, manual_required)
```

- 检查文件编码
- 验证CSV格式
- 确认字段名存在

### 修复后验证

- 重新运行检查
- 确认问题已解决
- 统计修复成功率

### 回滚机制

- 保留原始文件不变

- 可随时对比原始和修订版本
