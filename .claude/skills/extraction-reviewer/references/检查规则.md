# 检查规则详解

本文档详细说明数据审查的三层检查规则。

---

## 一、完整性检查

### 1.1 必填字段检查

**规则**：所有必填字段必须存在且非空（不能是空字符串、null、NaN）

**各表必填字段**：

| 表名 | 必填字段 |
|------|---------|
| M1.1_时序本体 | period_id, name, time_range, source_book |
| M1.2_空间地理本体 | location_id, modern_address, source_book |
| M1.3_图像志题材分类 | icon_id, name_cn, source_book |
| M2.1_历史人物核心表 | person_id, name, primary_role, source_book |
| M2.2_履历与时空轨迹 | cv_id, person_ref, event_type, source_book |
| M2.3_社会关系实例 | rel_id, source_id, target_id, relation_type, source_book |
| M3.1_作品实体表 | work_id, title, creator_ref, source_book |
| M3.2_文献著录与品评 | lit_id, target_ref, source_book |

**检查逻辑**：
```python
for field in required_fields:
    missing = df[df[field].isna() | (df[field] == '')]
    if len(missing) > 0:
        # 记录问题：文件名、行号、字段名
```

**问题级别**：严重（必须修复）

---

### 1.2 推荐字段检查

**规则**：推荐字段覆盖率应 >50%，否则发出警告

**推荐字段**（所有表通用）：
- source_chapter（强烈推荐）
- source_text（推荐）
- period_ref（人物、作品表推荐）

**检查逻辑**：
```python
coverage = df[field].notna().sum() / len(df) * 100
if coverage < 50:
    # 警告：字段覆盖率过低
```

**问题级别**：警告（建议修复）

---

### 1.3 字段覆盖率统计

**规则**：计算每个字段的填充率，生成统计报告

**计算公式**：
```
填充率 = (非空值数量 / 总行数) × 100%
```

**分级标准**：
- ✅ 优秀：≥90%
- ⚠️ 良好：50-89%
- ❌ 不足：<50%

---

## 二、准确性检查

### 2.1 书名格式检查

**规则**：source_book 必须使用中文书名号《》

**正确示例**：
- ✅ `《历代名画记》`
- ✅ `《宣和画谱》`
- ✅ `《古画品录》`

**错误示例**：
- ❌ `历代名画记`（缺少书名号）
- ❌ `<历代名画记>`（错误符号）
- ❌ `"《历代名画记》"`（多余引号）

**检查逻辑**：
```python
pattern = r'^《.*》$'
invalid = df[~df['source_book'].str.match(pattern, na=False)]
```

**自动修复**：
```python
def fix_book_name(name):
    name = name.strip().strip('"').strip("'")
    if not (name.startswith('《') and name.endswith('》')):
        if not name.startswith('《'):
            name = '《' + name
        if not name.endswith('》'):
            name = name + '》'
    return name
```

---

### 2.2 章节格式检查

**规则**：source_chapter 应为"卷次·章节名"格式

**标准格式**：
- ✅ `卷一·顾野王`
- ✅ `卷三·西京寺观等画壁·兴善寺`
- ✅ `卷五`（无具体章节时）

**可接受格式**：
- ⚠️ `卷一 顾野王`（空格分隔，可自动修复）

**检查逻辑**：
```python
# 检查是否包含卷次
if not ('卷' in chapter or '序' in chapter):
    # 警告：章节格式不规范
```

---

### 2.3 ID格式检查

**规则**：各类ID必须有正确的前缀

**ID前缀规范**：

| 表名 | 字段 | 前缀 | 示例 |
|------|------|------|------|
| M1.1_时序本体 | period_id | `period_` | period_tang |
| M1.2_空间地理本体 | location_id | `loc_` | loc_changan |
| M1.3_图像志题材分类 | icon_id | `icon_` | icon_portrait |
| M2.1_历史人物 | person_id | `meso_` | meso_gu_kaizhi |
| M2.2_履历 | cv_id | `cv_` | cv_001 |
| M2.3_社会关系 | rel_id | `rel_` | rel_001 |
| M3.1_作品 | work_id | `micro_` | micro_luoshen_fu |
| M3.2_文献 | lit_id | `lit_` | lit_001 |

**检查逻辑**：
```python
pattern = r'^meso_'  # 根据表类型选择
invalid = df[~df['person_id'].str.match(pattern, na=False)]
```

**自动修复**：
```python
if not value.startswith('meso_'):
    value = 'meso_' + value
```

---

### 2.4 枚举值检查

**规则**：枚举字段的值必须在允许范围内

**枚举字段定义**：

**primary_role**（人物角色）：
- Painter（画家）
- Critic（评论家）
- Collector（收藏家）
- Calligrapher（书法家）
- Scholar（学者）
- Patron（赞助人）
- Other（其他）

**status**（作品状态）：
- Extant（存世）
- Lost（佚失）
- Attributed（传）
- Copy（摹本）
- Unknown（未知）

**support**（作品材质）：
- Silk（绢本）
- Paper（纸本）
- Wall（壁画）
- Wood（木板）
- Other（其他）
- Unknown（未知）

**relation_type**（关系类型）：
- Teacher（师）
- Student（徒）
- Friend（友）
- Colleague（同僚）
- Spouse（配偶）
- Parent（父母）
- Child（子女）
- Sibling（兄弟姐妹）
- Other（其他）

**检查逻辑**：
```python
allowed_values = ['Painter', 'Critic', 'Collector', ...]
invalid = df[~df['primary_role'].isin(allowed_values)]
```

**问题级别**：警告（需人工确认正确的枚举值）

---

### 2.5 JSON格式检查

**规则**：JSON字段必须是有效的JSON格式

**JSON字段**：
- birth_death：`{"birth": 年份, "death": 年份}` 或 `{}`
- time_range：`{"start": 数字, "end": 数字}`
- authority_ids：`{"Wikidata": "Q123", "CBDB": "456"}`

**检查逻辑**：
```python
import json
try:
    json.loads(value)
except json.JSONDecodeError:
    # 错误：JSON格式不正确
```

## 三、一致性检查

### 3.1 外键完整性检查

**规则**：所有外键引用必须在目标表中存在

**外键关系**：

| 源表 | 外键字段 | 目标表 | 目标字段 |
|------|---------|--------|---------|
| M2.2_履历 | person_ref | M2.1_历史人物 | person_id |
| M2.3_社会关系 | source_id | M2.1_历史人物 | person_id |
| M2.3_社会关系 | target_id | M2.1_历史人物 | person_id |
| M3.1_作品 | creator_ref | M2.1_历史人物 | person_id |
| M3.1_作品 | period_ref | M1.1_时序本体 | period_id |
| M3.1_作品 | icon_ref | M1.3_图像志 | icon_id |
| M3.2_文献 | target_ref | M3.1_作品 或 M2.1_历史人物 | work_id 或 person_id |
| M3.2_文献 | author_ref | M2.1_历史人物 | person_id |

**检查逻辑**：
```python
# 读取目标表
persons_df = pd.read_csv('M2.1_历史人物.csv')
person_ids = set(persons_df['person_id'].dropna())

# 检查外键
works_df = pd.read_csv('M3.1_作品实体表.csv')
invalid_refs = works_df[~works_df['creator_ref'].isin(person_ids)]
```

**问题级别**：严重（必须修复）

---

### 3.2 跨表一致性检查

**规则**：同一实体在不同表中的基本信息应一致

**检查项**：
- 同一person_id在不同文献中的name、courtesy_name应一致
- 同一period_id的time_range应一致

**检查逻辑**：
```python
# 合并所有文献的人物表
all_persons = pd.concat([df1, df2, df3])
# 检查同一person_id的name是否一致
duplicates = all_persons.groupby('person_id')['name'].nunique()
inconsistent = duplicates[duplicates > 1]
```

---

### 3.3 逻辑一致性检查

**规则**：相关字段的值应符合逻辑

**检查项**：
- period_ref对应的时代应与birth_death年份匹配
- 作品的period_ref应与creator的period_ref一致或接近

**示例**：
- 如果person的period_ref是period_tang（618-907），birth_death不应是1200年

**问题级别**：警告（可能是数据错误或特殊情况）

---

## 检查优先级

1. **严重问题**（必须修复）：
   - 必填字段为空
   - 外键引用不存在

2. **警告问题**（建议修复）：
   - 推荐字段覆盖率低
   - 格式不规范
   - 枚举值错误
   - 逻辑不一致

3. **提示问题**（可选修复）：
   - 可选字段为空
   - 格式可接受但不标准
