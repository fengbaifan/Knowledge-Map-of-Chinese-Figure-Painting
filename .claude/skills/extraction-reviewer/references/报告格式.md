# 报告格式说明

本文档说明审查报告的结构和内容。

---

## 报告类型

审查完成后生成三类报告：

1. **审查报告.md** - 人类可读的综合报告
2. **问题清单.csv** - 可筛选排序的问题列表
3. **统计数据.json** - 机器可读的统计信息

---

## 二、问题清单（CSV）

### 文件命名

```
问题清单_YYYYMMDD_HHMMSS.csv
```

### 字段定义

| 字段名 | 说明 | 示例 |
|--------|------|------|
| file | 文件名 | M2.1_历史人物_古画品录.csv |
| row | 行号（从2开始，1为表头） | 15 |
| field | 字段名 | source_book |
| issue_type | 问题类型 | 必填字段为空 |
| severity | 严重程度 | 严重/警告/提示 |
| current_value | 当前值 | 历代名画记 |
| suggested_value | 建议值 | 《历代名画记》 |
| auto_fixed | 是否已自动修复 | true/false |
| status | 处理状态 | ✅ 已自动修复 / ⚠️ 需人工处理 |
| description | 问题描述 | 书名缺少书名号 |

### 示例数据


### 使用方式

可以使用Excel、Numbers或pandas进行筛选和排序：

```python

# 筛选需人工处理的严重问题

# 按文件分组统计
```

## 报告阅读指南

### 快速定位问题

1. **查看审查摘要**：了解整体情况
2. **查看严重问题**：优先处理必须修复的问题
3. **查看字段覆盖率**：识别数据缺失严重的字段
4. **使用问题清单CSV**：筛选特定类型的问题

### 处理优先级

1. **严重问题** → 必须修复
   - 必填字段为空
   - 外键引用不存在

2. **警告问题** → 建议修复
   - 格式不规范（已自动修复的可忽略）
   - 枚举值错误
   - 推荐字段覆盖率低

3. **提示问题** → 可选修复
   - 可选字段为空
   - 格式可接受但不标准

### 迭代改进

1. 处理人工问题后，重新运行审查
2. 对比前后两次报告，查看改进情况
3. 持续迭代直到数据质量达标
