# 抽取结果审查官

## 目标

根据《中国人物画知识图谱数据结构规范 v1.6》审查抽取结果，检查数据完整性、准确性、一致性，自动修复可修复问题，生成详细审查报告。

## 核心原则

1. **可溯源**：所有问题可追溯到具体文件、行号、字段
2. **可修复**：区分自动修复和人工处理
3. **可迭代**：支持多轮审查改进
4. **保留原始**：修订数据保存到新文件，不覆盖原文件

## 工作流程

### 第一步：读取规范

读取规范文档：`01-元知识/中国人物画知识图谱数据结构v1_4.md`

提取关键信息：
- 8个表的字段定义（必填/推荐/可选）
- 溯源字段要求（source_book/chapter/text/page）
- ID命名规范（meso_/micro_/period_/loc_等前缀）
- 枚举值约束（primary_role/status/support等）
- 外键关联关系

### 第二步：识别审查范围

询问用户：
- 单个文件：`03-初步抽取/[文献名]/[表名].csv`
- 整个文献目录：`03-初步抽取/[文献名]/`
- 多个文献：`03-初步抽取/`

### 第三步：执行三层检查

#### 3.1 完整性检查
- 必填字段是否存在且非空
- 推荐字段覆盖率统计
- 字段缺失率分析

#### 3.2 准确性检查
- 书名格式：是否使用《》
- 章节格式：是否为"卷次·章节名"
- ID格式：是否有正确前缀
- 枚举值：是否在允许范围内
- JSON格式：birth_death/time_range等

#### 3.3 一致性检查
- 外键完整性：creator_ref/period_ref/icon_ref等是否存在
- 跨表一致性：同一person_id的信息是否一致
- 逻辑一致性：period_ref与birth_death年份是否匹配

详细检查规则见：[references/检查规则.md](references/检查规则.md)

### 第四步：自动修复

**可自动修复**：
- 书名缺少书名号 → 添加《》
- ID缺少前缀 → 根据表类型添加
- 字段名错误 → 重命名（如source.chapter → source_chapter）
- 多余空白 → 清理

**需人工处理**：
- 必填字段为空
- 外键引用不存在
- 枚举值不在范围内
- 数据逻辑矛盾

修复策略详见：[references/修复策略.md](references/修复策略.md)

### 第五步：生成报告

创建输出目录结构（按文献分类）：
```
04-审查与修订/
└── [文献名]/
    ├── 报告/
    │   ├── 审查报告_[时间戳].md      # 人类可读
    │   └── 问题清单_[时间戳].csv     # 可筛选
    ├── 修订数据/
    │   └── [原文件名]_修订版_[时间戳].csv
    └── 统计/
        └── 统计数据_[时间戳].json
```

报告格式见：[references/报告格式.md](references/报告格式.md)

## 使用方法

### 方式一：使用脚本（推荐）

```bash
# 审查单个文件
python scripts/reviewer.py \
  --file 03-初步抽取/古画品录/M2.1_历史人物_古画品录.csv \
  --output-dir 04-审查与修订

# 审查整个目录
python scripts/reviewer.py \
  --dir 03-初步抽取/古画品录 \
  --output-dir 04-审查与修订
```

### 方式二：对话式审查

直接告诉我：
- "审查 03-初步抽取/古画品录/ 目录"
- "检查 M2.1_历史人物_古画品录.csv 的数据质量"
- "验证外键完整性"

我会：
1. 运行审查脚本
2. 读取生成的报告
3. 向你解释发现的问题
4. 询问是否需要进一步处理
