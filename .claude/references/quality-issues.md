# 数据质量问题分类

本文档详细列举中国人物画知识图谱数据抽取过程中的常见质量问题。

## 问题分类体系

### 1. 字段名错误

#### 1.1 使用点号分隔符
- **错误**：`source.book`、`source.chapter`、`source.text`
- **正确**：`source_book`、`source_chapter`、`source_text`
- **原因**：CSV 字段名应使用下划线，不使用点号

#### 1.2 字段名拼写错误
- **错误**：`originalsource_sentence`、`source_sentense`
- **正确**：`source_text`
- **原因**：字段名不符合规范定义

#### 1.3 字段名缺失前缀
- **错误**：`book`、`chapter`、`text`
- **正确**：`source_book`、`source_chapter`、`source_text`
- **原因**：溯源字段必须使用 `source_` 前缀

### 2. 格式不规范

#### 2.1 书名缺少书名号
- **错误**：`历代名画记`、`古画品录`
- **正确**：`《历代名画记》`、`《古画品录》`
- **严重性**：高（必填字段格式错误）

#### 2.2 章节格式错误
- **错误**：`卷一 顾野王`、`卷一-顾野王`、`第一卷·顾野王`
- **正确**：`卷一·顾野王`
- **说明**：使用中文间隔号（·）分隔卷次和章节名

#### 2.3 ID 前缀错误
- **错误**：`work_001`、`person_001`、`painting_001`
- **正确**：`micro_001`、`meso_001`、`micro_001`
- **说明**：必须使用规范定义的前缀

#### 2.4 ID 前缀缺失
- **错误**：`gu_kaizhi`、`luoshen_fu`
- **正确**：`meso_gu_kaizhi`、`micro_luoshen_fu`
- **严重性**：高（影响外键关联）

### 3. 必填字段为空

#### 3.1 溯源字段为空
- **字段**：`source_book`
- **严重性**：严重（违反核心原则）
- **处理**：必须人工补充

#### 3.2 主键为空
- **字段**：`person_id`、`work_id`、`period_id`
- **严重性**：严重（无法建立关联）
- **处理**：必须人工补充

#### 3.3 核心字段为空
- **字段**：`person_name`、`work_title`
- **严重性**：高（影响数据可用性）
- **处理**：必须人工补充

### 4. 枚举值错误

#### 4.1 使用中文枚举值
- **字段**：`primary_role`
  - **错误**：`画家`、`评论家`、`收藏家`
  - **正确**：`Painter`、`Critic`、`Collector`

- **字段**：`status`
  - **错误**：`存世`、`佚失`、`传`
  - **正确**：`Extant`、`Lost`、`Attributed`

- **字段**：`support`
  - **错误**：`绢本`、`纸本`、`壁画`
  - **正确**：`Silk`、`Paper`、`Wall`

#### 4.2 使用非标准枚举值
- **字段**：`primary_role`
  - **错误**：`画师`、`书画家`、`文人画家`
  - **正确**：`Painter`、`Painter`、`Scholar`

- **字段**：`status`
  - **错误**：`现存`、`失传`、`疑似`
  - **正确**：`Extant`、`Lost`、`Attributed`

#### 4.3 枚举值大小写错误
- **错误**：`painter`、`PAINTER`、`Painter`
- **正确**：`Painter`（首字母大写）

### 5. 外键引用错误

#### 5.1 外键指向不存在的记录
- **示例**：`M2.2.person_ref = meso_xxx` 但 `M2.1` 中不存在 `meso_xxx`
- **严重性**：严重（破坏引用完整性）
- **处理**：必须人工检查

#### 5.2 外键前缀不匹配
- **错误**：`M3.1.creator_ref = person_001`
- **正确**：`M3.1.creator_ref = meso_001`
- **说明**：外键必须使用正确的前缀

#### 5.3 外键为空但应有值
- **示例**：作品记录的 `creator_ref` 为空
- **严重性**：中（影响数据完整性）
- **处理**：根据上下文补充或标记为 Unknown

### 6. 数据类型错误

#### 6.1 年份格式错误
- **错误**：`公元618年`、`618-907`（应为单一年份）
- **正确**：`618`、`907`
- **说明**：年份字段应为纯数字

#### 6.2 布尔值格式错误
- **错误**：`是`、`否`、`yes`、`no`
- **正确**：`TRUE`、`FALSE` 或 `1`、`0`

#### 6.3 数值字段包含文本
- **错误**：`约618年`、`618年左右`
- **正确**：`618`（不确定性在备注字段说明）

### 7. 编码和特殊字符问题

#### 7.1 编码错误
- **表现**：乱码、问号、方框字符
- **原因**：文件编码不是 UTF-8-BOM
- **处理**：转换为 UTF-8-BOM 编码

#### 7.2 特殊字符错误
- **错误**：使用全角逗号、全角引号在 CSV 中
- **正确**：CSV 分隔符使用半角逗号，引号使用半角双引号

#### 7.3 换行符问题
- **表现**：字段内容意外换行
- **处理**：字段内容包含换行时必须用双引号包裹

### 8. 数据一致性问题

#### 8.1 同一实体 ID 不一致
- **示例**：同一人物在不同表中使用不同 ID
- **严重性**：严重（破坏数据关联）

#### 8.2 同一实体名称不一致
- **示例**：`顾恺之` vs `顾恺之（顾长康）`
- **处理**：统一使用标准名称，别名放在专门字段

#### 8.3 书名格式不一致
- **示例**：同一文献有时写《历代名画记》，有时写《歷代名畫記》
- **处理**：统一使用简体中文标准书名

### 9. 逻辑错误

#### 9.1 时间逻辑错误
- **示例**：`birth_year = 700`, `death_year = 650`
- **严重性**：高（违反常识）

#### 9.2 关系逻辑错误
- **示例**：A 是 B 的老师，B 是 A 的老师（循环关系）
- **严重性**：中（需人工判断）

#### 9.3 地理逻辑错误
- **示例**：唐代人物出生地为"北京"（应为"幽州"）
- **严重性**：中（历史地名使用错误）

### 10. 数据完整性问题

#### 10.1 推荐字段覆盖率低
- **字段**：`source_chapter`
- **标准**：覆盖率应 >80%
- **处理**：尽量补充

#### 10.2 可选字段覆盖率低
- **字段**：`source_page`、`notes`
- **标准**：覆盖率应 >50%
- **处理**：有则补充，无则不强求

#### 10.3 关键信息缺失
- **示例**：人物记录缺少朝代信息
- **严重性**：高（影响数据可用性）

## 问题严重性分级

### 严重 (Critical)
- 必填字段为空
- 主键/外键错误
- 破坏引用完整性
- 违反核心原则（可溯源性）

### 高 (High)
- 格式严重不规范（书名号、ID 前缀）
- 枚举值错误
- 逻辑错误
- 核心字段为空

### 中 (Medium)
- 推荐字段为空
- 格式轻微不规范
- 数据一致性问题
- 外键为空但可接受

### 低 (Low)
- 可选字段为空
- 格式美观性问题
- 非关键信息缺失

## 自动修复 vs 人工处理

### 可自动修复
- 字段名错误（重命名）
- 书名号缺失（添加《》）
- 枚举值中英文转换（有明确映射）
- ID 前缀缺失（根据表类型添加）
- 编码转换

### 需人工处理
- 必填字段为空（需补充内容）
- 外键指向不存在（需确认正确引用）
- 逻辑错误（需专业判断）
- 数据一致性冲突（需人工决策）
- 复杂格式错误（需理解语义）

## 质量检查清单

### 完整性检查
- [ ] 所有必填字段非空
- [ ] 推荐字段覆盖率 >80%
- [ ] 可选字段覆盖率 >50%

### 准确性检查
- [ ] 书名格式 100% 正确
- [ ] ID 前缀 100% 正确
- [ ] 枚举值 100% 正确
- [ ] 外键引用 100% 有效

### 一致性检查
- [ ] 同一实体 ID 一致
- [ ] 同一实体名称一致
- [ ] 书名格式一致
- [ ] 时间逻辑一致

### 规范性检查
- [ ] 文件编码为 UTF-8-BOM
- [ ] 字段名符合规范
- [ ] 数据类型正确
- [ ] 无特殊字符问题
