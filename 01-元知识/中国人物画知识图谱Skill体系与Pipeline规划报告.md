## 1. 项目背景与目标

### 1.1 项目定位

中国人物画知识图谱项目旨在构建一个基于古代画史文献的结构化知识库，采用 **M3 三层框架**（宏观层-中观层-微观层）组织数据，实现对中国人物画历史的系统性知识表达。

### 1.2 核心目标

- **学术溯源**：所有数据必须可追溯到原始文献（书名、章节、原文）
- **结构化表达**：将非结构化的古籍文本转化为结构化的知识图谱
- **三层架构**：
  - M1 宏观层：时序本体、空间本体、图像志分类
  - M2 中观层：历史人物、社会关系、履历轨迹
  - M3 微观层：作品实体、文献著录、风格技法

### 1.3 数据来源

主要文献包括：
- 《历代名画记》（唐·张彦远）
- 《宣和画谱》（宋）
- 《图绘宝鉴》（元·夏文彦）
- 《画史》（宋·米芾）
- 其他画史、画论文献

## 2. 史记项目 Skill 体系分析

### 2.1 体系架构

史记项目建立了一套成熟的 **双层 Skill 体系**：

#### **元技能层（Meta-Skills）**：14 个通用方法论

基于 `shiji-skills/INDEX.md` 与核心元技能文档，可直接迁移的元能力如下：

1. **反思闭环（00-M01）**：按章反思、按类型反思、多轮收敛。
2. **迭代工作流（00-M02）**：MVP 先行，迭代提质，避免一上来“全量完美化”。
3. **冷启动（00-M03）**：手工样本 -> 规则草案 -> 试点 -> 扩展。
4. **柳叶刀方法（00-M04）**：复杂问题分层拆解，人机分工明确。
5. **知识压缩（00-M05）**：将长文本压缩为可计算、可检索知识单元。
6. **SKILL 演化（00-M06）**：从日志和失败样本回灌规则，持续版本升级。
7. **可读性（00-M07）**：规范命名、文档结构清晰、可协同维护。
8. **标注体系设计（00-M08）**：统一标注边界和类型定义，减少歧义。
9. **提示词工程（00-M09）**：用 SKILL 驱动提示词，而非一次性 Prompt。
10. **质量控制（00-M10）**：规则型 lint 与语义型反思分层协同。
11. **数据体感（00-M11）**：人工抽样观察不可替代，仅统计不足以保真。
12. **数据融合（00-M12）**：多源对齐、消歧、去重与证据保留。
13. **技能迁移（00-M13）**：从“史记场景”迁移到“画史场景”的方法论。

### 2.2 可直接迁移的 Pipeline 机制

史记项目的“00-10”阶段化管线，迁移到人物画项目后，仍保留以下骨架：

- 前置治理：文本校勘、结构切分、命名规范先于抽取。
- 中段抽取：实体 -> 事件/关系 -> 事实三元组逐层提升。
- 后段治理：本体、推理、覆盖度、成本、质量、应用联动。
- 横向管理：Issue/TODO/Git/日志贯穿全流程。

### 2.3 需适配与重构点（人物画场景）

与史记相比，本项目的关键变化：

1. **对象中心变化**：从“政治历史事件”转为“人物-作品-品评”三元核心。
2. **证据形态变化**：大量“著录/品评语句”需要原文粒度保留。
3. **本体重心变化**：M3 微观层（作品、题材、品评）权重显著更高。
4. **关系重心变化**：师承、交游、鉴藏、题跋、著录链路更关键。
5. **时序难点变化**：匿名作品、约略年代、版本互异更常见。

## 3. 两项目对比分析

### 3.1 共性

- 都属于古籍知识工程，依赖文本校勘与结构化抽取。
- 都要求“可溯源、可复核、可迭代”的数据生产机制。
- 都适合“规则 + LLM + 人工抽检”的混合范式。

### 3.2 差异

| 维度 | 史记项目 | 人物画项目 |
|---|---|---|
| 核心对象 | 人物-事件-政体 | 人物-作品-品评 |
| 时间结构 | 纪传体，事件密集 | 画史体，著录与评语密集 |
| 关系主轴 | 政治关系、战争关系 | 师承交游、鉴藏流转、题材风格 |
| 微观证据 | 相对较少 | 极多（尺寸、材质、藏所、评语） |
| 本体重点 | 事件与关系本体 | 图像志与作品本体 |

### 3.3 迁移策略结论

- **可直接复用（约 60%-70%）**：元技能体系、质量控制框架、项目管理框架。
- **需适配（约 20%-30%）**：实体类型、关系类型、章节结构规则、抽取模板。
- **需新建（约 10%-20%）**：作品著录规则、品评语义分层、图像志判别规则。

## 4. 中国人物画知识图谱 Skill 体系设计

### 4.1 总体结构

建议沿用“元技能层 + 管线技能层”双层架构：

1. **元技能层**：继续复用 14 个 META-SKILL 作为跨工序方法论。
2. **管线技能层**：按本项目 M1/M2/M3 的业务目标重排执行重点。

### 4.2 管线技能映射（人物画版）

| 阶段 | 目标 | 对应本项目任务 |
|---|---|---|
| 00 总览 | 定义流程与依赖 | 明确“文献 -> M1/M2/M3”全流程 |
| 01 校勘 | 文本可信 | 统一书名、卷次、标点、异体字 |
| 02 结构 | 结构可抽取 | 卷-条-段切分，评语/著录分区 |
| 03 实体 | 识别人、地、题材 | 人物、地名、题材、机构、术语 |
| 04 事件 | 提取生平轨迹 | 任职、交游、师承、活动、创作 |
| 05 关系 | 构建网络 | 人物关系、作品关系、文献互证 |
| 06 本体 | 统一语义模型 | 映射到 M1/M2/M3 字段与约束 |
| 07 推理 | 发现隐含知识 | 断代推断、同名消歧、矛盾检测 |
| 08 管驭 | 量化质量与覆盖 | 覆盖度、置信度、成本、版本控制 |
| 09 应用 | 形成可用产品 | 检索、时间线、图谱浏览、可视化 |
| 10 管理 | 工程化协同 | Issue/TODO/日志/提交规范 |

### 4.3 面向 M3 Framework 的专项 Skill 组

围绕 `中国人物画知识图谱数据结构v1_4.md`（文档头部标注 v1.6）建议新增/强化以下专项：

1. **M1-时空本体构建 Skill**：朝代-年号-公元映射、古今地名映射、题材层级树构造。
2. **M2-人物网络 Skill**：人物别名归并、履历轨迹标准化、社会关系方向性建模。
3. **M3-作品证据 Skill**：作品实体归档、著录与品评拆分、语句证据与字段对齐。
4. **溯源一致性 Skill**：`source_book/source_chapter/source_text` 必填校验与抽样复核。

## 5. 数据处理 Pipeline 规划

### 5.1 端到端流程

```text
原始文献文本
  -> 校勘与规范化（01）
  -> 结构切分（02）
  -> 初步抽取（03/04/05）
  -> M1/M2/M3 映射入库（06）
  -> 推理与一致性检查（07）
  -> 覆盖度与质量评估（08）
  -> 应用发布（09）
```

### 5.2 冷启动到规模化的执行节奏

依据 META-03（冷启动）与 META-13（迁移）：

1. **Phase A：试点（3-5 本）**
   - 每本抽样 2-3 章，手工标注基线。
   - 生成首版规则与抽取模板。
2. **Phase B：扩展（10 本）**
   - 按书级目录批量抽取，建立错误模式库。
   - 每轮修正规则后回跑样本对比。
3. **Phase C：稳定化（全库）**
   - 固化 lint 与验收阈值。
   - 形成版本化发布节奏（v1/v2/v3）。

### 5.3 五层质量控制（落地到本项目）

复用 META-10 五层 QC，并对应本项目数据：

- **L1 文本完整性**：去标记后与原文一致；文件无缺失。
- **L2 语法正确性**：CSV/Markdown/编号/字段格式合法。
- **L3 数据完整性**：主键唯一、必填字段不空、枚举合法。
- **L4 业务逻辑**：生卒年与活动年不冲突；作者与时代一致性。
- **L5 跨工序一致性**：M2 人物与 M3 作者外键一致；题材引用存在。

### 5.4 溯源字段强制策略

对所有 M1/M2/M3 记录执行强制校验：

1. `source_book`：必须使用书名号格式。
2. `source_chapter`：卷次与条目规则统一。
3. `source_text`：保留关键引文，支持抽样回查。

## 6. 实施路线图

### 6.1 三阶段里程碑

| 阶段 | 时间建议 | 目标产出 | 验收指标 |
|---|---|---|---|
| P0 冷启动 | 1-2 周 | 试点规则、首版抽取模板 | 试点抽样准确率 >= 70% |
| P1 扩展 | 2-4 周 | 10 本书稳定抽取、错误模式库 | 关键字段完整率 >= 95% |
| P2 固化 | 2-3 周 | 质检脚本、版本发布规范 | L1-L3 全通过，L4/L5 重大错误为 0 |

### 6.2 角色分工

- **学术负责人**：定义实体边界、审定争议样本、把关解释口径。
- **数据工程负责人**：脚本、批处理、质检、版本发布。
- **Agent 执行层**：批量抽取、格式修复、差异比对、报告生成。

### 6.3 文档与目录治理

结合 `SKILL_10e_文件组织与目录结构.md`，建议本项目继续遵循：

- `data/`：仅保留核心数据资产，不混入临时分析。
- `doc/`：规范与报告。
- `logs/`：反思、抽样、质检记录。
- `skills/`：稳定方法论。
- `archive/`：淘汰方案与历史版本。

### 7.3 总结

本报告建议以“史记 SKILL 体系”为方法底盘，以“M3 Framework”作为数据语义骨架，通过“冷启动试点 -> 迭代扩展 -> 质量固化”的节奏推进中国人物画知识图谱建设。核心原则是：**先建立可复用流程，再追求极致准确；先保证溯源与一致性，再做高阶推理与应用扩展。**
