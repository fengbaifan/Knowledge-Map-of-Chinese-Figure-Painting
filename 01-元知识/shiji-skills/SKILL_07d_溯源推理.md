# SKILL 07d: 溯源推理 — 史源溯源与传播链构建

> 追溯太史公司马迁获取历史信息的渠道，重建从历史现场到《史记》的传播链。

---

## 一、核心问题

**太史公怎么知道的？**

对于《史记》中的每一个事实、事件和细节，推断其信息来源：
- 是档案文献？还是口述传闻？
- 是第一手材料？还是辗转相传？
- 传播链有几层？每一层是谁？
- 从历史现场到太史公，信息经过了哪些传递？

**本SKILL的界限**：
- ✅ **关注**：史源是什么、传播链如何、几手史料
- ❌ **不关注**：这个史源是否可信、记载是否真实（见SKILL_07e）

---

## 二、史源类型分类

### 2.1 六大史源类型

| 史源类型 | 定义 | 识别特征 | 传播层级 |
|---------|------|---------|---------|
| **档案文献** | 国家档案、诏令、奏疏、典籍 | 太史公明确引用书名、档案 | 0-1层 |
| **家传文献** | 世家族谱、私人记录 | "太史公曰"引"某家之传" | 1-2层 |
| **目击口述** | 当事人或目击者口述 | 太史公游历访谈记录 | 2-3层 |
| **二手传闻** | 民间流传、道听途说 | "世传""闻之""人言" | 3-5层 |
| **多源拼图** | 多个来源拼接重建 | 同一事件多章节记载不同 | 混合 |
| **逻辑推测** | 太史公根据其他事实推断 | 细节过于戏剧化、对话完整 | 不适用 |

### 2.2 传播链层级定义

```
历史现场（事件发生）
    ↓ [0层] 官方档案记录
太史公直接访问档案
```

```
历史现场（亲历者目击）
    ↓ [1层] 亲历者 A
    ↓ [2层] A的子女或门生 B
    ↓ [3层] B告诉太史公
太史公记录
```

**层级越少，史源越直接；层级越多，失真风险越大。**

## 三、六步溯源分析法

本方法论来自案例01-04的实践总结，是《史记》知识库项目的标准溯源流程。

### 第一步：时空可达性分析

**目标**：判断太史公在时间和空间上是否可能接触到该史源。

**分析维度**：

#### 3.1 时间距离
```
时间差 = 事件年份 - 太史公生年(约-145)
```

| 时间差 | 口述传承可行性 | 典型情况 |
|-------|--------------|---------|
| < 30年 | 太史公可能亲见当事人 | 汉武帝时事 |
| 30-70年 | 当事人子女在世，家传一代 | 汉初事件（鸿门宴、荆轲刺秦）|
| 70-150年 | 家传两代，需要世家传承 | 秦末汉初 |
| 150-300年 | 家传三代以上，或依赖文献 | 战国后期 |
| > 300年 | 必须依赖古文献，口述不可能 | 战国前期、春秋 |
| > 500年 | 完全依赖典籍，无口述可能 | 上古史 |

#### 3.2 地理可达性
- 查询太史公游历路线（《太史公自序》）
- 是否到过事件发生地？
- 是否可能接触到当事人后代？

**检测方法**：
- 从事件索引提取事件年代
- 计算时间距离
- 查询太史公游历路线与事件地点匹配度

---

### 第二步：明确引用检查

**目标**：识别太史公明确标注的史源。

**检查位置**：
1. 《太史公自序》：列举所依据的典籍
2. 原文中的引用标记："余读""见于""闻之"
3. 三家注：注明出处的史源线索
4. 章节互引："语具在某传中"

**识别模式**：
```regex
余(读|见|闻|观|访|从|问).*?[《》]
见于[《》].*
语具在.*传中
```

**输出**：
- 若有明确引用 → 史源类型：档案文献
- 若无明确引用 → 进入下一步推断

---

### 第三步：信息性质分析

**目标**：根据信息的公开性和细节程度，推断可能的史源类型。

#### 3.1 公开性判断

| 信息性质 | 定义 | 可能史源 | 示例 |
|---------|------|---------|------|
| **公开事实** | 国家大事、会盟、战争、诏令 | 官方档案、多源记载 | 即位、封禅、战役 |
| **半公开事实** | 朝堂会议、外交谈判 | 官员家传、朝臣口述 | 廷议、使节往来 |
| **私密事实** | 密谋对话、闺房之言 | 知情者泄密、推测补白 | 沙丘之谋、鸿门宴对话 |

#### 3.2 细节程度判断

| 细节程度 | 定义 | 可能史源 | 示例 |
|---------|------|---------|------|
| **宏观叙事** | 某年某战，结果如何 | 档案、史书 | "汉三年，项羽围汉王于荥阳" |
| **中观叙事** | 主要人物行动 | 家传口述 | "樊哙闯帐，项羽赐之酒" |
| **微观细节** | 对话、神态、心理 | 目击者口述或文学加工 | "范增数目项王" |

**推断规则**：
- 公开 + 宏观 → 可能有档案
- 半公开 + 中观 → 可能有家传
- 私密 + 微观 → 史源存疑，需深入分析

---

### 第四步：交叉验证

**目标**：检查同一事件在不同章节的记载，推断共同史源。

**方法**：
1. 查询事件关系数据（from `kg/events/`）
2. 识别同一事件在多章节的记载
3. 对比记载的一致性和差异性

**推断规则**：

| 情况 | 推断 |
|------|------|
| 多章节记载**高度一致** | 可能有共同可靠史源（档案或家传） |
| 记载**略有差异** | 可能来自不同视角的口述（如鸿门宴：项羽方vs刘邦方） |
| 记载**严重矛盾** | 可能来源不同，或某方推测成分多 |
| 某章注明"语具在某传中" | 明确指向史源一致 |

**检测方法**：
- 利用已有的**矛盾检测**成果（SKILL_07 二节）
- 查询事件关系索引
- 对比多章节原文

---

### 第五步：传承链重建

**目标**：重建从历史现场到太史公的信息传播路径。

#### 5.1 人物传承链分析

**关键问题**：
- 事件参与者有哪些？
- 他们的后代在汉朝是否为官？
- 后代是否可能向太史公提供家传？

**查询资源**：
- 人物关系图谱（`kg/relations/`）
- 父子关系、师徒关系
- 汉朝官员名录

**重建步骤**：
```
1. 识别事件参与者（from 事件索引）
2. 查询参与者后代关系（from 人物关系）
3. 检查后代是否在汉朝为官（from 《史记》本文）
4. 推断传承链
```

**案例：鸿门宴传承链**
```
历史现场（前206年）
    ↓
樊哙（亲历者）→ 樊市人 → 樊他广
    ↓              ↓         ↓
  (樊哙本人)    (儿子?)   (孙子/曾孙)
    ↓
太史公生年（约前145年）
    ↓
司马谈/司马迁 访问樊氏后人
```

**传播层级**：2-3层，家传口述

#### 5.2 文献传承链分析

**关键问题**：
- 事件是否有官方档案记录？
- 太史公是否有机会访问档案？
- 典籍如何传世？

**检测方法**：
- 检查《太史公自序》引用的文献目录
- 识别原文中的书名引用
- 三家注的史源线索

**案例：荆轲刺秦传承链**
```
历史现场（前227年）
    ↓
夏无且（太医，亲历者）
    ↓
公孙季功、董生（夏无且门生或友人）
    ↓
太史公（明确说明史源）
```

**传播层级**：2层，亲历者口述

---

### 第六步：综合判断与传播链报告

**目标**：汇总所有证据，输出史源类型和传播链重建。

#### 6.1 综合评估要素

| 要素 | 权重 | 判断依据 |
|------|------|---------|
| **明确引用** | 最高 | 有明确引用 → 档案文献 |
| **时间距离** | 高 | < 70年 → 家传可能；> 300年 → 必依赖文献 |
| **传承链** | 高 | 有后代传承链 → 家传口述；无 → 文献或推测 |
| **交叉验证** | 中高 | 多源一致 → 可靠史源；矛盾 → 存疑 |
| **信息性质** | 中 | 公开+宏观 → 档案；私密+微观 → 存疑 |
| **地理可达性** | 中 | 太史公到过 → 可能访谈；未到过 → 依赖传闻 |

#### 6.2 输出格式：传播链报告

```markdown
### 史源溯源报告：[事件名称]

#### 基本信息
- 事件ID：[事件索引编号]
- 事件时间：[公元纪年]
- 事件地点：[地点]
- 时间距离：[年数]（事件年份 - 太史公生年）

#### 第一步：时空可达性
- 时间距离：[X年]
- 口述传承可行性：[高/中/低]
- 太史公游历路线：[是否到过]

#### 第二步：明确引用检查
- 《太史公自序》：[有/无引用]
- 原文引用标记：[引用内容]
- 三家注线索：[史源说明]

#### 第三步：信息性质分析
- 公开性：[公开/半公开/私密]
- 细节程度：[宏观/中观/微观]
- 推断：[可能史源类型]

#### 第四步：交叉验证
- 相关章节：[章节列表]
- 一致性：[高度一致/略有差异/严重矛盾]
- 史源线索：[互引说明]

#### 第五步：传承链重建

**人物传承链**：
```
历史现场（事件参与者）
    ↓ [1层]
[人物A]（后代/门生）
    ↓ [2层]
[人物B]（汉朝为官/与太史公接触）
    ↓ [3层]
太史公
```

**传播层级**：[X]层

**关键史源人物**：
- [人物1]：[身份]，[与事件关系]，[与太史公关系]
- [人物2]：[身份]，[与事件关系]，[与太史公关系]

#### 第六步：综合判断

**史源类型**：[档案文献/家传文献/目击口述/二手传闻/多源拼图/逻辑推测]

**传播链总结**：
[用一句话总结从历史现场到太史公的传播路径]

**传播层级**：[X]层（[层级说明]）

**可用知识**：
- 人物关系：[引用关系数据]
- 事件关系：[引用事件索引]
- 时间索引：[年代计算]
```

## 六、可自动化程度

| 步骤 | 自动化方法 | 精度 | 数据依赖 |
|------|----------|------|---------|
| 明确引用检查 | 正则匹配+三家注解析 | 高(90%) | 文本标注、三家注数据 |
| 时间差计算 | 事件年代索引查询 | 高(95%) | 事件索引(kg/events) |
| 游历路线匹配 | 地点实体匹配 | 中(70%) | 地点索引、太史公游历数据 |
| 传承链查询 | 关系图谱遍历 | 高(85%) | 人物关系(kg/relations) |
| 交叉验证 | 矛盾检测结果调用 | 高(90%) | SKILL_07矛盾检测 |
| 信息性质判断 | LLM推理 | 中(75%) | 原文+事件上下文 |
| 综合评级 | 规则引擎+LLM | 中高(80%) | 以上所有数据 |

**总体评估**：
- 高度可自动化：明确引用、时间差、传承链、交叉验证
- 中度可自动化：信息性质判断、综合评级
- 建议模式：半自动化（机器初判 + 人工审核）
