## Step 1：启发式检测（发现候选错误）

运行启发式检测脚本，识别8类潜在边界错误：

```bash
python scripts/detect_entity_boundary_errors.py
```

此脚本将检测：
1. 身份词吞噬前文（数量词/范围词被吞噬）
2. 动词被标注吞噬（实体内包含动词）
3. 官职与人名分裂（人名+官职分开标注）
4. 姓氏与名字分裂（姓氏被标注，名字在外）
5. 地名吞噬后缀（地名包含结构助词）
6. 实体内嵌套其他标注（格式损坏）
7. 连续身份标注（可能需要合并）
8. 数量词被其他实体吞噬（非数量实体包含数字）

输出：
- `logs/entity_boundary_errors_report.json` - 完整检测报告
- `logs/entity_boundary_errors_review.tsv` - 人工审查表

预期：检测到数千至上万处"疑似错误"（高误报率）

## Step 2：高置信度错误确认（精确规则）

**关键原则**：不要只看启发式检测结果，要主动搜索已知的系统性误标模式！

### 2.1 "诸X"模式检查

搜索"诸"修饰词后跟官职标注的情况：

```bash
python scripts/find_real_boundary_errors.py --pattern "诸X"
```

或手动检查：
```bash
grep -roh '诸〖[^〗]*〗' chapter_md/*.tagged.md | sort | uniq -c | sort -rn
```

**典型错误**：
- `诸〖;侯〗` → 应为 `〖#诸侯〗`（身份群体，非官职）
- `诸〖;将〗` → 应为 `〖#诸将〗`（身份群体，非官职）
- `诸〖;侯王〗` → 应为 `〖#诸侯王〗`（身份群体，非官职）
- `诸〖^侯服〗` → 应为 `〖#诸侯〗服`（边界切分错误）

### 2.2 "群X"模式检查

搜索"群"修饰词后跟实体标注的情况：

```bash
grep -roh '群〖[^〗]*〗' chapter_md/*.tagged.md | sort | uniq -c | sort -rn
```

**典型错误**：
- `群〖;臣〗` → 应为 `〖#群臣〗`（身份群体，非官职）
- `群〖;公子〗` → 应为 `〖#群公子〗`（身份群体）
- `群〖;公〗` → 应为 `〖#群公〗`（身份群体）
- `群〖#X〗` → 应为 `〖#群X〗`（整体标注）

### 2.3 "众X"模式检查

```bash
grep -roh '众〖[^〗]*〗' chapter_md/*.tagged.md | sort | uniq -c | sort -rn
```

### 2.4 其他修饰词检查

可能的模式：
- `列〖;X〗` → 可能应为 `〖;列X〗`
- `万〖#X〗` → 可能应为 `〖#万X〗`
- `百〖#X〗` → 可能应为 `〖#百X〗`

## 二、边界错误的分类

### 2.1 切分错误（Segmentation Error）

**定义**：正确的实体被错误切分，或修饰词与实体分离。

| 错误类型 | 示例 | 修复后 | 原因 |
|---------|------|--------|------|
| 修饰词被切分 | 诸〖;侯〗 | 〖#诸侯〗 | "诸侯"是整体概念 |
| 复合词被切分 | 诸〖^侯服〗 | 〖#诸侯〗服 | "侯服"不是词汇 |
| 姓名被切分 | 〖@刘〗邦 | 〖@刘邦〗 | 姓名应整体标注 |

### 2.2 分类错误（Classification Error）

**定义**：边界正确，但实体类型错误。

| 错误类型 | 示例 | 修复后 | 原因 |
|---------|------|--------|------|
| 身份误标为官职 | 诸〖;将〗 | 〖#诸将〗 | "诸将"是身份群体 |
| 身份误标为动词 | 入〖#臣〗 | 入臣 | "入臣"="臣服"，是动词 |
| 地名误标为氏族 | 〖&秦〗 | 〖◆秦〗 | "秦"是邦国，非氏族 |

### 2.3 边界+分类双重错误

| 示例 | 问题分析 | 修复后 |
|------|---------|--------|
| 诸〖^侯服〗〖◆秦〗 | 1) 切分错误："侯服"不是词汇<br>2) 分类错误："诸侯"是身份，非制度 | 〖#诸侯〗服〖◆秦〗 |

## 三、已知系统性误标模式

### 3.1 "诸X"模式（高频）

**核心问题**："诸"（各位、众）是修饰词，应与后面的身份词一起标注。

| 错误模式 | 修复后 | 数量 | 原因 |
|---------|--------|------|------|
| 诸〖;侯〗 | 〖#诸侯〗 | 28处 | "诸侯"是身份，非官职 |
| 诸〖;侯王〗 | 〖#诸侯王〗 | 20处 | "诸侯王"是身份群体 |
| 诸〖;将〗 | 〖#诸将〗 | 11处 | "诸将"是身份群体 |
| 诸〖^侯服〗 | 〖#诸侯〗服 | 2处 | "侯服"误切分 |

**历史成因**：早期标注时将"侯"、"将"单独视为官职，忽略了"诸X"整体构成身份群体概念。

### 3.2 "群X"模式（中频）

**核心问题**："群"（所有、众）是修饰词，应与后面的身份词一起标注。

| 错误模式 | 修复后 | 数量 | 原因 |
|---------|--------|------|------|
| 群〖;公子〗 | 〖#群公子〗 | 3处 | "群公子"是身份群体 |
| 群〖;臣〗 | 〖#群臣〗 | 2处 | "群臣"是身份，非官职 |
| 群〖;公〗 | 〖#群公〗 | 2处 | "群公"是身份群体 |
| 群〖#臣〗 | 〖#群臣〗 | 1处 | 应整体标注 |
| 群〖#弟〗 | 〖#群弟〗 | 1处 | 应整体标注 |
| 群〖#母〗 | 〖#群母〗 | 1处 | 应整体标注 |

### 3.3 "入臣"特殊情况（低频但重要）

**核心问题**："入臣"不是身份，而是动词"臣服"。

| 错误模式 | 修复后 | 数量 | 语境示例 |
|---------|--------|------|---------|
| 入〖#臣〗 | 入臣 | 4处 | "韩、魏必入臣於秦"（臣服于秦） |

**辨析**：
- `〖#臣〗` = 名词"大臣"
- `入臣` = 动词"臣服、归顺"
- 判断依据：看前后语境，"入臣於X"结构中的"臣"是动词

## 四、方法论总结

### 4.1 检测方法的演进

本次反思经历了三个阶段：

| 阶段 | 方法 | 发现数量 | 问题 | 改进 |
|-----|------|---------|------|------|
| **第1次** | 简单模式匹配（"诸侯服"） | 2处 | 规则太保守，遗漏大量错误 | 用户质疑："只有这两处吗？" |
| **第2次** | 扩展"诸X"规则 | 59处 | 发现系统性误标，但仍不全面 | 用户提醒："不要只检查诸侯" |
| **第3次** | 全面检查"群X"等 | 14处 | 发现更多边界模式 | 累计修复75处 |

**关键洞察**：
1. **用户反馈至关重要**：初步只发现2处，经两次质疑后发现75处（37.5倍）
2. **系统性错误成批出现**："诸X"、"群X"等修饰词+身份词的组合普遍误标
3. **需要主动搜索**：不能只依赖启发式检测，要主动搜索已知模式

### 4.2 启发式检测的局限

| 局限性 | 表现 | 解决方案 |
|-------|------|---------|
| **误报率高** | 15,332处检测，仅75处真实错误（0.5%） | 使用白名单过滤固定词汇 |
| **语境缺失** | 无法理解"入臣"是动词 | 需要人工审查或NLP工具 |
| **规则维护成本** | 需要持续补充例外情况 | 建立分类白名单体系 |

**示例**：启发式检测误报

```
检测规则: 〖@X〗+单字 → 判定为"姓名分裂"
误报示例:
  〖@黄帝〗者 → ✅ 正确（"者"是判断词，非人名）
  〖@蚩尤〗作乱 → ✅ 正确（"作乱"是谓语动词）
```

### 4.3 三阶段修复法

| 阶段 | 置信度 | 处理方式 | 示例 |
|-----|--------|---------|------|
| **Phase 1** | 高 | 自动批量修复 | 诸〖;侯〗 → 〖#诸侯〗 |
| **Phase 2** | 中 | 生成TSV人工审查 | 万〖#民〗（可能需要合并） |
| **Phase 3** | 低 | 记录待查，不修复 | 复杂嵌套结构 |

## 六、典型案例

### 6.1 案例1：诸侯服边界错误

**原文**：
```
诸〖^侯服〗〖◆秦〗，莫敢〖^合从〗
```

**问题分析**：
1. 边界错误："侯服"不是一个词汇
2. 分类错误："诸侯"是身份（#），不是制度（^）
3. 语义混淆："服"是动词"服从"，不是"侯服"的一部分

**修复后**：
```
〖#诸侯〗服〖◆秦〗，莫敢〖^合从〗
```

**验证**：
- 〖#诸侯〗 = 身份（各国封侯）
- 服 = 动词（服从、归顺）
- 〖◆秦〗 = 邦国

**涉及章节**：086_刺客列传、087_李斯列传

---

### 6.2 案例2：诸将边界错误

**原文**：
```
当是时，诸〖;将〗皆慴服
```

**问题分析**：
1. "诸将"是身份群体概念，不是单纯的官职
2. "诸"是修饰词，应与"将"一起标注

**修复后**：
```
当是时，〖#诸将〗皆慴服
```

**辨析**：
- `〖;将〗` = 单个将领（官职）
- `〖#诸将〗` = 各位将领（身份群体）

**涉及章节**：007_项羽本纪（9处）、006_秦始皇本纪、088_蒙恬列传

---

### 6.3 案例3：入臣语义错误

**原文**：
```
韩、魏不能支秦，必入〖#臣〗於秦
```

**问题分析**：
1. "入臣"是动词"臣服"，不是名词"大臣"
2. 语境："入臣於X"结构表示"臣服于X"

**修复后**：
```
韩、魏不能支秦，必入臣於秦
```

**辨析**：
- `〖#臣〗`（名词） = "大臣、臣子"
- `入臣`（动词） = "臣服、归顺"
- 判断依据：看结构，"入臣於X"表示动作

**涉及章节**：069_苏秦列传（2处）、086_刺客列传（2处）

## 九、经验教训

### 9.1 用户反馈的价值

**案例**：本次反思的错误发现数量演进
- 初步：2处（过于保守）
- 第1次用户质疑："只有这两处吗？不太相信" → 发现59处
- 第2次用户提醒："不要只检查诸侯" → 发现另外14处
- **最终：75处（是初判的37.5倍）**

**启示**：
1. 不要过早满足于初步结果
2. 用户的质疑往往指向真实问题
3. 系统性错误比随机错误更常见

### 9.2 主动搜索优于被动检测

| 方法 | 效果 | 局限 |
|------|------|------|
| **启发式检测** | 发现15,332处"疑似错误" | 误报率99.5%，难以直接使用 |
| **主动搜索已知模式** | 精确发现75处真实错误 | 需要预先知道误标模式 |

**最佳实践**：
1. 先用启发式检测"探路"
2. 人工分析检测结果，提炼真实模式
3. 用精确规则主动搜索全书
4. 批量修复 + 验证

### 9.3 迭代优化的重要性

**三次迭代**：
1. 简单规则 → 2处错误
2. 扩展规则 → 59处错误
3. 全面规则 → 14处错误

**规律**：每次迭代都能发现新问题，不应一次性修复，应分批进行。

## 十一、标注规范的完善

本次反思揭示了以下标注规范需要明确：

### 11.1 修饰词+实体的标注原则

**原则**：修饰词与实体应整体标注，除非修饰词有独立语义。

| 模式 | 标注方式 | 理由 |
|------|---------|------|
| 诸侯 | 〖#诸侯〗 | "诸侯"是固定概念，整体标注 |
| 群臣 | 〖#群臣〗 | "群臣"是固定概念，整体标注 |
| 百姓 | 〖#百姓〗 | "百姓"是固定概念，整体标注 |
| 万民 | 〖#万民〗 | "万民"是固定概念，整体标注 |

**例外**：修饰词与实体分开标注的情况
```
〖$三〗〖;公〗 → 当"三"强调数量时，可分开
〖#三公〗 → 当"三公"指代官制时，整体标注
```

### 11.2 身份/官职/制度的区分标准

| 类型 | 定义 | 典型词汇 | 标记 |
|------|------|---------|------|
| **身份** | 社会地位、群体概念 | 诸侯、群臣、天子、百姓 | `〖#X〗` |
| **官职** | 具体职位、官僚体系 | 丞相、太尉、郡守、县令 | `〖;X〗` |
| **制度** | 制度体系、规则 | 郡县制、封建、三公九卿 | `〖^X〗` |

**难点辨析**：
- "诸侯"是身份（`〖#〗`），不是官职（`〖;〗`）
- "三公九卿"作为官制是制度（`〖^〗`），作为人群是身份（`〖#〗`）

### 11.3 动词与名词的区分

| 词汇 | 名词用法 | 动词用法 |
|------|---------|---------|
| 臣 | 〖#臣〗（大臣、臣子） | 入臣（臣服、归顺） |
| 王 | 〖;王〗（国王） | 王天下（统治天下） |
| 侯 | 〖;侯〗（封侯） | 侯之（封赏） |

**判断依据**：
1. 看语法结构："入臣於X"中"臣"是动词
2. 看语义："臣服"是动作，"臣子"是身份
3. 看搭配：动词后接"於/于"等介词
