# SKILL 07c: 反常推理 — 不符合常识的事实检测与解读

> 从《史记》文本中识别"反常事实"：单条记载本身没有矛盾，但违反常识、违反制度规则、或在当时语境下极为罕见。目标是发现值得深究的历史异常点。

**定位**: 异常检测器 — 快速扫描识别值得深究的异常点，输出研究线索清单。

**与相关SKILL的区别**:
- **vs SKILL_07 矛盾检测**: 矛盾检测关注**两处记载互相打架**；反常推理关注**单条记载本身令人意外**
- **vs SKILL_07e 真实性推理**: 反常推理只做**异常识别+初步分类**；真实性推理做**深度可信度评估+真伪判断**

**工作流程**:
```
SKILL_07c (反常推理) → 输出异常清单 → SKILL_07e (真实性推理) → 输出可信度评级
      ↓                                              ↓
 "这很奇怪"                                    "这是真是假？"
```

## 一、反常类型分类

### 1.1 数字反常

| 子类型 | 判断标准 | 典型案例 |
|--------|---------|---------|
| **兵力异常** | 单支军队超过50万，或双方兵力差距>10:1 | 长平之战赵军四十余万降卒 |
| **年龄异常** | 人物记载行为时年龄超出生理极限或常规 | 廉颇八十余岁仍欲为将 |
| **时间异常** | 事件发展速度违反地理/行军常识 | 数日内跨越千里 |
| **财富异常** | 赏赐/财富数量超出当时国家财政常规 | 陶朱公三致千金 |

### 1.2 行为反常

| 子类型 | 判断标准 | 典型案例 |
|--------|---------|---------|
| **身份越轨** | 庶民/女性/奴隶做了通常只有贵族/男性/自由人才能做的事 | 吕后临朝称制 |
| **亲情反常** | 父杀子、子弑父、手足相残且无明显动机 | 晋献公杀申生 |
| **忠奸反转** | 被《史记》正面评价的人物做了反道德的事，或反之 | — |
| **动机缺失** | 重大行为找不到合理动机，或现有解释自相矛盾 | — |

### 1.3 制度反常

| 子类型 | 判断标准 | 典型案例 |
|--------|---------|---------|
| **礼制违反** | 行为违反当时已知周礼/汉制的明文规定 | 诸侯僭用天子礼乐 |
| **继承异常** | 世系/王位继承跳过通常规则（嫡长子制等）且无记载原因 | — |
| **封赏异常** | 封号/爵位/赏赐规格与功绩明显不相称 | — |
| **刑罚异常** | 同类罪行处置轻重悬殊 | — |

### 1.4 叙事反常

| 子类型 | 判断标准 | 典型案例 |
|--------|---------|---------|
| **沉默异常** | 重要人物/事件在相关章节完全缺席（应有而无） | 秦始皇无皇后记载 |
| **详略失衡** | 次要事件大书特书，重要事件一笔带过 | — |
| **评价矛盾** | 太史公曰与正文叙事传递出相反的态度 | — |
| **时序倒置** | 叙述顺序与事件时序明显不符且无明显原因 | — |

## 二、检测流程

```
结构化数据（实体+事件+关系+本体规则）
    ↓ 加载常识库规则
史记常识库 (kg/common-sense/)
    ↓ 规则扫描（数字阈值/制度违反/关系缺失）
候选反常清单
    ↓ LLM评估（是真反常？还是有常见解释？）
有价值的反常列表
    ↓ 分类标注（类型 + 置信度 + 可能解释）
输出：anomaly_report.json + 研究线索
```

**常识库支撑**: 反常检测依赖 [史记常识库](../kg/common-sense/) 提供的基准规则：
- 数字常识: 人口、兵力、年龄、财富、距离的合理范围
- 制度常识: 爵位、官职、礼制、继承、刑罚的规范
- 社会常识: 等级、性别、宗法的惯例
- 军事常识: 兵力、战术、行军的规律
- 其他: 时间、地理、经济、文化等10大类常识

### 2.1 规则检测（可自动化）

| 规则 | 数据源 | 输出 |
|------|--------|------|
| 军队规模 > 阈值 | 事件索引 `count` 字段 | 候选异常 |
| 人物年龄 > 80 且仍有军事行动 | 生卒年推断 + 事件 | 候选异常 |
| 嫡长子未继位 且无废嫡记载 | 世系关系 + 继承事件 | 候选异常 |
| 重要人物（出现频次>N）无死亡事件 | 实体索引 + 事件索引 | 候选异常 |

### 2.2 LLM评估提示词框架

```
你是一位审视《史记》异常事实的历史学助手。

已知事实：[原文引用]
所属章节：[篇名]
类型标注：[数字反常 / 行为反常 / 制度反常 / 叙事反常]

常识基准：[从 kg/common-sense/ 中引用相关常识]
示例：
- 若为数字反常：引用"单支军队常规规模: 5-20万"
- 若为制度反常：引用"嫡长子继承规则"
- 若为年龄反常：引用"将领出征年龄上限: 通常<65岁"

请判断：
1. 这条记载是否违反[常识/制度/逻辑]？（是/否/存疑）
2. 违反了哪条具体常识？（引用常识库编号，如 [1.1 人口与兵力规模]）
3. 有无通常的历史学解释？（列出1-3条）
4. 若无合理解释，这是否值得深入研究？（高/中/低价值）
5. 推荐追问方向：（一句话）
```

## 七、常识推断方法 (Meta-Skill: 反思迭代)

> **核心思想**: 从具体案例中归纳普适性规律,通过反思迭代不断完善常识库。

### 7.1 常识提取流程 (反思迭代)

```
【第1轮：案例收集】
├─ 输入: 已完成的反常检测案例 (如长平之战、廉颇老矣)
├─ 提问: "这些案例违反了什么共同的规律?"
└─ 输出: 候选常识条目列表

【第2轮：普适性检验】
├─ 输入: 候选常识条目
├─ 提问: "这条规律是否适用于多个案例?"
│         "这条规律是否有明确的时代边界?"
│         "这条规律是否可量化/可验证?"
├─ 反思:
│   ✅ 若适用于3+案例 → 保留
│   ✅ 若有时代标注 → 保留
│   ✅ 若可量化 → 保留
│   ❌ 若仅适用单一案例 → 丢弃
│   ❌ 若无法验证 → 标记为"推测性"
└─ 输出: 精炼后的常识条目

【第3轮：来源标注】
├─ 输入: 精炼后的常识条目
├─ 提问: "这条常识的数据来源是什么?"
│         "置信度如何评估?"
├─ 行动: 标注来源(出土文献/传世典籍/学界共识/统计归纳)
│         标注置信度(高/中/低)
└─ 输出: 规范化常识条目

【第4轮：规则化编码】
├─ 输入: 规范化常识条目
├─ 提问: "如何将这条常识转化为可自动检测的规则?"
├─ 行动:
│   - 数字类常识 → 阈值规则 (如"单支军队 > 50万 → 异常")
│   - 制度类常识 → 违规规则 (如"嫡长子未继位 且 无废嫡记载 → 异常")
│   - 关系类常识 → 缺失规则 (如"君主 无 配偶关系 → 异常")
└─ 输出: 可执行的检测规则

【第5轮：实战验证】
├─ 输入: 可执行的检测规则
├─ 行动: 在全部130篇《史记》中批量扫描
├─ 提问: "这条规则产生了多少候选异常?"
│         "假阳性率是多少?"
│         "是否需要调整阈值或条件?"
├─ 反思:
│   ✅ 若假阳性<20% → 规则有效,录入常识库
│   ⚠️  若假阳性20-50% → 调整阈值,重新验证
│   ❌ 若假阳性>50% → 规则失效,重新设计
└─ 输出: 验证后的常识库条目

【第6轮：持续更新】
├─ 输入: 新的研究成果、考古发现、案例分析
├─ 提问: "现有常识是否需要修正?"
│         "是否有新的常识需要补充?"
├─ 行动: 定期审核 (季度/年度)
└─ 输出: 更新后的常识库版本
```

### 7.2 实例演示：从案例到常识

#### 案例1: 长平之战 → 兵力规模常识

**【第1轮：案例收集】**
- 案例: "长平之战赵军四十余万降卒"
- 问题: 为什么40万让人怀疑?

**【第2轮：普适性检验】**
- 检查其他战役:
  - 巨鹿之战: 秦军数十万
  - 垓下之战: 项羽军10万,汉军数十万
  - 井陉之战: 韩信数万,赵军20万
- 归纳规律: "单支军队常规规模 5-20万,超过30万为异常大军"
- 验证: ✅ 适用于多个案例
- 时代边界: 先秦至汉

**【第3轮：来源标注】**
- 数据源: 《史记》各战役记载统计 + 学界人口史研究
- 置信度: 中-高 (基于统计+学界共识)

**【第4轮：规则化编码】**
```python
# 检测规则
if event.type == "战役" and event.army_size > 300000:
    flag_as_anomaly("兵力异常", confidence="high")
```

**【第5轮：实战验证】**
- 扫描130篇 → 发现5处候选异常
- 假阳性: 1处 (20%) → 规则有效
- 录入常识库: [1.1 人口与兵力规模]

#### 案例2: 廉颇老矣 → 老将年龄常识

**【第1轮：案例收集】**
- 案例: "廉颇八十余岁仍欲为将"
- 问题: 80岁还能带兵吗?

**【第2轮：普适性检验】**
- 检查其他老将:
  - 白起死时约70岁
  - 王翦约65岁灭楚后退休
  - 李广约60岁死于匈奴之战
  - 赵充国70多岁仍征战
- 归纳规律: "将领出征年龄通常<65岁,超过70岁仍征战为罕见"
- 验证: ✅ 适用于多个案例
- 例外: 赵充国、廉颇 (但被视为异常)

**【第3轮：来源标注】**
- 数据源: 《史记》将领生卒年统计 + 人均寿命研究
- 置信度: 高 (明确统计数据)

**【第4轮：规则化编码】**
```python
# 检测规则
if person.role == "将领" and event.type == "征战" and person.age > 70:
    flag_as_anomaly("年龄异常", confidence="medium",
                   note="超出通常年龄上限,但有赵充国等例外")
```

**【第5轮：实战验证】**
- 扫描130篇 → 发现3处候选异常
- 假阳性: 0处 (0%) → 规则有效
- 录入常识库: [1.2 年龄与时间跨度]

#### 案例3: 吕后称制 → 女性参政常识

**【第1轮：案例收集】**
- 案例: "吕后临朝称制" (女性执政)
- 问题: 女性执政违反什么制度?

**【第2轮：普适性检验】**
- 检查其他女性:
  - 秦始皇无皇后记载
  - 窦太后临朝 (幼帝时)
  - 汉武帝废陈皇后
- 归纳规律: "女性通常无官职,仅在太后+幼帝时可临朝摄政"
- 验证: ✅ 适用于多个案例
- 时代边界: 先秦至汉

**【第3轮：来源标注】**
- 数据源: 《礼记》"三从规范" + 《史记》女性记载
- 置信度: 高 (明文制度)

**【第4轮：规则化编码】**
```python
# 检测规则
if person.gender == "女" and person.role in official_positions:

```

**【第5轮：实战验证】**
- 扫描130篇 → 发现2处候选异常 (吕后、窦太后)
- 假阳性: 0处 → 规则有效
- 录入常识库: [5.2 性别规范]

```
    - [ ] 考古实证 (如距离实测)

    - [ ] 统计归纳 (如从《史记》中统计)
    - [ ] 推测性知识 (基于逻辑推导)
Q6: 置信度如何评估?(高/中/低)

【规则化编码】
Q7: 如何将这条规律转化为可自动检测的规则?
    格式: IF [条件] THEN [标记为异常]
    示例: IF 军队规模 > 30万 THEN 标记为"兵力异常"

【假阳性检验】

Q9: 是否需要添加例外条件?

  - 类别: [数字/时间/制度/社会/军事/...]
  - 规则: [一句话描述]

```

### 7.4 常识库维护的反思循环

```
定期审核 (季度/年度)
    ↓
┌─────────────────────────┐
│ 反思问题1: 现有常识是否仍然有效?        │
│ - 有无新的考古发现推翻旧结论?           │
│ - 有无新的学界研究调整数值?             │
└─────────────────────────┘
    ↓ 若需更新
调整常识库
    ↓
┌─────────────────────────┐
│ 反思问题2: 是否有新的常识需要补充?      │
│ - 新完成的案例揭示了什么新规律?         │
│ - 用户反馈哪些常识缺失?                 │
└─────────────────────────┘
    ↓ 若需新增
执行常识提取流程 (7.1)
    ↓
┌─────────────────────────┐
│ 反思问题3: 常识库的组织是否合理?        │
│ - 分类是否清晰?                         │
│ - 是否需要新的专题常识库?               │
└─────────────────────────┘
    ↓ 若需重组
重构常识库结构
    ↓
版本更新 (记录变更日志)
    ↓
重新验证 (批量扫描检测假阳性率)
    ↓
回到定期审核
```

### 7.5 常识提取的优先级

基于项目当前需求,优先提取以下类型常识:

| 优先级 | 常识类型 | 原因 | 预期条目 |
|--------|---------|------|---------|
| **最高** | 数字常识 | 可自动化检测,假阳性率低 | 20+ |
| **最高** | 制度常识 | 有明文依据,规则清晰 | 30+ |
| **高** | 军事常识 | 项羽战争案例需求 | 15+ |
| **高** | 地理常识 | 行军路线验证需求 | 20+ |
| **中** | 社会常识 | 辅助行为反常检测 | 15+ |
| **中** | 时间常识 | 纪年转换需求 | 10+ |
| **低** | 文化常识 | 辅助理解背景 | 10+ |
