# SKILL 08a: 知识颗粒度管理（规划）

> 定义统一的知识单元标准，包含SKU（Standard Knowledge Unit）设计，确保知识的模块化和可复用性。

---

## 一、功能概述

知识库中的知识应该如何切分和组织？过细则碎片化难以理解，过粗则不便复用。本功能计划建立知识粒度管理体系，实现：

- **粒度标准**：定义不同类型知识的合适粒度
- **SKU设计**：标准知识单元的结构和字段
- **组织方式**：知识单元的分类和索引

---

## 二、核心概念

### 2.1 什么是知识颗粒度

知识颗粒度（Knowledge Granularity）指知识单元的大小和复杂度：

- **原子级**：不可再分的最小知识单元（如单个三元组）
- **分子级**：组合多个原子的小型知识单元（如人物简介卡片）
- **聚合级**：主题性的大型知识单元（如完整的人物传记）

### 2.2 SKU标准知识单元

SKU（Standard Knowledge Unit）是标准化的知识产品单元，类似商品库存单位：

- 统一的结构schema
- 明确的字段定义
- 标准化的元数据
- 可独立分发和复用

---

## 三、粒度设计原则

### 3.1 MECE原则

**Mutually Exclusive, Collectively Exhaustive**（互斥且完备）：
- 不同SKU之间无重叠
- 所有知识都能归入某个SKU
- 避免知识冗余和遗漏

### 3.2 用途驱动

**根据使用场景设计粒度**：
- 阅读场景：段落级（Purple Numbers）
- 查询场景：实体级（人物卡片）
- 学习场景：主题级（知识专题）
- 应用场景：功能级（技能卡、策略卡）

### 3.3 可组合性

**知识单元应支持灵活组合**：
- 小单元可组成大单元
- 大单元可拆解为小单元
- 支持多维度的组织方式

---

## 四、SKU类型设计

### 4.1 Factual SKU（事实型）

**定义**：描述客观事实的知识单元

**示例类型**：
- 人物卡片：姓名、生卒年、主要事迹、关系
- 事件卡片：时间、地点、参与者、经过、结果
- 地点卡片：位置、沿革、重要事件
- 制度卡片：名称、设立时间、内容、演变

**粒度标准**：
- 单个实体级别
- 包含核心属性和关键关系
- 200-500字说明

### 4.2 Procedural SKU（过程型）

**定义**：描述方法、策略、模式的知识单元

**示例类型**：
- 军事策略：背水一战、围魏救赵
- 治国方略：休养生息、重农抑商
- 外交手段：合纵连横、远交近攻
- 政治手腕：垂帘听政、挟天子以令诸侯

**粒度标准**：
- 单个策略/方法级别
- 包含背景、步骤、案例、效果
- 300-800字说明

### 4.3 Relational SKU（关系型）

**定义**：描述关系网络的知识单元

**示例类型**：
- 人物关系图：某人物的社会网络
- 事件因果链：某事件的前因后果
- 地理关系网：某地区的行政归属
- 概念关系图：某概念的上下位关系

**粒度标准**：
- 以某实体为中心的关系网
- 包含直接关系和一级间接关系
- 可视化图形+文字说明

## 五、SKU Schema设计

### 5.1 通用字段

所有SKU的共同字段：

```json
{
  "id": "SKU-类型-名称",
  "type": "factual/procedural/relational",
  "title": "标题",
  "summary": "一句话摘要",
  "tags": ["标签1", "标签2"],
  "source": {
    "chapters": ["章节号"],
    "paragraphs": ["段落ID"]
  },
  "created": "创建时间",
  "updated": "更新时间"
}
```

### 5.2 类型特定字段

**Factual SKU额外字段**：
```json
{
  "entity_type": "person/place/event/concept",
  "attributes": {"key": "value"},
  "relations": [{"type": "关系类型", "target": "目标实体"}]
}
```

**Procedural SKU额外字段**：
```json
{
  "domain": "领域",
  "steps": ["步骤1", "步骤2"],
  "cases": ["案例1", "案例2"],
  "effectiveness": "效果评估"
}
```

**Relational SKU额外字段**：
```json
{
  "center": "中心实体",
  "nodes": ["节点列表"],
  "edges": [{"from": "A", "to": "B", "type": "关系"}],
  "visualization": "图形数据"
}
```

## 六、粒度管理工具

### 6.1 SKU生成器

**功能**：从知识图谱自动生成SKU草稿

- 输入：实体/事件/关系数据
- 输出：标准格式的SKU JSON
- 需人工审核和补充

### 6.2 粒度分析器

**功能**：分析现有知识的粒度分布

- 统计不同粒度的知识数量
- 识别粒度不当的知识单元
- 建议合并或拆分

### 6.3 SKU验证器

**功能**：检查SKU是否符合标准

- 字段完整性检查
- 格式规范性检查
- 内容质量检查
