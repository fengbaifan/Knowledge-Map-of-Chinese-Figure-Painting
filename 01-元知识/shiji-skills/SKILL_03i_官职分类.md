# SKILL 03i: 官职职类分类规则

> 为《史记》1532 个 official 实体打上"职类"二级分类（三公/九卿/郡国长吏/军职/爵位/王号…），供 `docs/entities/official.html` 按类筛选。
>
> 原则：**规则沉淀 > 一次拍脑袋**。每发现一批误分类，提炼出规则并记录在本文档里，改进 `classify_officials.py`，重跑即可。
>
> **与 SKILL_03h 地名分类完全对称的工作流**：L1 白名单 → L2 章节上下文 → L2.5《汉书·百官公卿表》 → L3 共现动词/注释 → L4 后缀启发式 → L5 并列传播。

**多标签**：允许一个官职同时拥有多个类别。如"将军"可同时属 `军职` + `泛称`（有时是泛指将领）；"丞相"在特定时期属 `三公` 也是 `宫廷近侍`（近侍天子）。

## 二、分类流水线（决策顺序）

分类脚本 `kg/entities/scripts/classify_officials.py` 按以下层级打分：

```
L1   显式白名单（三公/九卿/列卿/军职/爵位/宫廷/文学/师傅/外邦/上古/泛称）
     + 王号判定（X+王 且 X 是邦国/地名）
L2   章节上下文规则（018-021 侯者年表 → 爵位；110-111 匈奴传 → 外邦职）
L2.5 《汉书·百官公卿表》白名单（KNOWN_HAN_OFFICIALS，由 extract_hanshu_baiguan.py 自动提取）
L3   共现动词/语境规则（读 chapter_md）
     + 三家注注释（读 corpus/shiji/史记集解三家注索隐正义.txt）
L4   后缀启发式（默认兜底）
L5   并列官职传播（同列 〖;X〗、〖;Y〗 属同类）
```

首轮上线顺序：L1 + L4 + L5（基础） → L2.5（批量扩充）→ L3 → L2（章节上下文）。

## 三、L1：显式白名单

手工维护的类别清单：`EXPLICIT_SANGONG` / `EXPLICIT_JIUQING` / `EXPLICIT_MILITARY` / `EXPLICIT_JUE` / `EXPLICIT_PALACE` / `EXPLICIT_SHIFU` / `EXPLICIT_FOREIGN` / `EXPLICIT_ANCIENT` / `EXPLICIT_GENERIC`。

**扩充原则**：
- 单字官名 + 语义无歧义（"相"、"卿"、"尹" 作为官衔）→ 根据时代/上下文决定是否白名单
- 高频且后缀规则误判的专名 → 白名单
- 不确定的 → 不进白名单，留给后续规则处理

## 五、L2.5：《汉书·百官公卿表》白名单

## 八、L5：并列官职传播

**规则**：在 `chapter_md` 正文中，两个 `〖;X〗` 标记之间若只有"、"（可含空白），则视为并列。对多个并列 official 构成的 peer 组，从已分类条目向未分类条目传播（多数决 ≥ 60%）。

**典型触发上下文**：
- `〖;丞相〗、〖;御史大夫〗、〖;太尉〗` — 三公序列
- `〖;丞相〗、〖;太尉〗、〖;廷尉〗、〖;典客〗` — 中央高官序列
- `〖;左贤王〗、〖;右贤王〗、〖;左谷蠡王〗` — 匈奴官序列

**约束**：仅对当前为空字符串的条目生效；已分类条目不被改写。

## 九、已登记的疑难与待办

### 9.1 "王" 的歧义
- `X王`（X 是邦国/地名）→ `王号`
- 单字"王" → `泛称`（帝王泛称）
- `X王`（X 是谥号如 霸/文/武）→ 视条目而定，可能 `泛称` 或 `王号`

### 9.2 "侯" 的歧义
- `X侯` → `爵位`（汉代列侯）
- 单字"侯" → `泛称`
- "列侯"、"彻侯"、"关内侯" → `爵位`
- 谥号+侯（如"安侯"作为官/爵称）→ 需上下文

### 9.3 "相/将/尉/令" 的时代分化
- "相" 战国时期是中央首辅；汉初诸侯王国也设"相"（郡级长官） → `cat-sangong` OR `cat-junli`
- "尉" 可以是 `cat-military`（都尉/校尉）也可以是 `cat-xianli`（县尉）
- "令" 多数是 `cat-xianli`（县令），少数是 `cat-jiuqing`（如 "廷尉令"）

### 9.4 X大夫 的歧义
- "五大夫"、"公大夫"（爵名）→ `爵位`
- "大夫"（身份通称、古职）→ `泛称`
- "谏议大夫"、"光禄大夫" → `文学顾问` 或 `宫廷近侍`

### 9.5 单字官名的处理
- `侯/王/相/将/尉/令/君/公` 等单字被标为 official，优先按"泛称"或根据章节时代分化
- 首轮作为 `cat-generic` 兜底；后续用 L3 精细化
