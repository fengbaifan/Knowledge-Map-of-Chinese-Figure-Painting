# SKILL 09c: 个性化阅读支持 — 阅读辅助开关

> 为读者提供可选的阅读辅助功能，适应不同水平和偏好：语法高亮显示、繁体/简体切换、多音字拼音标注、长段落智能分隔等。

---

## 一、设计理念

**核心原则**：
- **可选性**：用户可自由开启/关闭每个功能，偏好自动保存
- **合理默认**：语法高亮和智能分段默认开启（展示知识库特色），拼音和繁体默认关闭
- **即时性**：一键开启/关闭，无需刷新页面
- **准确性**：使用专用词库，避免通用工具的错误
- **适应性**：满足不同读者群体需求（学者/初学者/现代读者）

**设计哲学**：
- 不是"替代"传统阅读，而是"增强"阅读体验
- 合理的默认状态：展示知识库核心价值（语法高亮、智能分段），同时不过度干扰
- 尊重读者的主动选择，用户可随时调整并自动记忆偏好

## 二、核心功能

### 2.1 语法高亮开关

**功能**：显示/隐藏实体语法高亮着色

**适用场景**：
- 初学者：通过颜色识别实体类型（人名、地名、官职等）
- 学习模式：语法高亮帮助理解文本结构
- 传统阅读：关闭高亮，保持原始阅读体验

**实现**：
- **CSS类控制**：通过body的 `.syntax-highlight-enabled` 类控制
- **18类实体**：人名、地名、官职、时间、朝代等
- **数据准备**：详见 [SKILL_03a 实体标注](SKILL_03a_实体标注.md)
- **渲染方法**：详见 [SKILL_09a 语法高亮辅助阅读](SKILL_09a_语法高亮辅助阅读.md)
- **默认状态**：默认开启，展示知识库特色

**视觉效果**：
- 开启：实体带颜色和下划线，一眼识别关键信息
- 关闭：纯黑色文本，传统古籍阅读体验

**技术细节**：
```css
/* 关闭语法高亮时，移除所有实体样式 */
body:not(.syntax-highlight-enabled) .person,
body:not(.syntax-highlight-enabled) .place,
body:not(.syntax-highlight-enabled) .official {
    color: inherit;
    text-decoration: none;
    background: none;
}
```

### 2.2 繁简体切换

**功能**：简体↔繁体自动转换

**适用场景**：
- 现代读者：习惯简体，但史记原文为繁体
- 繁体偏好者：学术研究或个人偏好繁体
- 对照学习：切换对比繁简差异

**实现**：
- **专用词表**：史记繁简词表v4.0（覆盖130章，621个文件）
- **数据准备**：详见 [SKILL_01e 繁简体处理](SKILL_01e_繁简体处理.md)
- **构造方法**：词频统计 + OpenCC验证 + 人工校对
- **修正错误**：修正通用库常见错误（如"后妃"→"后妃"、"皇后"→"皇后"等）
- **词组优先**：优先匹配多字词，避免单字误转

**技术细节**：
```javascript
// 前端JavaScript实现，基于预构建的词表JSON
function convertToTraditional(text, dict) {
    // 按词长降序匹配，确保"皇后"优先于"后"
    // 使用专用词表，避免通用OpenCC的史记场景错误
}
```

**质量保证**：
- 词表来源：`docs/data/s2t-custom-variants-v4.0.json`
- 验证方法：与维基文库繁体版对比验证
- 覆盖率：130章全部覆盖
- **默认状态**：默认关闭（简体显示），用户可切换为繁体

### 2.3 拼音标注开关

**功能**：显示/隐藏多音字和生僻字的拼音标注

**适用场景**：
- 初学者：不确定读音，需要拼音辅助
- 教学场景：课堂讲解、朗读练习
- 特殊读音：古音、通假字、多音字

**实现**：
- **特殊读音词库**：v3.1版本，基于《史记》实际用字
- **数据准备**：详见 [SKILL_01d 拼音标注](SKILL_01d_拼音标注.md)
- **标注原则**：
  - 只标注多音字（如"乐"yào/"乐"lè）
  - 只标注生僻字和易读错字
  - 不标注常见单音字，避免干扰
- **删除高风险字**：v3.1删除了姓氏单字（易误标），贯彻v3.0设计原则

**标注示例**：
```
原文：高祖乐之
标注：高祖乐(lè)之

原文：召平使人为秋社
标注：召(shào)平使人为秋社
```

**技术细节**：
- 词库文件：`docs/data/special-pronunciation.json`
- 渲染方式：`<ruby>` 标签或上标样式
- CSS控制：通过 `.pinyin-enabled` 类控制显示/隐藏
- **默认状态**：默认关闭，避免过度干扰阅读

**版本演进**：
- v3.0：建立基本词库和标注原则
- v3.0.1：完善规范文档
- v3.0.2：修复"於"字系统性误标问题
- v3.1：删除高风险姓氏单字，减少误标

**相关文档**：
- [SKILL_01d 拼音标注规范](SKILL_01d_拼音标注.md)
- [多音字处理方案总结](../doc/spec/ANALYSIS_多音字处理.md)

### 2.4 智能分段开关

**功能**：显示/隐藏AI辅助的段落分隔符

**适用场景**：
- 长段落阅读：原文段落过长（>500字），阅读疲劳
- 语义理解：AI识别段内语义单元，辅助理解结构
- 现代阅读习惯：习惯短段落的现代读者

**实现**：
- **基础结构**：基于Purple Numbers段落编号系统，详见 [SKILL_02a 章节切分与编号](SKILL_02a_章节切分与编号.md)
- **区块处理**：识别太史公曰、赞等特殊文体，详见 [SKILL_02b 区块与韵文处理](SKILL_02b_区块与韵文处理.md)
- **AI识别**：基于语义分析，识别段内的逻辑单元
- **数据准备**：详见 [SKILL_01e 智能分段](SKILL_01e_智能分段.md)
- **分隔方式**：插入视觉分隔符（如淡色横线、空行）
- **保留原貌**：不改变原文段落结构，只是视觉辅助

**分段原则**：
- 句群边界：识别主题转换、时间跳跃、场景切换
- 因果关系：识别因果关系的起承转合
- 对话分隔：对话与叙述之间
- 列举分隔：并列项之间

**视觉设计**：
```css
/* 智能分段分隔符 */
.smart-segment-divider {
    height: 1px;
    background: linear-gradient(to right, transparent, #ddd, transparent);
    margin: 1.5em 0;
    opacity: 0.5;
}
```

**技术细节**：
- 分段数据：预处理生成，存储在章节JSON中
- JavaScript控制：通过 `.smart-segment-enabled` 类控制显示
- 无侵入性：关闭后完全恢复原始排版
- **默认状态**：默认开启，辅助长段落阅读

**相关文档**：
- [智能分段功能说明](../docs/spec/智能分段功能_说明.md)
- [智能分段功能完成总结](../docs/spec/智能分段功能_完成总结.md)

## 三、用户界面设计

### 3.1 开关位置

```
┌─────────────────────────────────────┐
│  史记 > 高祖本纪                    │
│  ⚙️ 显示设置                        │
│    ☑ 语法高亮  ☐ 繁体显示          │
│    ☐ 拼音注释  ☑ 智能分段          │
├─────────────────────────────────────┤
│  正文内容...                         │
```

**位置**：页面右上角浮动按钮，点击展开配置面板

**特点**：
- 固定位置，始终可见（或sticky定位）
- 紧凑排列，不占用过多空间
- 清晰的开/关状态指示

### 3.2 样式实现

配置面板使用checkbox样式，关键样式文件：
- **HTML结构**：`docs/chapters/*.html` 中的 `<div id="settings-panel">`
- **CSS样式**：`docs/css/settings-panel.css` - 面板容器、checkbox样式
- **JavaScript生成**：`docs/js/settings-panel-config.js` - 动态生成配置项

**设计特点**：
- 浮动按钮（⚙️图标）位于页面右上角
- 点击展开/收起配置面板
- checkbox样式清晰，易于操作
- 响应式设计，适配移动端

### 3.3 状态指示

| 状态 | 视觉 | 说明 |
|------|------|------|
| 关闭 | 灰色背景 | 功能未启用 |
| 开启 | 蓝色背景+白字 | 功能已启用 |
| 悬停 | 浅灰背景 | 可点击提示 |

### 3.4 配置面板说明

配置面板包含四个开关，用户可独立控制每个功能：
- **语法高亮**：显示实体颜色标注（✓ 默认开启）
- **繁体显示**：简体/繁体切换（默认简体）
- **拼音注释**：显示多音字拼音（默认关闭）
- **智能分段**：显示AI辅助分段（✓ 默认开启）

**默认状态说明**：
- 默认开启的功能（语法高亮、智能分段）展示知识库核心特色
- 默认关闭的功能（拼音、繁体）根据用户需求自主开启
- 用户调整后的偏好会保存在浏览器localStorage中

## 十一、常见问题

**Q1: 为什么繁简转换不用通用的OpenCC？**
A: OpenCC是通用工具，在史记场景下会产生错误（如"后妃"误转），史记专用词表通过词频统计和人工校对，准确率更高。

**Q2: 拼音标注是否会标注所有字？**
A: 不会。只标注多音字、生僻字和易读错字，避免过度标注干扰阅读。

**Q3: 智能分段会改变原文吗？**
A: 不会。只是插入视觉分隔符，不改变原文字符和段落结构。关闭后完全恢复原貌。

**Q4: 开关状态是否会保存？**
A: 会。使用localStorage保存用户偏好，下次访问自动恢复。

**Q5: 移动端是否支持？**
A: 支持。开关按钮采用响应式设计，在手机上也能正常使用。
