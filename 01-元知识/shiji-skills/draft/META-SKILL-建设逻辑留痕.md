# 元技能（草稿）: 建设逻辑留痕 — AI时代的可复现性保障

---

## 一、核心思想

> **"数据的价值不仅在于结果本身,更在于它是如何产生的"**

**建设逻辑留痕**（Construction Logic Documentation）是指在知识工程过程中,系统性地记录数据生成的完整逻辑链条,包括推理规则、提示词模板、决策依据、迭代版本,确保任何人都能理解和复现数据的产生过程。

### 核心洞察

```
传统数据记录方式:
  数据文件(data.json) + 代码(script.py) = ❌ 无法复现

问题:
  - AI Agent如何推理出这1000条结果？（提示词丢失）
  - 为什么选择这个分类规则？（决策依据缺失）
  - 第一版到第五版经历了什么演化？（迭代历史不清）
  - 人工审查时做了哪些判断？（审查逻辑未记录）

建设逻辑留痕方式:
  数据文件 + 代码 + 推理规则 + 提示词模板 + 决策过程 + 版本历史

优势:
  - 完全可复现（任何人可重新执行）
  - 完全可追溯（每个数据点可追溯到来源）
  - 完全可审计（决策依据透明）
  - 完全可演化（迭代逻辑清晰）
```

```markdown
# 姓氏推理规则体系（R1-R14）

## R1: 直接记载
**定义**: 原文明确记载人物姓氏
**模式**: "姓X"/"氏X"/"X氏"
**示例**:
  - "高祖，姓刘氏" → 刘
  - "孔子，姓孔" → 孔
**置信度**: 高（原文依据）
**优先级**: 1（最高）

## R2: 邦国姓氏推理
**定义**: 根据人物所属邦国推断姓氏
**前置条件**:
  - 人物有明确国别标记（如"〖&齐&〗〖@管仲@〗"）
  - 该国有主流姓氏记录
**推理逻辑**:
  - 齐国君主 → 姜姓
  - 晋国君主 → 姬姓
  - 楚国君主 → 芈姓
**示例**:
  - "〖&齐&〗桓公" → 姜（齐国君主为姜姓）
**置信度**: 高
**优先级**: 2

## R3: 氏族姓氏推理
**定义**: 根据人物氏族推断姓氏
**前置条件**: 人物有氏族标记
**推理逻辑**:
  - 识别氏族标记（如"〖%公孙%〗"）
  - 查找氏族姓氏对照表
**示例**:
  - "鲁昭公，名〖@稠@〗，〖%孟孙氏%〗" → 姬（孟孙氏为鲁国三桓之一，姬姓）
**置信度**: 中高
**优先级**: 3

... (R4-R14规则展开)

## 规则冲突解决
1. R1直接记载 > 所有推理规则
2. 高置信度 > 低置信度
3. 优先级数字小的 > 优先级大的
4. 冲突时保留多个候选，标记"需人工确认"

## 规则演化历史
```

- 原文两处记载冲突
- 需要确定哪个更可靠

**人工决策**:
1. 查阅编年表:《中国历史大事年表》记载"孔子卒于鲁哀公十六年（-479）"

- 虚岁73岁 = 实岁72岁
- -551 至 -479（实岁72）

**经验教训**:
- 古代年龄常用虚岁，需要转换
- 多源交叉验证必不可少

## 三、留痕的核心价值

### 价值1: 完全可复现

**场景**: 新团队成员接手项目
```
传统方式:
  "这1000条数据是怎么来的？"
  → "不知道，是AI生成的"
  → "提示词呢？"
  → "丢了..."
  → ❌ 无法复现

建设逻辑留痕:
  "这1000条数据是怎么来的？"
  → 查看`prompts/templates/xxx_v2.1.md`（提示词）
  → 查看`SKILL_07b_姓氏推理.md`（规则）
  → 查看`logs/infer_xingshi_2026-03-16.md`（执行日志）
  → ✅ 完全复现
```

### 价值2: 完全可追溯

**场景**: 发现数据错误
```
传统方式:
  "为什么商鞅的姓氏是'姬'？"
  → "不知道，应该是AI推理的"
  → ❌ 无法追溯

建设逻辑留痕:
  "为什么商鞅的姓氏是'姬'？"
  → 查看`kg/entities/data/xingshi_reasoning/商鞅.md`

  → 证据: "公孙鞅，卫国人，卫为姬姓诸侯国"
  → ✅ 追溯到原文依据
```

**场景**: 学术论文审查
```
传统方式:
  "你的数据如何保证准确性？"
```

## 六、反模式（Anti-patterns）

### 反模式1: 只记录结果，不记录过程

**错误做法**:
```
data.json  # 只有最终数据
```

**正确做法**:
```
data.json                        # 最终数据
prompts/templates/xxx.md         # 提示词模板
SKILL_07b_规则.md               # 推理规则
logs/reasoning_process.md        # 推理过程
doc/version_history.md           # 版本历史
```

---

### 反模式2: 提示词散落各处

**错误做法**:
```python
# ❌ 提示词硬编码在代码中
prompt = "你是专家，请推理..."
```

**正确做法**:
```python
# ✅ 提示词模板化存档
prompt_template = load_file('prompts/templates/xxx_v2.1.md')
prompt = prompt_template.format(**context)
```

---

### 反模式3: 人工决策无记录

**错误做法**:
```
人工审查 → 修改数据 → 无记录
```

**正确做法**:
```
人工审查 → 生成决策报告 → 修改数据 → 关联报告ID
```

---

### 反模式4: 版本演化无文档

**错误做法**:
```
v1.0 → v2.0 → v3.0 （无变更记录）
```

**正确做法**:
```
v1.0 → v2.0 → v3.0
  ↓      ↓      ↓
v1.0   v2.0   v3.0
文档   文档   文档
```
