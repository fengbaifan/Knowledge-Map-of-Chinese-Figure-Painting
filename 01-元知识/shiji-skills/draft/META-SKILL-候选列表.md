# 元技能候选列表

基于项目git演化过程和最近两周工作实践分析（2026-03-26），识别出以下尚未系统化的元技能候选。

## 优先级分类

### 高优先级（建议优先文档化）

#### 批量反思管线
- **核心思想**: 将大规模质量审查拆解为"提示词生成→Agent执行→结果审查→批量应用"四阶段管线
- **解决问题**: 130章文本如何高效进行多轮质量反思
- **应用实例**:
  - 事件年代五轮反思（累计2119处修正）
  - 实体标注三轮反思（22章，99处错误）
  - 姓氏推理R1→R6c迭代
- **可迁移性**: 高 - 适用于任何大规模语料质量提升任务
- **状态**: ✅ 已完成初稿

#### 增量式数据更新
- **核心思想**: 保留已有数据成果，通过合并/覆盖机制避免重复劳动和数据丢失
- **解决问题**: AI Agent多轮推理产生的数据如何持续积累而不被覆盖
- **应用实例**:
  - `update_xingshi_from_md.py`: MD(616人) + 旧JSON(2053人) → 合并(2095人)
  - 姓氏推理v1.0→v2.1增量更新
  - 表格数据`publish_tables_data.py`增量复制
- **可迁移性**: 高 - 适用于任何需要多轮迭代的知识工程项目
- **状态**: ✅ 已完成初稿（已简化代码）

#### SKILL驱动的Agent协作
- **核心思想**: 将专家知识编码为SKILL文档，作为Agent执行任务的方法论输入
- **解决问题**: 如何让AI Agent理解领域专业知识并保持一致性
- **应用实例**:
  - 41个SKILL文件（YAML frontmatter结构化）
  - 三层SKILL架构（基础层→中间层→高级层）
  - 司马迁文风SKILL实验
- **可迁移性**: 高 - AI时代的核心方法论
- **状态**: 🔄 可参考现有SKILL_00_管线总览（暂不需要单独文档）

#### Lint工具链
- **核心思想**: 构建自动化检查工具链，保障数据质量的多个维度
- **解决问题**: 如何确保标注不破坏原文完整性
- **应用实例**:
  - `lint_text_integrity.py`: 去标注后逐字比对
  - 白名单机制（合理校勘差异）
  - `verify_table_links.py`: 链接验证
- **可迁移性**: 高 - 适用于任何文本处理项目
- **状态**: ⏳ 待写（优先级降低）

#### 建设逻辑留痕
- **核心思想**: 不仅记录数据和代码，更记录数据生成过程和决策依据
- **解决问题**: AI Agent生成的数据如何保证可复现和可追溯
- **应用实例**:
  - `lifespan_reasoning_process.md`(986行)
  - 五轮事件年代反思全流程文档
  - 22章实体反思报告
- **可迁移性**: 高 - 适用于任何涉及复杂推理的AI项目
- **状态**: ✅ 已完成初稿

### 中优先级（可后续补充）

#### META-20: 三层索引架构
- **核心思想**: 构建"类型索引+专项索引+关系索引"三层体系，支持多维度知识访问
- **应用实例**: 18类实体索引、谥号/姓氏/韵文专项索引、关系索引
- **可迁移性**: 高

#### META-21: 内联标注语法
- **核心思想**: 设计极简且不破坏原文可读性的标注语法，支持嵌套和消歧
- **应用实例**: `〖TYPE 内容〗`统一格式、`〖@项羽|项籍〗`消歧语法
- **可迁移性**: 高

#### META-22: 混合架构设计
- **核心思想**: 混合使用多种技术栈发挥各自优势
- **应用实例**: Markdown(源)+JSON(数据)+HTML(展示)+Python(转换)+JavaScript(交互)
- **可迁移性**: 高

#### META-23: 交互式数据查看器模式
- **核心思想**: 在静态站点中嵌入富交互组件，提供类似数据库查询的体验
- **应用实例**: ag-Grid十表查看器、姓氏查询表
- **可迁移性**: 高

#### META-24: 多源数据融合验证
- **核心思想**: 利用多个独立数据源的交叉验证发现和修正错误
- **应用实例**: 事件年代外部参照、十表年表交叉验证、姓氏推理多规则验证
- **可迁移性**: 中
