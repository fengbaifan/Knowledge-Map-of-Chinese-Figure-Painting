## 技能目标

对《史记》十表中的表格内容进行校勘，确保原文完整性，统一表格格式为Markdown，为后续的表格语义分析提供可靠基础。

## 适用范围

- 《史记》十表（013-022章）的表格校勘
- 其他包含表格结构的古籍文献

## 核心原则

1. **维基文库为校勘底本**: 《史记》十表的表格结构以维基文库版本为准
2. **原文完整性优先**: 表格校勘必须严格遵守标注铁律，不得修改原文字符
3. **格式归一化**: 所有表格统一使用Markdown格式表示
4. **表头明确性**: 表头的有无、列数、列名必须与维基文库一致
5. **分隔符保留**: 原文中的表格分隔标记（如"XXX... :"）必须保留
6. **合并单元格处理**: 维基文库使用colspan的地方，需要在Markdown中等价表示

## 版本资源与校勘底本

### 十表专用校勘底本

**《史记》十表的表格结构以维基文库版本为唯一权威底本。**

**版本说明**:

| 版本 | 文件位置 | 表格准确性 | 适用场景 |
|------|---------|-----------|---------|
| **维基文库** | `corpus/shiji/wikisource_shiji/` | ⭐⭐⭐⭐⭐ | **表格结构校勘的唯一底本** |
| 简体底本 | `docs/original_text/` | ⭐⭐ | 仅作参考，存在表头缺失、列数错误等问题 |
| 繁体版本 | `corpus/shiji/` | ⭐⭐⭐ | 辅助参考 |

**维基文库为校勘底本的理由**:

1. **表格结构完整**: 包含所有表头、列名、合并单元格信息
2. **历史准确性**: 表格列数和列名符合历史事实（如016章项羽分封18王，维基文库有21列，简体底本仅7列）
3. **格式清晰**: HTML表格使用`<th>`、`colspan`等标签明确表示表格结构
4. **数据匹配**: 表头列数与数据行列数一致

**具体案例**:

016_秦楚之际月表的第二张表"项羽分封后"：
- **维基文库版本**: 21列（公元前 + 楚 + 項系4国 + 趙系2国 + 齊系3国 + 漢系4国 + 燕系2国 + 魏系2国 + 韓系2国）
- **简体底本版本**: 7列（秦、楚、项、赵、齐、汉、燕），**缺少魏、韩两国**
- **校勘结论**: 采用维基文库版本，简体底本需要补充

详见：[logs/curation/collation_016_项羽分封后表头.md](../logs/curation/collation_016_项羽分封后表头.md)

**操作规范**:

1. **表头确认**: 优先查阅维基文库HTML文件的`<table>`结构
2. **列数确定**: 以维基文库的列数为准（包括合并单元格展开后的列数）
3. **列名确定**: 以维基文库的`<th>`内容为准
4. **简体底本修正**: 如果简体底本表头与维基文库不符，需要修正简体底本
5. **校勘记录**: 重大差异必须记录在`logs/curation/`目录

## 已发现的表格校勘问题

### 问题1: 表头缺失或错误

**案例**: 015_六国年表

**问题描述**:
- 原txt文件缺少明确的表头行
- 标注文件中添加了Markdown表头 `| 周 | 秦 | 魏 | 韩 | 赵 | 楚 | 燕 | 齐 |`
- 完整性检查报错："插入（标注多字）"

**解决方案**:
```
1. 在原txt文件中添加纯文本表头（每行一个国名）：
   周
   秦
   魏
   韩
   赵
   楚
   燕
   齐

2. 在标注文件中添加对应的Markdown表头：
   |  | 〖◆周〗 | 〖◆秦〗 | 〖◆魏〗 | 〖◆韩〗 | 〖◆赵〗 | 〖◆楚〗 | 〖◆燕〗 | 〖◆齐〗 |
   | --- | --- | --- | --- | --- | --- | --- | --- | --- |
```

**校勘记录**: 无（原文本身缺失，通过其他版本补充）

---

### 问题2: 表头列名与原文不符

**案例**: 017_汉兴以来诸侯王年表

**问题描述**:
- 标注文件中有Markdown表头 `| 纪年 | 楚 | 齐 | 荆 | 淮南 | 燕 | 赵 | 梁 | 淮阳 | 代 | 长沙 |`
- 原txt文件中无对应表头，直接是数据行："高祖元年\n楚。\n齐。..."
- 完整性检查报错："插入（标注多字）"

**解决方案**:
```
删除Markdown表头，因为原txt中不存在：
汉兴以来...  :
|  [r1]  |  〖@高祖〗〖%元年〗  |  楚。  |  ...
```

**原则**: 如果原文没有表头，标注文件也不应添加表头（即使为了可读性）

### 问题3: 表格分隔符遗漏

**案例**: 014_十二诸侯年表

**问题描述**:
- 表格末尾缺少："十二诸侯... :\n太史公曰：儒者断其义..."
- 完整性检查报错："删除（标注少字）"

**解决方案**:
```
在表格最后一行后添加：
|  [r365]  |  ...  |

十二诸侯... :
〖@太史公〗曰：〖_儒者〗断其义，驰说者骋其辞，不务综其终始；历人取其年月，数家隆於神运，谱谍独记世谥，其辞略，欲一观诸要难。於是谱十二〖#诸侯〗，自〖%共和〗讫〖@孔子〗，表见〖^春秋〗、〖^国语〗〖#学者〗所讥盛衰大指著於篇，為成学治古文者要删焉。
```

**校勘记录**: 无（标注时遗漏，不是异文）

### 问题4: 赞文缺失

**案例**: 013_三代世表、019_惠景间侯者年表、021_建元已来王子侯者年表

**问题描述**:
- 标注文件末尾删除了赞文（认为是后人补写）
- 但赞文在原txt中存在，应该保留

**解决方案**:
```
1. 在原txt文件末尾添加/恢复赞文（如013章）：
   高辛之胤，大启祯祥。脩己吞薏，石纽兴王。天命玄鸟，简狄生商。姜嫄履迹，祚流岐昌。俱膺历运，互有兴亡。风馀周召，刑措成康。出彘之后，诸侯日彊。

2. 在标注文件中添加对应的标注版本：
   ## 赞

   ::: 赞
   [10] 〖?高辛〗之胤，大启祯祥。
   〖@脩己〗吞〖+薏〗，〖=石纽〗兴王。
   〖_天命〗〖?玄鸟〗，〖@简狄〗生〖◆商〗。
   〖@姜嫄〗履迹，祚流〖=岐昌〗。
   俱膺历运，互有兴亡。
   风馀〖@周召〗，刑措〖@成康〗。
   出〖=彘〗之後，〖#诸侯〗日彊。
   :::
```

**校勘说明**: 赞文是否为司马迁原作存疑，但既然在底本中存在，应予保留并标注

### 问题6: 多表换表头

**案例**: 016_秦楚之际月表

**问题描述**:
- 016章有两张表："主表"和"项羽分封后"表
- 第二张表的表头可能与第一张表不同
- 原Agent修复时曾将第二张表的20列表头简化为9列

**原始情况**:
```
第一张表表头：秦、楚、项、赵、齐、汉、燕、魏、韩（9列）
第二张表表头（错误）：秦、西楚、衡山、临江、九江、常山、代、临菑、济北、胶东、汉、雍、塞、翟、燕、辽东、西魏、殷、韩、河南（20列）
```

**问题分析**:
- 第二张表的20列确实是项羽分封后的诸侯国
- 但原txt中第二张表的列名与第一张表相同（都是9列）
- 标注文件不应擅自扩展列名

**解决方案**:
```markdown
### 项羽分封后

秦楚之际... :
|  | 〖◆秦〗 | 〖◆楚〗 | 项 | 〖◆赵〗 | 〖◆齐〗 | 〖◆汉〗 | 〖◆燕〗 | 〖◆魏〗 | 〖◆韩〗 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  [r33]  |  ...  |  ...  |  ...  |  ...  |  ...  |  ...  |  ...  |  ...  |  ...  |
```

**原则**:
1. **多表换表头必须以原文为准**: 即使数据列数增加，如果原文表头未变，标注文件也不能改变
2. **多版本对校**: 表头列名需要多版本互校确认
3. **校勘记录**: 如果不同版本的表头不一致，需要记录在校勘日志中

**校勘记录**:

根据维基文库校勘，016章第二张表"项羽分封后"的表头应该使用合并单元格表示分封后的诸侯国扩展。

详见：[logs/curation/collation_016_项羽分封后表头.md](../../logs/curation/collation_016_项羽分封后表头.md)

**校勘结论**:
```
维基文库版本使用了更详细的表头：
- 公元前（1列）
- 楚（1列）
- 項（colspan=4，包含：西楚、衡山、临江、九江）
- 趙（colspan=2，包含：常山、代）
- 齊（colspan=3，包含：临菑、济北、胶东）
- 漢（colspan=4，包含：汉、雍、塞、翟）
- 燕（colspan=2，包含：燕、辽东）
- 魏（colspan=2，包含：西魏、殷）
- 韓（colspan=2，包含：韩、河南）

简体底本版本使用简化表头：
- 秦、楚、项、赵、齐、汉、燕（7列，缺魏、韩）

采用方案：以维基文库为准，原txt需要补充完整表头
```

## 表格格式规范

### Markdown表格结构

```markdown
## 表

[分隔标记]  ← 如"十二诸侯... :"、"汉兴以来...  :"（如果原文有）
|  | 列1 | 列2 | 列3 | ... |  ← 表头行（如果原文有）
| --- | --- | --- | --- | --- |  ← 分隔行（必须，Markdown规范）
|  [r1]  | 数据 | 数据 | 数据 | ... |  ← 数据行
|  [r2]  | 数据 | 数据 | 数据 | ... |
...
```

### 关键规则

1. **表头的有无**:
   - 有表头的章节（如015、016）: 必须保留表头
   - 无表头的章节（如017）: 不得添加表头

2. **分隔标记**:
   - 原文中的"XXX... :"标记必须保留
   - 位置：表格前或表格间（取决于原文）

3. **Markdown分隔行**:
   - `| --- | --- | ... |` 是Markdown规范要求
   - lint脚本会自动删除此行后再与原文对比
   - 因此此行不影响完整性检查

4. **行号标记**:
   - `[r1]`、`[r2]` 等行号标记放在第一列
   - 便于定位和引用

## 校勘流程

### 第一步：确认表头

```bash
# 查看原txt文件的表格开头部分
sed -n '开始行号,结束行号p' docs/original_text/XXX章.txt

# 对比其他版本（维基文库等）确认表头
```

**判断标准**:
- 如果原文有独立的列名行 → 有表头
- 如果原文直接是数据行 → 无表头
- 如果不确定 → 多版本互校

### 第二步：归一化为Markdown

```markdown
1. 确定列数（包括行号列）
2. 添加Markdown表头（如果原文有）
3. 添加Markdown分隔行 | --- | --- | ...
4. 逐行转换数据
```

### 第三步：完整性验证

```bash
# 运行完整性检查
python scripts/lint_text_integrity.py --chapters XXX

# 查看详细报告
cat logs/lint_text_integrity_XXX.txt
```

### 第四步：修正错误

根据lint报告的三类错误修正：
- **插入（标注多字）**: 删除多余内容
- **删除（标注少字）**: 补回缺失内容
- **替换**: 核对原文，修正字符

### 第五步：生成校勘记录

如果涉及异文，在 `logs/curation/` 目录下创建校勘记录：

```markdown
# collation_XXX_问题描述.md

## 问题定位
章节: XXX章
位置: 第XX行

## 异文对照
| 版本 | 文本 |
|------|------|
| 简体底本 | ... |
| 维基文库 | ... |
| 四库版本 | ... |

## 判断依据
...

## 采用方案
底本: ...
理由: ...
```

## 白名单机制

对于表格结构差异（非异文），使用白名单豁免：

```python
# scripts/lint_whitelist.txt
013_三代世表:表格标题行和内容重组:已确认的结构化展示
```

**适用场景**:
- Markdown表格重组了原文的线性数据
- 添加了行号标记 `[r1]`、`[r2]` 等
- 表格标题行（如果原文隐含但未明写）

**不适用场景**:
- 表头列名与原文不符
- 数据内容有增删改
- 分隔标记遗漏

## 常见问题FAQ

### Q1: Markdown表头与原文格式不同怎么办？

**A**: Markdown表头是为了可读性，但必须基于原文：
- 如果原文有列名（任何形式） → Markdown表头必须对应
- 如果原文无列名 → Markdown也不能添加表头
- 原文的纯文本列名（如015章） → 需要同步到txt中

### Q2: "XXX... :"这类分隔符是什么？

**A**: 这是原文中的表格分隔标记，功能类似于"表格开始"或"段落分隔"：
- 014章："十二诸侯... :"
- 015章："六国年表" 无此标记
- 016章："秦楚之际... :" (多次出现)
- 017章："汉兴以来...  :"

这些标记必须保留在标注文件中。

### Q3: 赞文是否应该保留？

**A**:
- 如果赞文在底本中存在 → 必须保留
- 赞文可能是褚少孙等后人补写，但仍是底本的一部分
- 可以在校勘记录中说明赞文来源

### Q4: 表格完整性检查失败怎么办？

**A**: 按以下顺序排查：
1. 检查表头是否与原文一致
2. 检查分隔标记是否遗漏
3. 检查是否有字符增删改
4. 查看lint详细报告确定具体位置
5. 对照原txt文件逐字核对

### Q5: 如何处理多表章节？

**A**:
1. **识别多表**: 查看Markdown中是否有多个`### 表名`或多个分隔标记
2. **表头独立校勘**: 每张表的表头都需要独立校勘，不能假设相同
3. **行号连续**: 多张表的行号应该连续（如[r1]-[r32]，[r33]-[r50]）
4. **分隔标记**: 每张表前的分隔标记都要保留
5. **案例参考**: 见016章"秦楚之际月表"的两张表格

### Q6: 多表的第二张表表头与第一张不同怎么办？

**A**:
- **原则**: 严格以原txt为准，不得擅自修改
- **校勘**: 如果不同版本表头不一致，需要多版本对校
- **记录**: 表头差异需要记录在校勘日志中
- **示例**: 016章第二张表虽然数据有20列，但表头仍为9列（与原文一致）
