# SKILL 03h: 地名地段分类规则

> 为《史记》2116 个 place 实体打上"地段"二级分类（水域/山脉/郡/县/城邑…），供 `docs/entities/place.html` 按类筛选。
>
> 原则：**规则沉淀 > 一次拍脑袋**。每发现一批误分类，提炼出规则并记录在本文档里，改进 `classify_places.py`，重跑即可。

## 二、分类流水线（决策顺序）

分类脚本 `kg/entities/scripts/classify_places.py` 按以下层级打分：

```
L1   显式白名单（区域/水域/原野/山脉/关隘/建筑/陵墓/国家/诸侯王国）
     + 郡白名单（KNOWN_COMMANDERIES ∪ 汉书地理志提取 ∪ HAN_VASSAL_KINGDOMS）
L2   章节上下文规则（侯者年表 → 县）
L2.5 汉代县名白名单（KNOWN_HAN_XIAN ∪ 汉书地理志提取 ∪ ）
L3   共现动词/词汇规则（读 chapter_md）
     + 三家注注释（读 corpus/shiji/史记集解三家注索隐正义.txt）
L4   后缀启发式（默认兜底）
L5   并列地名传播（同列 〖=X〗、〖=Y〗 属同类）
```

全部已实现（2026-04-18）。

## 三、L1：显式白名单（已实现）

手工维护的类别清单：`EXPLICIT_REGION` / `EXPLICIT_HAI` / `EXPLICIT_RIVER` / `EXPLICIT_MOUNTAIN` / `EXPLICIT_PASS` / `EXPLICIT_BUILDING` / `EXPLICIT_TOMB` / `KNOWN_COMMANDERIES`。

**扩充原则**：

- 单字地名 + 语义无歧义（"海"、"江"、"河" 指黄河/长江）→ 白名单
- 高频且后缀规则误判的专名 → 白名单
- 不确定的 → 不进白名单，留给后续规则处理

## 四、L2：章节上下文规则

### 4.1 汉代侯者年表 → 县（**已实现** @ 2026-04-18）

**规则**：若 `place.refs` 中出现在以下章节的频次 ≥ 所有引用的 50%，且 L4 后缀规则给出的类别是空或"城邑"（弱类别），则改判为 **县**（汉代侯国 = 县级行政区）。

涉及章节（前 3 位数字匹配）：

- `018_高祖功臣侯者年表`
- `019_惠景间侯者年表`
- `020_建元以来侯者年表`
- `021_建元已来王子侯者年表`

**保护类别**（不被改写）：郡、州、山脉、水域、建筑、陵墓、关隘、区域、海野、乡里、道桥、以及 L1 白名单中已显式分派者。

**关键参数**（`classify_places.py`）：
- `HOUZHE_MAJORITY = 0.5`
- `HOUZHE_MIN_REFS = 1`（单条 ref 命中即可，因大量冷僻侯国只在年表出现一次）

**已知误判**：
- `楚国`/`瓯骆` 之类在年表中以"来源王国"身份出现的，会被误判为县，实际是王国/部族。
- 对策：逐步追加显式清单（参见 §7.3）。

### 4.2 河渠书、封禅书、天官书 → 线索（待实现）

- `029_河渠书` 中大量 `〖=X〗` 通常是河流/运河
- `028_封禅书` 中大量 `〖=X〗` 通常是山岳 / 祭祀建筑
- `027_天官书` 中极少出现地名（多为星官）

## 五、L3：共现动词与词汇规则（**部分已实现** @ 2026-04-18）

当前实现：扫描 `chapter_md/*.tagged.md`，对每个 place 累积"类别票数"（模式命中次数）。若主导类别占比 ≥ 60% 且 L1/L2/L4 未给出强类别，采纳此推断。

已实现的模式（`classify_places.py` 中 `CONTEXT_PATTERNS`）：

| 模式 | 推断 | 示例 |
|------|------|------|
| `都/徙都/建都/居〖=X〗` | 城邑 | 都咸阳 → 咸阳 |
| `城/筑〖=X〗` | 城邑 | 城武阳 → 武阳 |
| `渡/济/涉/临/溯/游〖=X〗` | 水域 | 渡河 → 河 |
| `登/上/陟/望〖=X〗` | 山脉 | 登泰山 → 泰山 |
| `禅(于)?〖=X〗` / `〖^禅〗〖=X〗` / `登封〖=X〗` | 山脉 | 禅梁父 → 梁父 |
| `〖=X〗(谥号)?侯`（非 "X侯国"） | 县 | 安陵侯 → 安陵 |
| `封…为〖=X〗侯` | 县 | 封萧何为酂侯 → 酂 |
| `封…于〖=X〗` | 县 | 封周章弟于夏虚 → 夏虚 |
| `治〖=X〗` | 郡 | 治临淄 → 临淄（郡治）|
| `〖=X〗〖;...太守〗` | 郡 | 南郡太守 → 南郡 |
| `葬(于)?〖=X〗` | 陵墓 | 葬霸陵（最小命中 2）|
| `会/盟〖=X〗` | 城邑 | 会柯、盟夹谷 → 会盟地 |
| `献/割/予〖=X〗` | 城邑 | 韩献垣雍、割上庸 |
| `迁/徙〖=X〗` | 城邑 | 迁州来、徙莱 |
| `战/败(于/於)〖=X〗` | 城邑 | 战河曲、败澮 |

## 五·一、L2.5：汉代郡县白名单（**已实现** @ 2026-04-18）

**数据源**：
- 手工精选：`classify_places.py::KNOWN_HAN_XIAN`（100+ 个史记高频县名）
- 自动提取：`kg/entities/data/hanshu_dili.json`，由 `extract_hanshu_dili.py` 从 `corpus/other/汉书.txt` 的《地理志》全文解析而来。首次运行提取 **104 个郡 + 1049 个县**。

**加载与应用**：
- 启动时 `_load_hanshu_dili()` 加载 json，合并到内存中的郡/县集合
- 郡名同时剥去 "X郡/X国/X尹/X内史/X冯翊/X扶风" 后缀（史记中多不带后缀）后并入 `KNOWN_COMMANDERIES`
- L1 直接命中 → 郡
- L2.5：主分类为空或"城邑"时，若在县集合内 → 县

**更新方式**：
1. 若《汉书》文本更新 → 重跑 `python3 kg/entities/scripts/extract_hanshu_dili.py`
2. 若发现误分类：扩充 `KNOWN_HAN_XIAN` 手工清单，或修正 `extract_hanshu_dili.py` 的过滤规则

**已知局限**：
- 《汉书·地理志》反映的是西汉末行政区划，不完全等同于司马迁时代；少量战国都邑（如临淄/大梁）在汉代是县，但史记中多作为列国都会提及，目前归为"城邑"仍合理
- 自动提取有~5% 噪声（描述片段被误当县名），但只要该噪声不在 entity_index 的 place 集合里就不会影响分类结果

## 五·二、L5：并列地名传播（**已实现** @ 2026-04-18）

**规则**：在 `chapter_md` 正文中，两个 `〖=X〗` 标记之间若只有"、"或"，"（可含空白），则视为并列。对多个并列 place 构成的 peer 组，从已分类条目向未分类条目传播（多数决 ≥ 60%）。

**典型触发上下文**：
- `⟦◈攻⟧〖=铚〗、〖=酂〗、〖=苦〗、〖=柘〗` — 被攻取的城邑/县序列
- `封...于〖=X〗、〖=Y〗、〖=Z〗` — 封邑列表
- `〖=X〗、〖=Y〗、〖=Z〗之民` — 州郡列表

**迭代**：最多 3 轮传播，直到无更新。

**约束**：仅对当前为空字符串的条目生效；已分类条目不被改写。

**效果**（2026-04-18 首次启用）：发现 588 组并列序列，升级 151 个 place。

## 五·三、三家注注释（**已实现** @ 2026-04-18）

扫描 `corpus/shiji/史记集解三家注索隐正义.txt`，匹配 `X，Y名` / `X，Y名也` 格式的条目定义，累积到 L3 hints。

已支持的模式（`classify_places.py::SANJIA_PATTERNS`）：

| 注释模式 | 推断类别 |
|----------|----------|
| `X，山名` | 山脉 |
| `X，水名` / `X，湖名` / `X，泽名` / `X，海名` | 水域 |
| `X，县名` | 县 |
| `X，郡名` | 郡 |
| `X，乡名` / `X，亭名` | 乡里 |
| `X，邑名` | 城邑 |
| `X，关名` / `X，塞名` | 关隘 |
| `X，国名` | 国家 |

**典型命中**：
- "涿鹿，山名" / "空桐，山名" → 山脉
- "西陵，国名也" → 国家
- "洞庭，湖名" / "彭蠡，湖名" → 水域

**局限**：当某地同时有多重注释（如 "涿鹿，山名" 与 "涿鹿县" 并存），优先级规则可能取其一。如需更细的时代语境（史记/汉书），需进一步的规则。

## 七、已登记的疑难与待办

### 7.1 陵 / 邑 / 城的歧义

- `X陵`（长度≥2）当前默认为 **城邑**（因汉代多数为陵邑）
- 但部分为皇陵：**霸陵**、**茂陵**、**长陵**、**阳陵**、**杜陵**、**平陵** 等
  - 已在 `EXPLICIT_TOMB` 中显式归陵墓
- 若发现漏归的皇陵，追加白名单即可

### 7.2 X门 的歧义

- 城门（建筑）："东都门"、"北门"、"司马门"
- 关口（关隘）："龙门"、"雁门"（→ 郡）、"孟门"
- 当前全部留空。可在 L3 引入："大门之守"、"守〖=X门〗" 之类上下文消歧

### 7.3 X国（作为 place 出现）的处理

- 如果 `〖=X国〗` 出现，当前留空
- 规则草案：
  - `封...于〖=X国〗` → 县（汉代侯国）
  - `至〖=X国〗` / `入〖=X国〗` → 区域（泛指该国之地域）

### 7.4 X林 / X谷 / X沙

- 林：`桂林` = 郡、`桃林` = 地区、`上林` = 苑（建筑）
- 谷：`函谷` = 关隘、`鬼谷` = 山谷
- 沙：`博浪沙` = 具体点、`流沙` = 区域
- 全部留空；L2/L3 规则成熟后再处理
