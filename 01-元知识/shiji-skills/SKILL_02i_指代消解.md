# SKILL 02i: 指代消解 — 人称代词与身份指代的语义消解

> 识别文本中的指代词（人称代词、身份称谓），并在可推断的情况下标注其所指向的具体人物实体。将隐性的人物指代关系显性化，提升知识图谱的语义完整性。

**当前状态**: 📋 规划阶段，计划在第四轮实体反思时实施

---

## 一、任务定义

### 1.1 核心目标

在已标注文本中识别**可推断的人物指代**，并标注其所指向的具体人物实体：

```
原文：〖#美人〗和之
标注：〖#美人→虞姬〗和之

原文：〖;上〗大怒
标注：〖;上→汉高祖〗大怒

原文：吾起兵至今八岁矣
标注：〖→项羽〗吾起兵至今八岁矣
```

### 1.2 范围界定

**DO（做消解）**:
- ✅ 可推断到具体人名的指代
- ✅ 上下文明确的人称代词
- ✅ 章节主角的身份简称
- ✅ 对话中的吾/余/汝/子

**DON'T（不做消解）**:
- ❌ 通用群体身份词（百姓、士卒、诸侯）→ 交给 03a 实体标注
- ❌ 虚指/泛指（人、民、天下）→ 交给 02e 词性标注
- ❌ 歧义候选超过2个且置信度接近 → 留待人工或后续推理

### 1.3 典型案例

**案例1：身份指代（007_项羽本纪.tagged.md:1084）**
```
原文：〖#美人〗和之
上下文：项王夜起，饮帐中。有〖#美人〗名〖@虞〗…
消解：〖#美人→虞姬〗和之
理由：上文明确提到"有美人名虞"，章内唯一美人
```

**案例2：尊称简称（008_高祖本纪）**
```
原文：〖;上〗欲都洛阳
消解：〖;上→汉高祖〗欲都洛阳
理由：高祖本纪章内"上"默认指汉高祖
```

**案例3：对话人称（007_项羽本纪.tagged.md:1105）**
```
原文：谓其骑曰："吾起兵至今八岁矣"
消解：〖@项王〗谓其骑曰："〖吾→项羽〗起兵至今八岁矣"
理由：说话主语是项王，对话内"吾"指项王
```

## 二、指代类型分类

### 2.1 人称代词类（优先级：高）

| 代词 | 人称 | 消解难度 | 典型场景 |
|------|------|----------|----------|
| **吾、余、我** | 第一人称 | 低 | 对话中指说话人 |
| **朕、寡人、孤** | 第一人称（君主专用） | 低 | 帝王自称 |
| **汝、女、尔** | 第二人称 | 低 | 对话中指听话人 |
| **子、君、公** | 第二人称（敬称） | 中 | 需结合尊卑关系 |
| **其、之** | 第三人称 | 高 | 需追溯前文主语 |
| **彼** | 第三人称（远指） | 高 | 需语境推断 |

**消解规则**：
```
对话内吾/余 → 说话人
对话内汝/子 → 听话人
朕/寡人/孤 → 章节主角（如为帝王）
其/之 → 前句主语（窗口5句内）
```

### 2.2 身份指代类（优先级：高）

| 指代词 | 类型 | 消解方式 | 示例 |
|--------|------|----------|------|
| **美人、夫人、宠姬** | 后妃 | 上下文+人物关系 | 美人名虞 → 虞姬 |
| **上、帝、王** | 君主简称 | 章节默认主角 | 高祖本纪"上" → 汉高祖 |
| **太后、后、公主** | 女性尊称 | 当前帝王+关系推理 | 汉文帝时"太后" → 薄太后 |
| **将、相、令、守** | 职官简称 | 上下文主语 | 令尹 → 楚国令尹（需人名） |

**消解规则**：
```
章节级默认规则：
  - 007_项羽本纪.tagged.md 中"王" → 项羽
  - 008_高祖本纪.tagged.md 中"上" → 汉高祖

关系推理规则：
  - "太后" → 当前帝王的母亲（查询人物关系图谱）
  - "夫人" → 当前人物的妻子
```

### 2.3 亲属称谓类（优先级：中）

| 称谓 | 消解依据 | 难度 |
|------|----------|------|
| **父、母** | 当前主语的父母关系 | 中 |
| **兄、弟、子、女** | 人物关系图谱 | 中 |
| **伯、叔、舅** | 扩展亲属关系 | 高 |

**消解前置条件**：
- 需要完成 SKILL_05c 人物关系构建
- 可查询亲属关系三元组

### 2.4 职官简称类（优先级：低）

| 简称 | 全称示例 | 消解策略 |
|------|----------|----------|
| 将 | 将军、大将 | 前文最近职官实体 |
| 相 | 丞相、相国 | 章节默认丞相 |
| 令 | 县令、郡令 | 地点+职官组合 |
| 守 | 太守、郡守 | 同上 |

**当前阶段**: 暂不处理，留待第四轮或更后版本

## 三、标注格式规范

### 3.1 新增标记符号

**符号**: `→` （U+2192 RIGHTWARDS ARROW）

**语法**: `〖TYPE 显示名→消解目标〗`

**三种使用模式**:

#### 模式A：身份指代消解
```
格式：〖#身份词→人名〗
示例：〖#美人→虞姬〗和之
说明：保留原文"美人"，标注消解为"虞姬"
```

#### 模式B：职官/尊称消解
```
格式：〖;职官→人名〗
示例：〖;上→汉高祖〗大怒
说明：保留原文"上"，标注消解为"汉高祖"
```

#### 模式C：人称代词消解
```
格式：〖→人名〗代词
示例：〖→项羽〗吾起兵至今八岁矣
说明：在代词前添加消解标注
```

### 3.2 消解目标规范

**规则1：消解目标必须是已标注的人名实体**
```
✅ 正确：〖#美人→虞姬〗（虞姬已在前文标注为〖@虞〗）
❌ 错误：〖#美人→虞美人〗（虞美人非史记标准人名）
```

**规则2：使用规范化人名**
```
✅ 正确：〖;上→汉高祖〗（使用庙号）
✅ 正确：〖;上→刘邦〗（使用本名）
⚠️ 注意：应与人物实体索引保持一致
```

**规则3：消歧优先于消解**
```
如果人物本身存在多名，优先标注消歧：
〖#美人→虞|虞姬〗 （同时做实体消歧和指代消解）
```

### 3.3 与现有标注体系的兼容性

**已有标注保持不变**：
```
原：〖#美人〗和之
改：〖#美人→虞姬〗和之
说明：〖#〗符号保留，表示这是群体身份词
```

**嵌套标注示例**：
```
〖@项王〗谓其骑曰："〖→项羽〗吾起兵至今〖%八岁|时长〗矣"
说明：同时包含人名标注、指代消解、时间消歧
```

## 四、消解规则体系

### 4.1 第1层：上下文窗口规则（Rule-based）

**规则1.1：对话说话人规则**
```python
IF 句式匹配 "〖@人名〗曰：「...吾/余...」"
THEN 吾/余 → 人名
置信度：A级（确定）

示例：
〖@项王〗谓其骑曰："吾起兵至今八岁矣"
→ 〖→项羽〗吾 （置信度：A）
```

**规则1.2：对话听话人规则**
```python
IF 句式匹配 "〖@人A〗谓〖@人B〗曰：「...汝/子...」"
THEN 汝/子 → 人B
置信度：A级（确定）

示例：
〖@项王〗谓〖;亭长〗曰："吾知公〖;长者〗"
→ 汝 指代亭长
```

**规则1.3：前句主语规则**
```python
IF 前5句内出现〖@人名〗作主语
AND 当前句"其/之"无其他候选
THEN 其/之 → 前句主语人名
置信度：B级（可能）
```

### 4.2 第2层：章节全局规则（Pattern-based）

**规则2.1：章节主角默认规则**
```python
章节默认主角表（本纪/世家）：
  001_五帝本纪 → 黄帝/尧/舜（分段）
  007_项羽本纪 → 项羽
  008_高祖本纪 → 汉高祖
  ...

IF 出现〖;上〗或〖;王〗
AND 当前章节 in 本纪/世家
THEN 上/王 → 章节主角
置信度：A级（确定）
```

**规则2.2：身份词章内唯一规则**
```python
IF 章内统计〖#美人〗仅指向1个人名
AND 上下文明确提到"美人名〖@X〗"
THEN 所有〖#美人〗 → X
置信度：A级（确定）
```

### 4.3 第3层：关系推理规则（Graph-based）

**规则3.1：亲属关系推理**
```python
前提：已构建人物关系图谱（SKILL_05c）

IF 句式匹配 "〖@人A〗之父/母/子"
AND 人物关系图谱存在 (A, has_father/mother/son, B)
THEN 父/母/子 → B
置信度：A级（确定）

示例：
"〖@项籍〗之季父〖@项梁〗" → 季父 指代 项梁
```

**规则3.2：君臣关系推理**
```python
IF 当前段落主语为〖@帝王X〗
AND 出现"太后"
AND 关系图谱存在 (X, has_mother, Y)
THEN 太后 → Y
置信度：A级（确定）
```

### 4.4 第4层：LLM辅助推断（AI-assisted）

**使用场景**：
- 前3层规则无法覆盖
- 需要深度语义理解
- 候选多个但有明显最优解

**Prompt模板**：
```
任务：推断以下句子中指代词的所指对象

原文：[段落文本]
指代词：[吾/其/美人/...]
候选对象：[人A, 人B, 人C]（从前文提取）

请判断指代词最可能指向哪个人物，输出JSON：
{
  "target": "人名",
  "confidence": 0.0-1.0,
  "reason": "推理依据"
}

要求：
1. confidence < 0.9 时输出 null（不标注）
2. reason必须引用原文具体线索
```

**置信度阈值**：
```
>= 0.95: A级（确定），自动标注
0.90-0.94: B级（可能），人工复核后标注
< 0.90: C级（存疑），不标注
```

## 五、置信度分级体系

### 5.1 A级：确定（Auto）

**特征**：
- ✅ 规则明确，无歧义
- ✅ 唯一候选或候选间概率差距>0.8
- ✅ 可自动标注

**示例**：
```
1. 对话说话人：
   〖@项王〗曰："吾起兵至今八岁矣"
   → 吾 = 项王（置信度：1.0）

2. 章节主角简称：
   008_高祖本纪 中 "上大怒"
   → 上 = 汉高祖（置信度：1.0）

3. 上下文明确：
   "有〖#美人〗名〖@虞〗，常幸从…〖#美人〗和之"
   → 美人 = 虞姬（置信度：1.0）
```

### 5.2 B级：可能（Review）

**特征**：
- ⚠️ 有2-3个候选，但最可能的明显
- ⚠️ 需要一定推理但逻辑链完整
- ⚠️ 需人工复核后标注

**示例**：
```
1. 前句主语推断：
   "〖@项梁〗常为主办…〖@项梁〗以此知其能。其少时…"
   → 其 可能= 项籍（前文提到项籍，但中间隔了项梁）

2. 多人对话场景：
   〖@张良〗曰："…"〖;上〗曰："善。"乃从其计。
   → 其 可能= 张良（最近提到的谋士）
```

### 5.3 C级：存疑（Skip）

**特征**：
- ❌ 候选超过3个且概率接近
- ❌ 缺乏足够上下文
- ❌ 跨段落/跨章节指代
- ❌ **不标注**，留待后续人工或更强推理

**示例**：
```
1. 远距离指代：
   "其父尝为楚将…（中间50句）…乃从其计"
   → 其 = ? （无法确定是其父还是其他人）

2. 多候选等概率：
   场景中出现3个将军，"将大怒" 无法确定指谁
```

### 7.1 Lint检查规则

**规则1：消解目标实体存在性检查**
```python
def lint_target_exists(tagged_file):
    """检查消解目标是否为已标注的人名实体"""
    errors = []
    for line in tagged_file:
        matches = re.findall(r'〖[#;]([^→]+)→([^〗]+)〗', line)
        for (display, target) in matches:
            if target not in entity_index:
                errors.append(f"消解目标不存在：{target}")
    return errors
```

**规则2：同段一致性检查**
```python
def lint_consistency(paragraph):
    """检查同一段落内同一指代词的消解一致性"""
    coreferences = extract_coreferences(paragraph)
    for word in set(c.word for c in coreferences):
        targets = [c.target for c in coreferences if c.word == word]
        if len(set(targets)) > 1:
            warn(f"同段不一致：{word} → {targets}")
```

**规则3：循环指代检测**

```python
def lint_circular(chapter):
```

```
[173.1] 谓其骑曰："吾起兵至今〖%八岁〗矣，身〖$七十馀〗战，
        所当者破，所击者服，未尝败北，遂霸有〖_天下〗。"
```

**消解分析**：
```
候选词：吾
上下文：谓其骑曰："..."（说话主语需前溯）
前句：〖@项王〗... 谓其骑曰
消解规则：对话说话人规则
置信度：A级（1.0）

标注结果：
〖@项王〗谓其骑曰："〖→项羽〗吾起兵至今〖%八岁|时长〗矣..."
```

### 案例4：复杂场景-存疑不标注

**原文**（假设）：
```
〖@张良〗、〖@陈平〗、〖@韩信〗皆在侧。〖;上〗问策，其对曰："..."
```

**消解分析**：
```
候选词：其
候选对象：张良、陈平、韩信（3个等概率）
上下文：无明确线索指示谁回答
消解规则：规则1-3均无法确定
LLM推断：confidence = 0.65（低于0.90）
置信度：C级（存疑）

标注结果：不标注（保持原文）
```

## 附录：常见问题

**Q1: 为什么不在第三轮反思就做指代消解？**
A: 第三轮重点是实体标注质量提升（时间消歧、群体身份），指代消解依赖这些基础标注完善后才能准确进行。

**Q2: 指代消解会改变原文吗？**
A: 不会。严格遵守标注铁律，只添加`→`符号和消解目标，原文字符保持不变。

**Q3: "其"字消解为什么这么难？**
A: "其"在古汉语中既可指人，也可指事/物，且经常省略先行词，需要深度语义理解。当前策略是只消解高置信度场景。

**Q4: 如何处理消解歧义？**
A: 置信度<0.90的场景不标注，留待人工或未来更强的推理能力处理。宁可漏标，不可错标。

**Q5: 消解结果如何在阅读器中展示？**
A: 鼠标悬停指代词时，悬浮框显示"→ 目标人物"，点击可跳转到该人物首次出现处或人物卡片。
