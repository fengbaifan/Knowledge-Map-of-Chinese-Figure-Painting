## A. 类型判断规则

### A.0 红线（最高优先级）

本子区为绝对不可违反的铁律，任何反思建议都必须先通过 A.0 的检查。

**本子区条目**（1条）：A70

#### A70. 严禁修改动词标注格式（2026-03-24新增，最高优先级）

> **红线提示**（2026-04-16 标记）：A70 为"最高优先级格式红线"，建议读者把此条视为 §0 前置规则优先阅读；保留 A 区编号以免破坏外部引用。


**问题描述**：
在实体反思任务中，**严格禁止**修改动词标注的格式。动词和实体使用不同的标注符号，两者不可互相转换。

**错误模式**（严重违规）：

| 错误转换 | 说明 | 严重程度 |
|---------|------|---------|
| `⟦◈伐⟧` → `〖[伐〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦◉杀⟧` → `〖[杀〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦◉诛⟧` → `〖[诛〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦○废⟧` → `〖[废〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `〖[攻〗` → `⟦◈攻⟧` | 实体格式→动词格式 | ⚠️ 仅使用专用工具 |

**正确做法**：

✅ **保持格式不变**：
- 看到`⟦◈⟧`/`⟦◉⟧`/`⟦○⟧`/`⟦◇⟧`格式 → 保持不变
- 看到`〖[〗`格式 → 保持不变（不在反思任务中迁移）

✅ **第三轮反思任务聚焦内容，不涉及格式**：
- 检查实体标注遗漏
- 检查时长消歧
- 检查人名省称规范
- 检查原文完整性
- **不修改动词标注格式**

**规范依据**：

根据`doc/spec/SPEC_动词标注.md`（v3.3版）：
- **动词标注**：必须使用`⟦TYPE动词⟧`（数学双方括号 U+27E6/U+27E7）
- **实体标注**：必须使用`〖TYPE内容〗`（白色龟甲括号 U+3016/3017）
- **动词TYPE符号**：
  - `◈` (U+25C8) = 军事动词（伐、攻、击、围等）
  - `◉` (U+25C9) = 刑罚动词（杀、诛、弑、族等）
  - `○` (U+25CB) = 政治动词（废、立、封等）
  - `◇` (U+25C7) = 经济动词（贸、贡、赋等）

**历史遗留说明**：

项目中存在两种刑法标注格式：
- **旧格式**（v2.8及之前）：`〖[verb〗`（使用实体标注符号）
- **新格式**（v3.0及之后）：`⟦◈verb⟧`/`⟦◉verb⟧`（使用动词标注符号）

**正确的格式迁移方向**：`〖[verb〗` → `⟦◈verb⟧`/`⟦◉verb⟧`

**⚠️ 常见错误理解（2026-03-30新增）**：

❌ **错误理解**：将 `⟦◈⟧`/`⟦◉⟧` 误认为"旧格式"
- 错误表述："章节使用⟦◈⟧/⟦◉⟧旧格式"
- 错误来源：混淆了格式演进的方向

✅ **正确理解**：
- `⟦◈⟧`/`⟦◉⟧`/`⟦○⟧`/`⟦◇⟧` = **动词的正确格式**（新规范，v3.0+）
- `〖[〗` = **用实体符号标注动词的历史遗留格式**（旧格式，v2.8-）
- 反思时看到 `⟦◈⟧`/`⟦◉⟧` → **这是正确的**，保持不变
- 反思时看到 `〖[〗` → **这是待迁移的旧格式**，保持不变（等待专用工具）

**记忆要诀**：
- 数学双方括号 `⟦⟧` (U+27E6/U+27E7) = 动词专用符号（新/正确）
- 龟甲括号 `〖〗` (U+3016/3017) = 实体专用符号
- 动词和实体各有专用符号，不可混用

**错误案例分析**（2026-03-24发现）：

在021-030章和031-040章第三轮反思中，Agent错误地进行了**反向转换**：
- 025_律书：7处 `⟦◈◉⟧` → `〖[〗` ❌
- 028_封禅书：10处 `⟦◈◉⟧` → `〖[〗` ❌
- 029_河渠书：1处 `⟦◈◉⟧` → `〖[〗` ❌
- 031-045章：多处错误（已恢复）❌

**影响**：
- 破坏动词标注体系的一致性
- 导致动词-实体混淆
- 影响后续知识图谱构建
- 需要使用git恢复所有错误修改

**格式迁移的正确方式**：

❌ **禁止**：在反思任务中手动转换格式
✅ **正确**：使用专门的迁移工具：
```bash
python kg/entities/scripts/migrate_verb_tags.py --all --backup
```

**违反后果**：

1. 立即停止任务
2. 使用`git restore`恢复所有被修改文件
3. 删除错误的反思报告
4. 重新执行反思任务（正确理解）

**适用范围**：所有反思任务（第一轮、第二轮、第三轮）
**严重程度**：❌❌❌ 严重（违反基础规范）

### A.1 人物与身份类（@/#/;）

涵盖人名、身份、官职、群体称谓、亲属/敬谦称、帝号/庙号等与人物实体相关的判断规则。

**本子区条目**（32条）：A1, A2, A3, A14, A15, A16, A21, A48, A54, A55, A56, A59, A71, A72, A73, A74, A75, A76, A77, A85, A86, A88, A89, A90, A92, A93, A94, A95, A96, A97, A99, A101

#### A1. 帝号/谥号/庙号→人名（009-010章归纳）

帝号/谥号/庙号指代具体某人→`〖@人名〗`，不是身份`〖#〗`或官职`〖;〗`。

**皇室称谓归类总表**（本主题相关：A1, A3, A77, A90, A99；2026-04-16 汇总）：

| 类型 | 标注 | 示例 |
|------|------|------|
| 具体帝号/谥号/庙号 | `〖@〗` | 高帝、高后、孝武皇帝、幽王、厉王 |
| 通称（活帝王） | `〖#〗` | 天子、皇帝、太后、陛下、上、**今上/圣上**（见 A99） |
| 通称（已故帝王） | `〖#〗` | **先帝/先王**（见 A3） |
| 帝王自称 | `〖#〗` | **朕、寡人、孤、予一人、不穀**（见 A90） |
| 敬称/谦称 | `〖#〗` | **夫子、先生、阁下、足下、小子、愚人**（见 A77） |
| 泛称 | `〖#〗` | 今王、人主、单于 |

**判断标准**：能替换为具体人名→`〖@〗`；泛指一类身份→`〖#〗`。

#### A2. 特指后妃→人名（012章首发现）

特指某具体后妃的称号→`〖@〗`，泛称→`〖#〗`。
- `康后`（胶东康王后）→`〖@〗`；`太后`/`皇后`（泛称）→`〖#〗`

#### A3. 先帝/先王→身份（006章首发现）

"先帝""先王"是追述已故帝王的称号→`〖#身份〗`，不是官职`〖;〗`。

#### A14. 庙号消歧（013章首发现）

殷商庙号加消歧前缀：中宗→`〖#中宗|帝太戊〗`。

#### A15. 撰述者→人名（013章首发现）

太史公/褚先生在"X曰"语境→`〖@人名〗`。

#### A16. 稷字语境判断（001第二轮）

#### A54. 年表侯号+人名组合检查（021章第二轮）

年表类章节中，`〖;侯号〗刘X〖%元年〗` 格式的人名容易遗漏，需系统性扫描。

**检查命令**：
```bash
grep -n '〖;.*侯〗刘[^〗]' chapter_md/NNN_*.tagged.md
```

**修正模式**：
```
〖;X侯〗刘Y  →  〖;X侯〗〖@刘Y〗
```

**021章案例**：
- L147-151：长沙定王子分封集中区域，4处人名遗漏
- `〖;敬侯〗刘义` → `〖;敬侯〗〖@刘义〗`
- `〖;靖侯〗刘狗彘` → `〖;靖侯〗〖@刘狗彘〗`

**易错点**：
1. 首次出现时标注完整
2. 集中分封区域（同一父王的多个儿子）后续行遗漏
3. 复合名（如"狗彘"）更易遗漏

**适用章节**：年表类（014-021）

#### A55. 年表"侯X"结构词性判断（021章第二轮）

`侯X不...` 结构中X的词性判断：人名 vs 副词 vs 动词。

**判断流程**：
1. 查找前文是否有"X元年" → 有：人名 / 无：继续
2. 检查X是否为常见副词 → 是：副词，不标注
3. 检查X是否为动词 → 是：不标注
4. 无法确定 → 保守不标注

**常见副词清单**：
| 副词 | 意义 | 示例 |
|------|------|------|
| 阴 | 暗中、私下 | 侯阴不使人为秋请 |
| 阳 | 公开地 | 侯阳拒之 |
| 明 | 明显地 | 侯明示其意 |
| 暗 | 暗中 | 侯暗通款曲 |

**021章案例**：
- L126: `侯〖@阴〗不使人为秋请` → `侯阴不使人为秋请`
- 分析：
  1. 前文无"阴元年"记录
  2. "阴"为常见副词（暗中）
  3. 语法："侯+副词+否定+动词"符合汉代语法
- 结论："阴"是副词，不标注

**误判原因**：
- 将"阴"理解为继承人名字
- 忽略时间逻辑：刘摇元年后仅一年即国除，无时间给继承人

**类似易混淆词**：
- 阳（副词/人名/地名）
- 明（副词/人名/形容词）

#### A56. 人名边界切分错误检查（021章第二轮）

`〖;侯号〗刘〖@X〗` 格式中，"刘"被拆分到标注外的错误。

**错误模式**：
```
〖;康侯〗刘〖@不害〗  →  错误
〖;康侯〗〖@刘不害〗  →  正确
```

**检查命令**：
```bash
grep -n '〖;.*侯〗刘〖@' chapter_md/NNN_*.tagged.md
```

**021章案例**：
- L75: `〖;康侯〗刘〖@不害〗` → `〖;康侯〗〖@刘不害〗`

**规律**：汉代刘姓人名，"刘"必须包含在人名标注内。

**适用章节**：所有含刘姓人名的章节（尤其年表类）

#### A59. 年表侯号官职用法判断（022章新增）

年表职官列中，侯号作为官员身份使用时标注为〖;官职〗。

**判断标准**：

| 句式 | 判断 | 标注 | 示例 |
|------|------|------|------|
| 侯号+人名+为+官职 | 侯号作官衔 | 〖;侯号〗 | 〖;绛侯〗〖@周勃〗为〖;太尉〗 |
| 封+人名+侯号 | 侯号作封号 | 〖@侯号〗 | 封〖@平津侯〗 |
| 侯号+人名（主语） | 整体是人名 | 〖@侯号人名〗 | 〖@魏其侯窦婴〗为丞相 |

**关键区分**：
- 年表职官列 → 侯号是身份标识 → 〖;〗
- 封赏/初封 → 侯号是封号 → 〖@〗

#### A94. 「王」作政治动词标 ⟦○王⟧（2026-04-16新增，009章）

#### A103. 并列王号共享后缀"王"字的标注法（2026-04-16新增，011章）

**场景**：原文使用"A、B王"结构（A 省略"王"字，与 B 共享末位"王"字）——史记常见修辞省略。

**规则**：
- A 在上下文中有独立王号语义时，应标 `〖;A|A王〗`（显示保持原文省略，规范名补全"王"字）
- **不得降级为地名** `〖=A〗`（会丢失身份/官职语义）
- B 作为末位保留完整王号 `〖;B王〗`

**典型案例**（011章 L35）：
- 原：`〖=广川〗、〖;长沙王〗皆之国`
- 误：把"广川"降级为地名（丢失王号身份）
- 正：`〖;广川|广川王〗、〖;长沙王〗皆之国`

**识别要点**：
- 两个并列项同属受封王的身份（"皆之国"/"皆王某地"等谓语）
- 前项是封国/地名而后项带"王"字
- 避免误判为"A 是地名 + B 是某王"的组合

#### A105. 汉年表「谥号王 + 名」拆分标注（2026-04-16新增，017章；原 017 报告称"A91 延伸"）

**场景**：汉诸侯王年表中常见 `X〖@王Y〗` 形态（X=谥号字、Y=名），爵位"王"被错误吞入人名标注。应拆分为两种正确形态之一：

**规则**：
- 形态 A：谥号王作整体人名 → `〖@X王〗〖@Y|刘Y〗`
  - 示例：`惠〖@王武〗` → `〖@惠王〗〖@武|刘武〗`（对于世系表"谥号王 + 本名"格式）
- 形态 B：谥号作前缀 + 名消歧 → `X王〖@Y|刘Y〗`
  - 示例：`靖王建` → `靖王〖@建|刘建〗`（当谥号不作独立人名时）

**识别特征**：
- 同章存在 `初王〖@Y〗〖%元年〗` 作为模板对照
- 同名前后出现 `Y，高祖/文帝/景帝子` 的旁证句
- Y 前紧邻的单字是常见谥号（惠/文/景/靖/哀/昭/康/顷等）

**典型案例**（017章 r25/r31/r78/r80/r99）：
- `王武/王建/靖王建/慧王武` 等黏连形态

**与相关规律区分**：
- A62：汉代「氏族（封国）+谥号+王」整体（中山靖王/河闲献王）
- A82a：年表"王+本名"成词（`王〖@司马卬〗`）
- A104：战国年表「邦国+谥号+王」整体（`〖@韩宣惠王〗`）
- A105（本条）：汉年表「谥号王+名」拆分（`〖@惠王〗〖@武|刘武〗`）

#### A106. 副词"卒/竟/遂"与刑法动词的辨析（2026-04-16新增，011章）

**场景**：古文"卒"字兼有"终于"（副词）、"士兵/卒"（名词）、"死亡/终"（动词）三义。本纪/议论段末常见"卒以安/卒以定/卒亡其国"等结构，"卒"为时间副词，**不标注**。同类辨析词：竟/遂/终/卒。

**判断清单**：
- 后接介词"以/于"且其后为状态词（安/定/亡/至） → 副词"终于"，**不标注**
- 作谓语且主语为个人（"某某卒"） → 死亡义动词
- 作名词指士兵 → `〖#卒〗` 或 `〖#士卒〗`

**典型修正**（011章 L342）：
- 误：`〖[卒〗以安`（把副词错作刑法动词）
- 正：`卒以安`（去标）

### A.2 时间类（%）

涵盖纪年、时长、干支、日月多义等时间实体的标注规则。

**本子区条目**（3条）：A8, A57, A91

#### A8. 时长vs纪年（008章归纳，001章第三轮细化）

**消歧标注规范**：

时间类型〖%〗包含两大子类，必须通过消歧标注区分：

| 子类型 | 标注格式 | 说明 |
|--------|---------|------|
| **时长/持续时间** | `〖%XX年\|时长〗` | 必须加 `\|时长` 消歧 |
| **纪年格式（年月日）** | `〖%XX年〗` | 无需消歧标注 |

**默认规则**：没有消歧标注时，默认为**纪年格式**（年月日）

**时长类型（6类，均需标注 `|时长`）**：

1. **任期时长**：在位XX年、立XX年、践帝位XX年、摄政XX年
   - `在位〖%七十载|时长〗`、`〖@尧〗立〖%七十年|时长〗得〖@舜〗`
2. **经过时长**：XX年而老/崩、XX年得、功用不成
   - `〖%二十年|时长〗而老`、`〖@尧〗於是听岳用〖@鲧〗。〖%九岁|时长〗，功用不成`
3. **守丧/哀悼时长**：三年丧毕、三年四方莫举乐、哭X日、服X年、居丧X年、哀X日（2026-04-16补充哀悼触发词，031章）
   - `〖%三年|时长〗丧毕`、`〖%三年|时长〗，四方莫举乐`
   - `哭於军门外〖%三日|时长〗`（夫差为齐悼公哀悼）
   - 触发词：**哭 X 日/X 年、服 X 年、居丧 X 年、哀 X 日**——命中即需 `|时长` 消歧
4. **制度周期**：三岁一考功、五岁一巡狩
   - `〖%三岁|时长〗一考功`、`〖%五岁|时长〗一〖^巡狩〗`
5. **发展阶段**：一年成聚、二年成邑、三年成都
   - `〖%一年|时长〗而所居成聚，〖%二年|时长〗成邑，〖%三年|时长〗〖=成都〗`
6. **已经时长**：三年矣、数月矣
   - `女谋事至而言可绩，〖%三年|时长〗矣`
7. **年龄**（A8-7，2026-04-16新增；004章 N2、001章实操）：年X岁、春秋已五十、年二十以孝闻
   - 标注格式：`〖%XX|年龄〗`（不带"年/岁"字时也保留数字主体）
   - 示例：`年〖%二十|年龄〗以孝闻`、`〖%春秋〗已〖%五十|年龄〗矣`、`年〖%六十一|年龄〗代〖@尧〗践帝位`
   - 触发词：「年X」「春秋已X」「X岁以…」（紧跟动词或描述）
   - 与时长区分：年龄是某时点的累计岁数，不可替换为"过了X年"
8. **年号**（A8-8，2026-04-16新增；020章）：武帝朝起的帝王纪元号，如建元/元光/元朔/元狩/元鼎/元封/太初/天汉/太始/征和/后元；昭帝朝始元/元凤/元平；宣帝朝本始/地节/元康/神爵/五凤/甘露/黄龙；元帝朝初元/永光等
   - 标注格式：`〖%XX|年号〗`
   - 示例：`〖%元凤|年号〗`、`〖%太初|年号〗四年`、`至〖%征和|年号〗中`
   - 与纪年区分：年号是朝代+帝王+时期的复合标识（可跨具体年数），后续的「元年/二年」紧随时是纪年；如「太初元年」→ `〖%太初|年号〗〖%元年〗`
   - 扫描命令：`grep -nE '〖%(建元|元光|元朔|元狩|元鼎|元封|太初|天汉|太始|征和|后元|始元|元凤|元平|本始|地节|元康|神爵|五凤|甘露|黄龙|初元|永光|建昭|竟宁)〗' chapter_md/`
   - 通常应加 `|年号` 显式消歧

**纪年格式类型（3类，无需消歧）**：

1. **编年纪年**：元年、二年、三年（单独使用表某年）
   - `〖%元年〗`、`〖%三年〗`
2. **月份日期**：正月、八月、正月上日、八月丙子
   - `〖%正月上日〗，〖@舜〗受终於〖=文祖〗`
   - `〖%岁二月〗，东〖^巡狩〗；〖%五月〗，南〖^巡狩〗`
3. **完整纪年**：XX年XX月XX日
   - `〖%八月丙子〗`

**判断关键三问**：

1. **能否替换为"第X年"且不改变语义？** → 是：时长（需`|时长`）
   - "立七十年得舜" ≈ "立了七十年后得到舜" → 时长
   - "元年正月" ≠ "第元年第正月" → 纪年
2. **有语法助词"而/矣/毕"修饰？** → 是：通常是时长
   - "二十年而老"、"三年矣"、"三年丧毕"
3. **是历法/编年体的时间坐标点？** → 是：纪年格式（无需消歧）
   - "元年"、"正月上日"是时间坐标 → 纪年

**典型案例对比**：

| 原文 | 错误标注 | 正确标注 | 类型 |
|------|---------|---------|------|
| 朕在位七十载 | `朕在位〖%七十载〗` | `朕在位〖%七十载\|时长〗` | 任期时长 |
| 二十年而老 | `〖%二十年〗而老` | `〖%二十年\|时长〗而老` | 经过时长 |
| 三年丧毕 | `〖%三年〗丧毕` | `〖%三年\|时长〗丧毕` | 守丧时长 |
| 五岁一巡狩 | `〖%五岁〗一〖^巡狩〗` | `〖%五岁\|时长〗一〖^巡狩〗` | 制度周期 |
| 一年成聚 | `〖%一年〗成聚` | `〖%一年\|时长〗成聚` | 发展阶段 |
| 三年矣 | `〖%三年〗矣` | `〖%三年\|时长〗矣` | 已经时长 |
| 元年正月 | `元年〖%正月〗` | `〖%元年〗〖%正月〗` | 编年纪年 ✓ |
| 正月上日 | `〖%正月上日〗` | `〖%正月上日〗` | 月份日期 ✓ |

**注意事项**：

- 旧规律"不标〖%〗"已废弃，时长必须标注但需加`|时长`消歧
- 001章第三轮发现19处时长遗漏消歧标注
- 此规范自第三轮反思起执行

#### A117. 文景武帝「前元/中元/后元」年号前缀合并规则（2026-04-17新增，006/009/010/026章）

**背景**：汉文帝、景帝、武帝在位时分段改元，常用"前元、中元、後元"作为年号段落前缀：

| 帝王 | 年号段 |
|---|---|
| 文帝 | 前元 16 年 + **后元 7 年** |
| 景帝 | **前元 7 年 + 中元 6 年 + 后元 3 年** |
| 武帝 | 建元/元光/.../征和/**后元** 共 11 个年号 |
| 高后/惠帝等 | 无年号制度 |

**合并规则**：`前/中/後/后〖%X年〗` 是否合并为 `〖%前X年/中X年/後X年〗`，按以下三层判断：

**第一层（100% 合并）**：`前/中/後/后元` 开头的明确年号段
```
❌ 後〖%元元年〗       ✅ 〖%後元元年〗
❌ 後〖%元三年〗       ✅ 〖%後元三年〗
❌ 前〖%元六年〗       ✅ 〖%前元六年〗
```

**第二层（按章合并）**：仅在以下章节合并 `前X年/後X年/中X年`：
- **010 孝文本纪** — 文帝前元/后元
- **011 孝景本纪** — 景帝前元/中元/后元
- **012 孝武本纪** — 武帝多段

#### A118. 闰月「後九月」标注规则（2026-04-17新增，009 吕太后本纪）

**背景**：汉初历法以十月为岁首，若遇闰月则置于九月之后，称"**後九月**"（即闰九月）。与"後元九月"（年号）无关。

**识别**：
- **吕后/汉初**语境下的"後九月/後X月"是闰月
- 汉武帝太初改历（公元前 104 年）后改为冬月，闰月分散

**标注规则**：闰月整体纳入时间实体，加纪年消歧：
```
✅ 〖%後九月晦日己酉|高后八年闰九月晦日〗
❌ 後〖%九月晦日己酉〗（拆散丢失闰月语义）
❌ 〖%後九月晦日己酉〗（无纪年消歧）
```

**典型出处**：
- 009 L621 "後九月晦日己酉" = 高后八年闰九月晦日（文帝入长安即位日）
- 011 孝景本纪 L105 "後九月" 是景帝后元九月（正常月，不是闰月）— 按 A117 合并为 `〖%後九月〗`

**判定流程**：
1. 查在位君主：景/武帝？→ 应用 A117 年号合并
2. 其他帝王 + 历法记载 → 查是否闰月（对照《史记·历书》或考定历表）
3. 闰月 → 作 A118 整词消歧（加"闰X月"注）

**情况A - 人名**：张释之（汉朝廷尉，见其他章节）

#### A45. "子将"等复合结构的语法识别（062章第二轮）

**错误**：`〖@子将〗隐矣`（误作人名）
**正确**：子将隐矣（"子"=您，"将"=将要）

**识别规则**：
- "子+动词"结构 → "子"是尊称代词，不是人名

**对比真实人名**：

#### A53. "出於"/"用事"动词短语误标（004章第二轮）

**"出於"误标问题**：
- ❌ "所弃妖子〖@出於〗路者"
- ✅ "所弃妖子出於路者"（动词短语"出于路"）

**"用事"误标问题**：
- ❌ "以X为卿，〖@用事〗"
- ✅ "以X为卿，用事"（执政/掌权之义）

**判断标准**：
- "出於X"：出自/来自X → 动词短语，不标注
- "用事"：执政掌权 → 动词短语，不标注

**004章案例**：

- L197/L242: "用事" → 去标注

#### A98. "得意"判断规则（006第二轮，原误编 A19，2026-04-16 重编为 A98）

"得意"需根据语境判断：

**人名语境**：`〖@张得意〗`（明确指某个叫"得意"的人）

**动词短语语境**：不标注（表示满意、称心、实现意图）

**2026-04-16 补充**（039 晋世家发现）：

#### A61. 嵌套刑法标注错误（2026-03-23新发现）

**问题描述**：刑法制度名词被错误地拆分成多个独立标注，导致嵌套。

**错误模式**：

| 错误标注 | 正确标注 | 说明 |
|---------|---------|------|
| `〖[当〖[斩〗〗` | `〖[当斩〗` | "当"="应当"+"斩"被错误分别标注 |
| `〖[当〖[死〗〗` | `〖[当死〗` | "当死"="应当处死"是完整法律用语 |
| `〖[腰〖[斩〗〗` | `〖[腰斩〗` | "腰斩"是特定刑罚名称，不可拆分 |
| `〖[〖[斩〗首〗` | `〖[斩首〗` | "斩首"是完整刑罚动词 |
| `〖[系狱〗〖[当〖[死〗〗` | `〖[系狱〗〖[当死〗` | 连续嵌套错误 |

**错误根源**：
1. 标注者将"当斩"、"当死"理解为两个独立动词："当"（应当）+"斩/死"（刑罚）
2. 实际上"当斩"、"当死"是完整的法律判决用语，应作为整体标注
3. "当"字本身不是刑法动词，只是助动词

**语义分析**：
- "当斩" = "应当被斩首"（法律判决语）
- "当死" = "应当处死"（法律判决语）
- "腰斩" = 特定刑罚名称（从腰部斩断）
- "斩首" = 特定刑罚动词（斩断头颅）

**检测命令**：
```bash
grep -n '〖\[.*〖\[' chapter_md/NNN_*.tagged.md
```


#### A87. 人名消歧标注规则（2026-04-15新增）

**核心规则**：

- **单字名**（无例外）→ **100% 必须消歧**：使用 `〖@显示名|规范名〗` 格式
- **别名 / 字号 / 省称 / 封号 / 谥号** → **必须消歧**
- **全名/规范名 / 仅一处出现 / 上下文无歧义** → 按上下文判断，**默认不消歧**

**⚠️ 取消的豁免**（2026-04-17）：
- ~~"首次全名紧随省称"豁免~~ 已取消：过去允许段落内首次出现全名后、紧随 2-3 行内的省称不消歧，**现在一律消歧**（见 A110 例外段更新）。代表案例：092 章"信"共 109 处，全部标 `〖@信|韩信〗`。

**必须消歧的场景**：

| 场景 | 错误 | 正确 |
|------|------|------|
| 单字名 | `〖@政〗` | `〖@政\|秦始皇〗` |
| 单字名（远古） | `〖@尧〗` | `〖@尧\|帝尧〗` |
| 字号 | `〖@子房〗` | `〖@子房\|张良〗` |
| 别名/旧名 | `〖@子楚〗` | `〖@子楚\|秦庄襄王〗` |
| 省称 | `〖@桓公〗`（多国均有桓公） | `〖@桓公\|齐桓公〗` |
| 字号"X子"（2026-04-17，083章） | `〖@曹子〗` / `〖@管子〗` / `〖@邹子〗` | `〖@曹子\|曹沫〗` / `〖@管子\|管仲〗` / `〖@邹子\|邹阳〗` |
| 封号 | `〖@安国君〗` | `〖@安国君\|秦孝文王〗` |
| 谥号（大夫） | `〖;简子〗` | `〖@简子\|赵简子〗` |
| 谥号（君主） | `〖;灵公〗` | `〖@灵公\|晋灵公〗` |
| 同名异人 | `〖@信〗`（同章多个） | `〖@信\|韩信〗` / `〖@信\|韩王信〗` |

**谥号标注规则**（2026-04-17新增）：
所有大夫/卿/君主的谥号简称，比照邦国君主谥号处理，使用 `〖@谥号|全名〗`，**不用** `〖;谥号〗`。谥号是人物别称，本质上是人名而非官职头衔。`〖;X〗` 仅用于纯粹的官职头衔（太子、大夫、相国等）。

**规范名优先本名原则**（2026-04-17新增）：
消歧语法 `〖@显示名|规范名〗` 中的"规范名"应**优先使用本名**（姓+名），而非爵号/封号/谥号/字号。

| 错误（规范名用爵号/封号） | 正确（规范名用本名） |
|---|---|
| `〖@绛\|绛侯〗` | `〖@绛\|周勃〗`（009 章统一惯例）|
| `〖@平原\|平原君〗` | `〖@平原\|赵胜〗` |
| `〖@信陵\|信陵君〗` | `〖@信陵\|魏无忌〗` |
| `〖@武安\|武安侯〗` | `〖@武安\|田蚡〗` |

**理由**：规范名的作用是在知识图谱中**唯一指向人物实体**。爵号/封号本身也是别称，会造成多层别名指向同一人的混乱。本名（姓+名）是实体的标准锚点，跨章检索、entity_index 构建、人物关系抽取均以此为基。

**查本名方法**：
1. 查 `kg/entities/data/entity_index.json`（人物实体索引）
2. 查 `doc/史记君主列表.md` 或相关君号索引
3. 对传主：直接看本章首段首次出现的全名
4. 对典故人物：查其专传（刺客列传/世家等）的全名

**默认不消歧的场景**：

- 全名规范且全书唯一所指（如 `〖@司马迁〗`、`〖@吕不韦〗`）
- 历史共识无歧义的上古帝王单名（`〖@尧〗`/`〖@舜〗`/`〖@禹〗`/`〖@汤〗`/`〖@纣〗` 等，参 C3）

**判断标准**：

1. **单字名 → 必须消歧**，上古帝王单名除外（`〖@尧〗`/`〖@舜〗`/`〖@禹〗`/`〖@汤〗`/`〖@纣〗` 等，保留 C3 豁免）
2. 名称片段去除上下文后，能否唯一指向一人？→ 否，必须消歧
3. 同章/同书中是否存在同名/同字号他人？→ 是，必须消歧
4. 名称形式是字/号/封号/谥号/省称等非本名？→ 是，必须消歧

#### B4. 梁魏国名转换（2026-03-30新增，070章）

**问题描述**：
"梁"在战国语境中常指魏国（都城大梁），需根据语境判断是地名（梁邑）还是国名（魏国别称）。

**历史背景**：
- 魏国定都大梁（今河南开封）后，常以都城"梁"代指魏国
- 类似于"秦"既可指秦国，也可指秦地；"齐"既可指齐国，也可指齐地

**语境判断规则**：

| 语境 | 标注 | 示例 |
|------|------|------|
| **国家主体** | `〖◆梁〗` | 从郑至梁、梁南与楚、秦驱韩梁攻齐 |
| **地理范围** | `〖◆梁〗` | 梁之地势、梁之亡可立而须 |
| **具体地名** | `〖=梁〗` | 攻大梁、围梁城 |
| **氏族** | `〖&梁〗` | 梁氏 |

**典型案例**（070章）：

```
✓ 正确：从〖◆郑〗至〖◆梁〗二百馀里（从郑国到魏国）
✓ 正确：〖◆梁〗南与〖◆楚〗境（魏国南边与楚国接壤）
✓ 正确：秦驱〖◆韩〗〖◆梁〗攻〖◆齐〗（秦国驱使韩国魏国攻打齐国）
✓ 正确：〖◆秦〗折〖◆韩〗而攻〖◆梁〗（秦国打败韩国后攻打魏国）
```

#### B6. "X氏"/"某某氏" 一律氏族；邦国是分封政治实体（2026-04-16新增，002章）

**核心判断**：
- **氏族 `〖&〗`**：指血缘/族姓集团，名号通常带"氏"字（陶唐氏、有虞氏、夏后氏、豢龙氏、有熊氏、高阳氏、神农氏、缙云氏…）。即使省略"氏"字（如"陶唐既衰"、"高阳嗣位"），只要语境指血缘集团/族姓延续，仍标 `〖&〗`。
- **邦国 `〖◆〗`**：指**分封制度下的政治实体**，有明确国界、君主、臣民、参与会盟征伐（齐、楚、秦、晋、鲁、卫、陈、宋、郑、吴、越…）。

**典型修正**：
- 〖◆陶唐〗既衰 → `〖&陶唐〗`（陶唐氏为尧之族姓集团）
- 〖◆有虞〗 → `〖&有虞〗`
- 〖◆夏后〗 → `〖&夏后〗`（"夏后氏"省称）

**判断流程**：
1. 名号本身或邻近文本含"X氏"→ 一律氏族 `〖&〗`
2. 上古族号（陶唐/有虞/有熊/高阳/高辛/神农/帝鸿/少暤/颛顼/缙云…）→ 氏族 `〖&〗`
3. 春秋以降参与分封、有国界君主的政治实体（齐楚秦晋等）→ 邦国 `〖◆〗`
4. 同名既可指族也可指国时，按当下语境判：
   - "X氏作乱/专政/衰微/兴起" → 氏族
   - "伐X/与X会盟/X之地/X君" → 邦国

#### B7. 邦国/氏族+人名的标注总则（2026-04-16 新增，014/015 批量深度反思）

**问题背景**：年表/世家中频繁出现 "[前缀][名]" 模式（如 "鲁昭公"、"楚堵敖谡"、"赵鞅"），前缀处理易错。本规则统一归总六类处理方式。

**判断流程**：

1. **判定前缀类型**（春秋/战国语境）：
   - 邦国：秦齐楚燕赵韩魏鲁卫陈蔡曹宋郑吴越晋周（春秋独立政治实体）
   - 氏族（晋卿族）：春秋时期 赵/魏/韩/范/中行/智/郤/栾/胥 等为**晋之卿族**，非独立邦国
   - 春秋末/战国初（~453 BC三家分晋后）：赵/魏/韩 才成为**邦国**

2. **判定Y的性质**：
   - 谥号：以 公/侯/王 结尾（文公/桓侯/庄王）
   - 本名：无公/侯/王 后缀（熊严/阖闾/寿梦/鞅/盾）
   - 通名：人/君/室/亡 等非专名

**六类处理规则**：

| # | 前缀 X | 跟随 Y | 处理 | 例 |
|---|--------|--------|------|------|
| 1 | **邦国** | **谥号**（含公/侯/王） | **合并**→`〖@国谥号〗` | `〖@晋文侯〗`、`〖@鲁昭公〗`、`〖@楚庄王〗` |
| 2 | **邦国** | **本名**（非公/侯/王） | **分标**→`〖◆国〗〖@名〗` | `〖◆楚〗〖@熊严〗`、`〖◆吴〗〖@阖闾〗`、`〖◆楚〗〖@若敖〗` |
| 3 | **氏族** | **名** | **合并**→`〖@氏名〗`（氏族作姓氏） | `〖@赵鞅〗`、`〖@赵盾〗`、`〖@魏绛〗`、`〖@郤缺〗` |
| 4 | **邦国** | **通名** | **去标通名**→`〖◆国〗通名` | `〖◆齐〗人`、`〖◆周〗室`、`〖◆燕〗君`、`〖◆秦〗亡` |
| 5 | **谥号**（已标） | **本名**（未标） | **补标本名**→`〖@谥号〗〖@名〗` | `〖@桓公〗〖@鲍〗`、`〖@晋文侯〗〖@仇〗`、`〖@鲁惠公〗〖@弗湦〗` |
| 6 | **汉代封爵**（留/淮阴/武安/绛/建成…侯） | 本名 | **分类分标**→`〖;X侯〗〖@本名〗` | `〖;留侯〗〖@张良〗`、`〖;淮阴侯〗〖@韩信〗`、`〖;武安侯〗〖@田蚡〗` |

**为什么区分#1和#6**：
- 春秋国君谥号（#1）：规范尊称，等同人物本身，无"其他本名"在文中普遍使用 → `〖@〗`
- 汉代封爵（#6）：服务后授予，本人已有本名（韩信/张良）常用 → `〖;〗` 作职衔

**年表格式特有模式**（#5 的实际应用场景）：

年表行内常见 `〖@谥号〗[本名]〖%元年〗`，中间的本名极易遗漏：
- 批量扫描命令：`grep -nE '〖@[^〗]+〗[一-龥]{1,3}〖%元年〗' chapter_md/NNN_*.tagged.md`

**复姓例外（不拆分）**：

下列 X 实为**复姓**，不适用 #2 分标：
- `〖@夏侯婴〗`/`〖@夏侯胜〗`（夏侯 复姓）
- `〖@屈侯鲋〗`（屈侯 复姓）
- `〖@逢侯丑〗`（逢侯 复姓）
- `〖@北宫伯子〗`（北宫 复姓）
- `〖@蘧伯玉〗`（蘧氏+字）
- `〖@铜鞮伯华〗`（铜鞮地名+字）
- `〖@崇侯虎〗`（崇侯虎 固定称谓）
- `〖@公伯禽〗`（公伯禽 作为整体称谓，伯禽是名）

**关键判据总则**：
- 前缀是"邦国"时：谥号→合并；本名→分标；通名→去标通名
- 前缀是"氏族"时：一律合并（作姓氏用）
- 前缀是"封爵"时：分标 `〖;〗` + `〖@〗`

**批量工具**：

```python
# 匹配 X〖@Y〗 模式，判定Y是否为谥号（以 公/侯/王 结尾）

def decide(y):
    if y.endswith(('公', '侯', '王')):
        return 'merge'    # 合并为〖@XY〗
    return 'split'        # 分标〖◆X〗〖@Y〗
```

#### C5. 跨时代人名消歧校验（2026-04-16新增，004章）

单字人名消歧时，必须校验候选规范名的**生卒年与当前语境年代重叠**。同字不同人（如「满」=周穆王 vs 王孙满；「广」=田广 vs 韩广）应按以下优先级选择规范名：

1. **最近在场人物优先**：当前段落已出现的全名
2. **同段/同篇人物优先**：本章主角及其家族成员
3. **同时代人物优先**：与当前纪年相符的历史人物

**典型错误案例**：
- 004 L147「立〖@昭王〗子〖@满〗」第三轮误消歧为 `|王孙满`（春秋人，相距约400年）→ 应为 `|周穆王`
- 007 L971「立〖@田荣〗子〖@广〗」第三轮误消歧为 `|韩广`（燕王）→ 应为 `|田广`（齐王）

**操作清单**：
- 消歧前 grep 检查全章是否有更近的同字全名
- 消歧后用人物所属国/朝代核验年代区间是否吻合
- 父子相承段落优先消歧为同氏族成员

**自动检查命令**（聚合一章中所有 `〖@X|YZ〗` 的候选规范名）：
```bash
grep -oE '〖@[^〗|]+\|[^〗]+〗' chapter_md/NNN_*.tagged.md | sort -u
```
- 若同一 X 对应多个 YZ 候选 → 必须人工复核每处归属
- 典型陷阱（同字异人）：满（周穆王/王孙满）、广（田广/韩广）、信（韩信/韩王信）、馀（陈馀/张馀）、敖（张敖/共敖；张敖=赵王，共敖=临江王）

**Gazetteer 建议**：项目层维护单字名规范名映射表（如 `scripts/entity_gazetteer`），按"姓氏+父名/封国+时代"三元组唯一标识。

**长沙王异姓例外**（2026-04-16 017章补充）：
C2 "汉同姓诸侯→刘X" 规则**不适用于异姓王**。长沙国自吴芮至吴著为异姓诸侯，规范名应为 `吴X`（臣/回/右/著/产/庸/发）：
- `〖@臣|吴臣〗`（吴芮子，第二代长沙王）
- 其余吴氏单字名类推

- **享国类**：享国XX年、有国XX年、传国XX年、世XX年

- **留居/滞留类**（2026-04-16 031/035章补充）：哭 X 日/服 X 年/居丧 X 年/哀 X 日/留 X 岁/留 X 年

**操作要点**：
- 凡上述触发词后跟数量+年/岁/月，默认是时长（A8 默认纪年），必须加 `|时长` 显式消歧

3. 发现半角引号**立即停止其他操作**，用 Python 脚本修复
4. **批量修改**（如全章替换）必须用 Python 脚本，绝不用 Edit 的 replace_all

**历史教训**：第四轮反思中多章半角引号污染均源于 Edit 工具的静默转换，部分甚至在"修复"过程中引入新污染。

**⚠️ 不适用的扩展规则**（2026-04-17 明确）：

> "全角引号左右数量必须相等"（LDQ == RDQ，LSQ == RSQ）—— **此规则不成立，不应强制执行**。

**理由**：《史记》长对话分段时，每段开头只有左引号 `"`，整段对话结束时才出现右引号 `"`。按行或按段统计时左右引号不等是正常现象。

**正确的检查方式**：
- 只扫描**半角引号**（U+0022/U+0027/U+0060）是否 0 处（E3 主规则）
- **不要**强制全角引号左右数量平衡
- 真正需要关注的：每对引号最终**都能闭合**（全文看），但不要求按段/按行对称

#### A115. 爵位/封号 vs 谥号侯 的 `;` / `@` 区分（2026-04-17新增，096/022章）
