**版本**: v1.0
**创建日期**: 2026-03-28

## 技能目标

对古籍进行多版本互校，建立标准底本（定本），为后续的实体标注、知识抽取等工作提供可靠的文本基础。

## 适用范围

- 《史记》130篇的版本校勘
- 可扩展至其他传世古籍（如《汉书》《左传》等）

## 核心原则

1. **底本优先**: 选择权威底本作为主校本
2. **多版本对校**: 至少使用3个以上版本进行对校
3. **异文记录**: 详细记录所有文字差异
4. **繁简统一**: 明确底本采用繁体或简体
5. **标点审慎**: 标点符号差异需结合句读判断
6. **标点符号禁令**: ⚠️ **绝对禁止修改标点符号类型**（全角半角、引号类型等）

### 版本选择建议

**推荐底本**: `corpus/archive/chapter/*.txt` (简体分章版)
**理由**:
- ✅ 已按130篇分章，便于逐章校对
- ✅ 简体字，符合当前项目标注文件格式
- ✅ 纯文本，无注释干扰
- ✅ 已有段落编号体系(`chapter_numbered/`)可参照

**主要参校本**:
1. `corpus/shiji/wikisource_shiji/` - 繁体权威版本
2. `corpus/shiji/史记四库.txt` - 四库全书版本
3. `corpus/shiji/wikisource_sanjia/` - 三家注版本(可参考校勘记)

### 校对参考版本定义 ⚠️

**重要说明**：在校对工作中，"其他版本"或"参考版本"有明确的范围界定。

#### 包括的参考版本（真正的"其他版本"）

以下版本可作为校对对比的参考依据：

| 版本 | 路径 | 说明 |
|------|------|------|
| **三家注繁体版** | `corpus/shiji/史記三家注.繁体.txt` | 含集解/索隐/正义校勘记 |
| **三家注合本** | `corpus/shiji/史记集解三家注索隐正义.txt` | 简体版，正文与注释混排 |
| **维基三家注** | `corpus/shiji/wikisource_sanjia/*.html` | 繁体，有校勘标记 |
| **维基正文** | `corpus/shiji/wikisource_shiji/*.html` | 繁体权威版本 |
| **四库版** | `corpus/shiji/史记四库.txt` | 四库全书版本 |
| **繁体正文** | `corpus/shiji/史記正文.繁体.txt` | 繁体纯文本 |
| **外部资源** | ctext.org、中华书局本等 | 在线或纸质权威版本 |

#### 不包括的版本（项目内部同底本派生文件）

以下文件都是**同一底本的不同处理阶段**，不算"其他版本"：

| 路径 | 说明 | 为什么不算"其他版本" |
|------|------|---------------------|
| `corpus/archive/chapter/*.txt` | 项目标准底本 | 这是校对的**基准本身** |
| `corpus/archive/chapter_improved/*.txt` | 改进版（保留空行） | 基于同一底本，只是格式调整 |
| `corpus/archive/chapter_numbered/*.txt` | 编号版（带段落号） | 基于同一底本，只是添加编号 |
| `docs/original_text/*.txt` | 工作定本 | 基于同一底本的工作副本 |
| `chapter_md/*.tagged.md` | 标注文件 | 基于同一底本添加知识标注 |

**判断原则**：
- ✅ **是参考版本**：来自不同文献传承系统（如三家注系统、四库系统）
- ❌ **非参考版本**：项目内部基于同一底本的格式变换或标注加工

**实际应用**：
```
用户问："这个字在其他版本中是什么？"

✅ 正确理解：查看三家注版本、四库版本、维基版本等
❌ 错误理解：查看 corpus/archive/chapter_improved/ 或 docs/original_text/
                （这些都是同一底本，字应该一样）
```

## 校对工作流程

### 阶段1: 准备工作

1. **提取分章文本**
   - 底本: 已有 `corpus/archive/chapter/NNN_篇名.txt`
   - 参校本: 需从整本中提取对应章节

2. **繁简转换**
   - 将繁体参校本转换为简体(便于对比)
   - 保留繁体原文用于复核

3. **文本对齐**
   - 去除标点符号
   - 统一空格/换行格式
   - 按段落或句子对齐

### 阶段2: 逐字对校

**对校方法**:

```python
# 伪代码示例
底本文本 = 读取("corpus/archive/chapter/001_五帝本纪.txt")
参校本A = 读取("wikisource_shiji/001_五帝本纪.html") |> 提取纯文本 |> 繁转简
参校本B = 读取("史记四库.txt") |> 提取第001章
参校本C = 读取("wikisource_sanjia/001_五帝本纪.html") |> 提取正文 |> 繁转简

异文列表 = []
for 每个字符位置 i in 底本文本:
    if 底本[i] != 参校A[i] or 底本[i] != 参校B[i] or 底本[i] != 参校C[i]:
        异文列表.append({
            "位置": i,
            "上下文": 底本[i-10:i+10],
            "底本字": 底本[i],
            "参校A": 参校A[i],
            "参校B": 参校B[i],
            "参校C": 参校C[i],
        })
```

### 阶段3: 异文判定

对每个异文点进行判定:

| 异文类型 | 处理方式 | 示例 |
|---------|---------|------|
| **异体字** | 统一为通行字 | 竝→并, 衞→卫 |
| **通假字** | 保留底本,出校记 | 蚤→早(保留"蚤") |
| **音同误写** | 择善而从 | 既→即, 其→期 |
| **脱字衍字** | 参考多数版本 | 某版多一"而"字 |
| **古今字** | 保留古字形 | 无→毋(保留"无") |
| **标点差异** | ⚠️ **只记录差异，禁止修改** | "天下病而丹朱得其利"句读 |

**判定原则**:
1. 多数版本一致 → 采用多数
2. 版本各异 → 参考三家注校勘记
3. 仍无定论 → 保留底本,出校记存疑

**⚠️ 标点符号修改铁律**:
- ❌ **绝对禁止**修改标点符号的类型（全角↔半角、左右引号类型等）
- ❌ **绝对禁止**添加或删除标点符号
- ❌ **绝对禁止**修改引号类型（`"` U+201C ↔ `"` U+0022）
- ✅ **只允许**修改原文汉字本身
- ✅ **只允许**在校对报告中记录标点差异，供人工审核决策

### 阶段4: 生成校对报告

每章生成一份校对报告:

```
logs/curation/reports/001_五帝本纪_校对报告.md
```

## 关键技术点

### 0. 标点符号处理规范 ⚠️

**底本标点必须原样保留**：

古籍底本使用的标点符号（尤其是全角标点）有其历史传承和文献学意义，**严禁**在校勘过程中修改标点符号的字符编码。

#### 常见标点符号陷阱

| 符号类型 | 正确（底本常用） | 错误（禁止转换为） | Unicode编码 |
|---------|----------------|------------------|------------|
| 左双引号 | `"` | `"` | U+201C vs U+0022 |
| 右双引号 | `"` | `"` | U+201D vs U+0022 |
| 左单引号 | `'` | `'` | U+2018 vs U+0027 |
| 右单引号 | `'` | `'` | U+2019 vs U+0027 |
| 全角逗号 | `，` | `,` | U+FF0C vs U+002C |
| 全角句号 | `。` | `.` | U+3002 vs U+002E |
| 全角冒号 | `：` | `:` | U+FF1A vs U+003A |
| 全角分号 | `；` | `;` | U+FF1B vs U+003B |

#### 修改文本时的安全操作

**⚠️ 适用对象**：本节针对AI助手（如Claude）进行自动化校勘修改时的操作规范。人工校勘者请使用支持Unicode的文本编辑器（如VS Code、Sublime Text）并手动核对每个字符。

**AI助手使用文本替换功能时的规范**：

1. **只匹配需要修改的汉字**：
   ```
   示例：修改 corpus/archive/chapter/002_夏本纪.txt

   ✅ 正确做法：只选择需要修改的字
   查找内容: "山行乘暐"
   替换为:   "山行乘檋"
   选项: 全文替换
   ```

2. **绝对避免选择包含标点的字符串**：
   ```
   ❌ 错误做法：包含引号会导致引号类型被自动转换

   查找内容: '禹曰："鸿水滔天...山行乘暐'
   替换为:   '禹曰："鸿水滔天...山行乘檋'

   ⚠️ 为什么会出错：
   - 文本编辑器或自动化工具可能对引号进行Unicode标准化
   - 原文的左双引号 " (U+201C) 可能被转为 ASCII引号 " (U+0022)
   - 这违反了"禁止修改标点符号"的铁律
   ```

3. **正确做法示例对比**：
   ```
   ✅ 推荐：最小化匹配范围
   查找: "山行乘暐"
   替换: "山行乘檋"

   ✅ 可接受：如果需要唯一定位，增加汉字上下文（不含标点）
   查找: "泥行乘橇，山行乘暐，行山"
   替换: "泥行乘橇，山行乘檋，行山"
   注意：此处包含全角逗号（U+FF0C），相对安全，但仍建议避免

   ❌ 禁止：跨越句子边界或包含引号、冒号
   查找: '禹曰："...山行乘暐...'  ← 包含冒号和引号，危险！
   ```

3. **修改后必须验证**：
   ```bash
   # 用十六进制检查标点字节码是否改变
   git diff | grep -A2 "@@" | xxd
   ```

4. **一旦发现标点被误改，立即回退**：
   ```bash
   git checkout HEAD -- corpus/archive/chapter/NNN_篇名.txt
   ```

#### 文本替换操作准则（AI助手专用）

AI助手在进行自动化文本替换时，必须遵守以下规则：

- **查找字符串长度**: 尽量短，只包含需要修改的字及最小上下文
- **避免跨标点**: 查找字符串不要跨越逗号、句号、引号、冒号等标点
- **全文替换优先**: 当同一个字需要全文替换时，优先使用"全文替换"功能，避免需要更多上下文定位
- **⚠️ 引号是最危险的**: ASCII引号(`"` `'`)和Unicode引号(`"` `"` `'` `'`)极易混淆，查找/替换字符串中绝对禁止包含任何引号

**🚨 AI助手关键记忆点**：
> 在古籍校勘中进行文本替换时，查找和替换的字符串**只能包含汉字和必要的全角标点**（如全角逗号`，`），**绝对不能包含冒号和引号**，否则会导致标点符号的Unicode编码被意外转换，违反"标点符号禁令"。
>
> **技术说明**：Claude Code的Edit工具在处理包含引号的字符串时，可能会将Unicode左右引号（U+201C/U+201D）转换为ASCII引号（U+0022），因此必须避免在`old_string`和`new_string`参数中包含引号。

**验证方法**：
```bash
# 提取修改前后的标点符号Unicode编码对比
git show HEAD:corpus/archive/chapter/002_夏本纪.txt | sed -n '21p' | od -An -tx1 > /tmp/old.hex
sed -n '21p' corpus/archive/chapter/002_夏本纪.txt | od -An -tx1 > /tmp/new.hex
diff /tmp/old.hex /tmp/new.hex
# 应该只有汉字字节变化，标点字节不变
```

### 1. 繁简转换注意事项

**常见陷阱**:

| 繁体 | 简体(误) | 简体(正确) | 说明 |
|------|---------|-----------|------|
| 後 | 后 | 后(方位)/後(时间) | 需区分"皇后"与"以後" |
| 髮 | 发 | 发(头发)/發(发展) | 需区分"理髮"与"發展" |
| 干 | 干 | 干(盾)/幹(树干) | "干戈"不同于"树幹" |
| 復 | 复 | 復(恢复)/複(重复) | 需区分语境 |
| 藉 | 借 | 藉(凭藉)/借(借贷) | 需区分语境 |

**处理方案**:
- 使用 OpenCC (开放中文转换) 工具
- 人工复核关键多音字

### 2. HTML文本提取

维基文库HTML文件包含大量标签,需提取纯文本:

```python
from bs4 import BeautifulSoup

def extract_text_from_wikisource(html_path):
    with open(html_path, 'r', encoding='utf-8') as f:
        soup = BeautifulSoup(f.read(), 'html.parser')

    # 去除注释标签
    for tag in soup.find_all(['sup', 'span'], class_=['reference', 'index']):
        tag.decompose()

    # 提取正文
    main_text = soup.find('div', id='bodyContent')
    return main_text.get_text()
```

### 3. 文本对齐算法

使用**最长公共子序列(LCS)**算法对齐不同版本:

```python
def align_texts(text1, text2):
    """
    返回两个文本的对齐结果,标记插入/删除/替换位置
    """
    from difflib import SequenceMatcher
    matcher = SequenceMatcher(None, text1, text2)
    opcodes = matcher.get_opcodes()

    alignment = []
    for tag, i1, i2, j1, j2 in opcodes:
        if tag == 'equal':
            alignment.append(('=', text1[i1:i2]))
        elif tag == 'replace':
            alignment.append(('~', text1[i1:i2], text2[j1:j2]))
        elif tag == 'delete':
            alignment.append(('-', text1[i1:i2]))
        elif tag == 'insert':
            alignment.append(('+', text2[j1:j2]))

    return alignment
```

## 派生文件同步更新规范

⚠️ **重要说明**：本章节针对《史记》知识库项目的特殊情况。理论上，多版本互校应该在所有工序之前第一个完成。但本项目因为在探索过程中，校对工作后置，导致已经产生了多个派生文件系统。因此需要按校对结果依次修订所有派生文档。

### 派生文件体系

《史记》知识库项目中，`corpus/archive/chapter/` 作为标准底本，衍生出以下派生文件系统：

```
corpus/archive/chapter/           ← 原始底本（不可变，校对基准）
├── NNN_篇名.txt
│
派生文件系统（从corpus/archive/chapter同步更新）：
├── corpus/archive/chapter_improved/   ← 改进版 (保留空行格式)
├── corpus/archive/chapter_numbered/   ← 编号版 (带[N]段落编号)
└── chapter_md/                 ← 标注文件 (带〖@〗等标记)
│
独立工作定本（供其他工序使用）：
└── docs/original_text/         ← 工作定本（可能与corpus/archive/chapter有差异）
```

**重要说明**：
- `corpus/archive/chapter/` 是lint完整性检查的唯一基准
- `docs/original_text/` 是历史遗留的工作定本，供渲染等工序使用
- 标注文件必须与 `corpus/archive/chapter/` 完全一致

### 同步更新原则

1. **只替换字符，不改格式**
   - 只修改校对变更的汉字
   - 严格保持每个派生系统的原有格式特征
   - 不增删空行、不修改编号、不改变标注符号

2. **格式特征保持**
   - `corpus/archive/chapter_improved/`: 保留段落间空行
   - `corpus/archive/chapter_numbered/`: 保留 `[N]` 编号格式
   - `chapter_md/`: 保留所有 `〖TYPE 〗` 标注符号
   - `docs/original_text/`: （独立维护，不强制同步）

3. **标注符号铁律**
   - `chapter_md/` 中的标注符号（`〖@〗` `〖◆〗` `⟦◈⟧` 等）绝对不可修改
   - 只替换标注符号内外的原文汉字
   - 标注的消歧语法 `〖TYPE 显示名|规范名〗` 不改变显示文本

### 标准操作流程 (SOP)

基于 001-004 章节的实际操作经验，形成以下标准流程：

#### 步骤1: 完成校对并生成报告

```bash
# 1. 对 corpus/archive/chapter/NNN_篇名.txt 进行多版本互校
# 2. 生成校对报告: logs/curation/reports/NNN_篇名_校对报告.md
# 3. 报告中列出所有字符变更（如：筴→策, 暐→檋, 饹→奔）
```

**输出文件**:
- `logs/curation/reports/NNN_篇名_校对报告.md`

#### 步骤2: 应用校对结果到标准底本

```bash
# 修改 corpus/archive/chapter/NNN_篇名.txt
# 严格按照校对报告中的"建议改动"逐字修改
```

**关键点**:
- 只修改"建议改动"项
- 保留"无需改动"项（如通假字、异体字）
- 使用精确的字符串替换，避免误改

#### 步骤3: 同步更新派生文件系统

按以下**顺序**依次更新各派生文件：

##### 3.1 更新 `corpus/archive/chapter_improved/`

**目标**: 保留段落间空行，只替换字符

**操作方法**:

#### 案例6: 045_韩世家 - 补"年"字

**问题发现**：
```
原文：宣惠王五年...八年...十一年...十四，秦伐败我鄢。
```
前后都有"年"字，唯独"十四"后缺失，不符合行文习惯。

**版本对比**：
- 武英殿本系统：`十四，秦伐败我鄢`
- 三家注繁体版：`十四，秦伐敗我鄢`
- 三家注合本：`十四，秦伐败我鄢`
- **所有版本都无"年"字**

**判定依据**：
1. 前文："五年"、"八年"、"十一年" - 都有"年"
2. 后文："十六年"、"十九年" - 都有"年"
3. 全篇《韩世家》其他所有纪年（数十处）都带"年"字
4. 唯独此处省略，不符合史记行文习惯

**校勘决策**：
- 虽然所有版本都无"年"字，但根据上下文和行文习惯判断为**脱字**
- 修改：`十四，秦伐败我鄢` → `十四年，秦伐败我鄢`
- **这是一种"校勘补字"** - 即使所有版本都缺失，但根据内证（上下文规律）判断应予补正

**校勘类型**：内证校勘（Context-based Collation）
- 外证校勘：依据其他版本修改
- **内证校勘**：根据文本内部规律修改（即使所有版本都一致）

**同步文件**：4个txt文件 + 标注文件（已包含修改）

**验证结果**：✅ 完整性检查通过（0处实质差异）

**Commit**: `e7a42255`

#### 案例7: 089_张耳陈馀列传 - "信诸侯"vs"信都侯"

**问题发现**：
```
原文（同一行内矛盾）：
寿为乐昌侯，侈为信都侯。高后崩，诸吕无道，大臣诛之，而废鲁元王及乐昌侯、信诸侯。
```
**前文明确写"侈为信都侯"，后文却写"废...信诸侯"** - 同一行文内出现矛盾。

**版本对比**：
- 简体底本（项目）：`而废鲁元王及乐昌侯、信诸侯` ❌
- 三家注合本：`而废鲁元王及乐昌侯、信诸侯` ❌
- 史记集解：`而废鲁元王及乐昌侯信都侯` ✅
- 史记四库：`而废鲁元王及乐昌侯信都侯` ✅
- 史记正义：`而废鲁元王及乐昌侯信都侯` ✅

**前文互证**（本行内）：
```
同一行前半句：侈为信都侯
同一行后半句：废...信诸侯  ← 矛盾！
```

**判定依据**：
1. **前文照应**：上文明确"侈为信都侯"
2. **版本支持**：史记集解、四库、正义三个权威版本均作"信都侯"
3. **历史逻辑**：废除的三人应为：
   - 鲁元王（张偃，张敖长子，因母为吕后女而封王）
   - 乐昌侯（张寿，张敖次子）
   - **信都侯**（张侈，张敖三子）
4. **文字讹误分析**："信都侯"→"信诸侯"可能为传抄误写：
   - "都"字形近于"诸"（在某些书写中）
   - 或OCR识别错误（数字化过程）

**校勘决策**：
- 修改：`而废鲁元王及乐昌侯、信诸侯` → `而废鲁元王及乐昌侯、信都侯`
- **校勘类型**：前文互证 + 多版本外证
  - **前文互证**：同一段落前后矛盾，采用前文
  - **外证支持**：权威版本多数支持"信都侯"

**特殊性**：
这是一个**同一行文内自我矛盾**的典型案例：
- 前句："侈为**信都侯**"
- 后句："废...**信诸侯**"
- 矛盾距离：不到30字

这类错误通常是：
1. 传抄笔误（形近字讹误）
2. 数字化OCR错误
3. 繁简转换失误

**验证结果**：✅ 所有文件已同步修正，完整性检查通过

#### 校勘方法论总结

**外证校勘** vs **内证校勘**：

| 类型 | 依据 | 示例 | 适用场景 |
|------|------|------|---------|
| **外证校勘** | 参考其他版本 | 011章：三家注有"免"字 → 补入 | 版本之间有差异 |
| **内证校勘** | 文本内部规律 | 045章：前后都有"年"字 → 补"年" | 所有版本一致但疑似有误 |

**内证校勘的判定标准**：
1. ✅ 上下文有明显规律（如纪年格式统一）
2. ✅ 缺失处打破规律（如唯独一处省略）
3. ✅ 无其他合理解释（如特殊语境、修辞手法）
4. ✅ 补正后符合文义和历史事实

**注意事项**：
- 内证校勘需要更谨慎，因为没有版本依据
- 应在校勘报告中明确说明判断理由
- 优先采用外证，内证作为补充

### 常见问题与解决方案

#### Q1: 如何处理断行错误？

**问题**:
```
原文: "乃命伯
       申诫太仆国之政，作
       命。"
应为: "乃命伯冏申诫太仆国之政，作冏命。"
```

**解决方案**:
```bash
# 使用sed的多行模式
sed -i ':a;N;$!ba;s/乃命伯\n申诫太仆国之政，作\n命。/乃命伯冏申诫太仆国之政，作冏命。/g' file.txt

# 或者手动使用文本编辑器修复
```

#### Q2: 如何避免误替换同形字？

**问题**: "饹"字在不同上下文有不同含义

**解决方案**:
```bash
# ❌ 错误做法：全文替换
sed -i 's/饹/奔/g' file.txt  # 可能误改其他"饹"字

# ✅ 正确做法：精确匹配上下文
sed -i 's/而饹戎狄之间/而奔戎狄之间/g' file.txt
sed -i 's/克饹/克奔/g' file.txt
sed -i 's/女饹之/女奔之/g' file.txt
```

#### Q3: 标注文件中的标注符号被误改怎么办？

**问题**: 替换时不小心改变了 `〖@〗` 等标注符号

**解决方案**:
```bash
# 1. 立即回退
git checkout HEAD -- chapter_md/NNN_篇名.tagged.md

# 2. 重新执行替换，使用更精确的匹配
# ✅ 正确：包含标注符号的精确匹配
sed -i 's/乘〖•暐〗/乘〖•檋〗/g' chapter_md/002_夏本纪.tagged.md

# 3. 验证标注符号完整性
grep -o "〖" chapter_md/NNN_篇名.tagged.md | wc -l
grep -o "〗" chapter_md/NNN_篇名.tagged.md | wc -l
# 两者数量应该相等
```

#### Q4: 如何处理繁简混用的情况？

**问题**: 参校本有繁体字，底本是简体

**解决方案**:
```bash
# 1. 识别异文是繁简差异还是真正的讹误
# 2. 如果是繁简差异，保持底本字形
# 3. 如果是讹误，改为正确字形（简体）

# 示例：
# 底本: "后稷"
# 参校: "後稷"
# 判定: 繁简差异，保留底本"后稷"
```
