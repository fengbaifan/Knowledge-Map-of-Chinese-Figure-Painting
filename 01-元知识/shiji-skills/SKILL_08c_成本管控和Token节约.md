# SKILL 08c: 成本管控和Token节约（规划）

> 优化LLM调用成本，管理计算资源，实现知识库建设的经济可持续性。

---

## 一、功能概述

知识库建设涉及大量LLM调用，Token成本不容忽视。本功能计划建立成本管控体系：

- **成本监控**：追踪Token消耗和费用
- **成本优化**：减少不必要的LLM调用
- **成本预测**：估算未来工作的成本
- **性价比评估**：衡量投入产出比

---

## 二、成本来源分析

### 2.1 LLM调用成本

**高成本操作**：
- 实体标注（130章×平均2-3万token/章）
- 实体反思（多轮×长文本）
- 事件提取（复杂prompt×长context）
- 关系推理（大规模图遍历）

**成本构成**：
- Input tokens（提示词+上下文）
- Output tokens（生成内容）
- 不同模型价格差异（GPT-4 vs Claude vs本地模型）

### 2.2 人工成本

**高人工操作**：
- 标注质量审查
- 错误修正
- 规则设计
- 知识验证

### 2.3 计算资源成本

- 向量化计算
- 推理引擎运行
- 数据库查询
- 存储成本

---

## 三、Token节约策略

### 3.1 Prompt优化

**策略**：
- 精简提示词，去除冗余示例
- 使用few-shot而非zero-shot（减少试错）
- 分步prompt而非一次性大prompt
- 复用prompt模板

**效果**：
- 减少30-50% input tokens
- 提高输出准确率，减少返工

### 3.2 Context压缩

**策略**：
- 只传递必要的上下文
- 使用摘要而非全文
- 分段处理长文本
- 缓存重复上下文

**效果**：
- 减少40-60% context tokens
- 加快响应速度

### 3.3 模型选择

**策略**：
- 简单任务用便宜模型（GPT-3.5、Claude Haiku）
- 复杂任务用高级模型（GPT-4、Claude Opus）
- 探索本地开源模型（Llama、Qwen）
- 混合使用多个模型

**效果**：
- 综合成本降低50-70%
- 保持关键任务质量

### 3.4 批处理优化

**策略**：
- 批量处理相似任务
- 复用embedding和计算结果
- 异步处理非紧急任务
- 并行化独立任务

**效果**：
- 减少重复计算
- 提高吞吐量

### 3.5 缓存策略

**策略**：
- 缓存常见查询结果
- 缓存中间计算结果
- 复用相似任务的输出
- 版本化管理缓存

**效果**：
- 避免重复调用LLM
- 提升响应速度

## 四、成本监控

### 4.1 实时监控

**监控指标**：
- 每日Token消耗量
- 每任务平均成本
- 不同模型的使用占比
- 异常高成本操作告警

### 4.2 成本分析

**分析维度**：
- 按工序分类（标注/反思/推理）
- 按章节分类（哪些章节成本高）
- 按模型分类（GPT-4 vs Claude）
- 时间趋势分析

### 4.3 预算管理

**管理策略**：
- 设定月度/季度预算
- 分配各工序的成本配额
- 超预算时的应对机制
- 成本效益评估

## 五、性价比评估

### 5.1 投入产出比

**评估公式**：
```
ROI = 知识价值 / 总成本

知识价值 = 
  实体数量×实体质量 +
  关系数量×关系质量 +
  推理知识数量×推理准确率

总成本 = 
  LLM费用 +
  人工时间成本 +
  计算资源成本
```

### 5.2 质量-成本曲线

**分析**：
- 哪些工序投入产出比最高？
- 成本增加10%能提升多少质量？
- 质量要求降低10%能节省多少成本？

### 5.3 边际效益

**问题**：
- 第N轮反思的边际收益是多少？
- 增加覆盖度的边际成本如何变化？
- 何时应该停止优化？

## 六、成本优化案例

### 案例1：实体标注

**原方案**：GPT-4直接标注全文
- 成本：$5/章 × 130章 = $650

**优化方案**：
1. Claude Haiku初筛 + GPT-4复审
2. 复用前章的prompt经验
3. 只对疑难部分用GPT-4

**优化后成本**：$2/章 × 130章 = $260
**节约**：60%

### 案例2：实体反思

**原方案**：每章3轮全文反思

**优化方案**：
1. 第1轮：规则自动检测（免费）
2. 第2轮：只对问题区域LLM反思
3. 第3轮：人工抽查

**节约**：70%
