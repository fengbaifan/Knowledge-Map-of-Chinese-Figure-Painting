# SKILL 01a: 标注完整性维护 — 标注铁律的守护者

> **标注铁律**：标注工作只能添加`〖TYPE〗`标记符号，不得对原文字符做任何修改（增删改）。

---

## 一、铁律的重要性

### 1.1 为什么需要铁律

标注的本质是**语义标记**，而非**文本编辑**：

**⚠️ 禁止使用半角符号**：
- 在使用 Edit 工具修改标注文件时，**严禁引入半角引号（U+0022 `"`）**
- 必须使用全角引号对：左引号（U+201C `"`）+ 右引号（U+201D `"`）
- 半角符号会破坏文本完整性验证，导致标点规范化差异

```
原文:  五子作歌，太康失政。
标注:  〖$五〗子作歌，〖@太康〗失政。
去标:  五子作歌，太康失政。  ← 必须与原文逐字相同
```

**违反铁律的后果**：
- ❌ 实体索引基于错误文本，指向不存在的词句
- ❌ 事件抽取对应不到原文段落
- ❌ 三家注锚点失效
- ❌ 读者对照原书时发现文字不一致，丧失信任

### 1.2 完整性的定义

**完整性** = 去除全部标注符号后，所得纯文本与 `corpus/archive/chapter/*.txt` 原始底本逐字相同

```
去除对象：
- Markdown标题 (# ### ...)
- 实体标注 (〖@人名〗 〖=地名〗 ...)
- 动词标注 (⟦◈伐⟧ ⟦◉诛⟧ ...)
- 语义区块 (::: 围栏)
- 段落编号 ([1] [1.1] ...)
- 列表符号 (- 项)
- 引用符号 (> 文字)
- 粗体 (**文字**)
- 分隔线 (---)
```

**特殊处理**：
- **原文第一行**：`corpus/archive/chapter/*.txt` 的第一行通常是章节标题（如不以书名号`《`开头），与标注文件的 Markdown 标题对应，比对前自动跳过
- 例如：`001_五帝本纪.txt` 第一行是 `五帝本纪第一`，对应 `001_五帝本纪.tagged.md` 的 `# 五帝本纪第一`

## 二、工具链

### 2.1 完整性检查脚本

```bash
# 检查全部130章
python scripts/lint_text_integrity.py

# 检查指定章节
python scripts/lint_text_integrity.py 002 005 013

# 显示全部差异（包括标点/编码变体）
python scripts/lint_text_integrity.py --all
```

**输出格式**：

```
  ✓ 001_五帝本纪: 0处实质差异  28处标点  1处编码
  ✗ 002_夏本纪: 1处实质差异  49处标点  2处编码
  ✓ 003_殷本纪: 0处实质差异  12处标点  0处编码
```

报告保存到 `logs/lint_text_integrity.txt`（全部章节）或 `logs/lint_text_integrity_XXX.txt`（指定章节）。

### 2.2 差异分类

脚本自动将差异分为三类：

| 分类 | 说明 | 是否需要修复 |
|------|------|--------------|
| **实质差异** | 汉字字符被增加/删除/改写 | ✅ 必须修复（P0） |
| **标点规范化** | 全角↔半角标点（，→, 等） | ❌ 可忽略 |
| **编码规范化** | PUA私用区字符→标准Unicode | ❌ 可忽略 |

### 2.3 规范化映射

脚本通过 `_NORM_TABLE` 消除等价变体差异：

```python
_NORM_TABLE = {
    # 引号全部删除（标注规范：原文无引号时标注可添加全角引号）
    '\u201c': '',  # "
    '\u201d': '',  # "
    # 繁简变体
    '\u540e': '\u5f8c',  # 后 → 後
    '\u4e8e': '\u65bc',  # 于 → 於
    # 通假字
    '\u8c37': '\u7a40',  # 谷 → 穀
    ...
}
```

**规范化后仍存在的差异** → 实质差异，需要修复。

### 2.4 规范化映射表完整列表

完整映射表定义在 `scripts/lint_text_integrity.py:51-81`，包含：

**引号类**（自动删除）：
- `" "` (U+201C/D 左右双引号)
- `「 」` (U+300C/D 直角引号)
- `『 』` (U+300E/F 双直角引号)
- `' '` (U+2018/9 左右单引号)
- `" '` (半角引号，历史遗留)

**繁简变体**（自动映射到繁体/标准形式）：

| 简体/变体 | 标准形式 | Unicode | 说明 |
|----------|---------|---------|------|
| 后 | 後 | \u540e → \u5f8c | 后来/之后 |
| 于 | 於 | \u4e8e → \u65bc | 介词"于" |
| 与 | 與 | \u4e0e → \u8207 | 参与/给予 |
| 为 | 為 | \u4e3a → \u70ba | 为了/作为 |
| 余 | 馀 | \u4f59 → \u9980 | 余/馀（剩余） |
| 修 | 脩 | \u4fee → \u8129 | 修养/脩脯 |
| 谷 | 穀 | \u8c37 → \u7a40 | 谷物/粮食 |
| 适 | 適 | \u9002 → \u9069 | 适合/前往 |
| 藩 | 籓 | \u85e9 → \u7c53 | 藩篱/籓屏 |
| 褒 | 襃 | \u8912 → \u8943 | 褒扬 |
| 儿 | 兒 | \u513f → \u5152 | 儿子 |
| 闲 | 间 | \u95f2 → \u95f4 | 闲暇/之间 |
| 阸 | 戹 | \u9638 → \u6239 | 险阻 |
| 干 | 幹 | \u5e72 → \u5e79 | 主干 |

**字形变体**：
- 巿 → 市 (\u5dff → \u5e02)

**收录原则**：
1. 在史记全文中**高频出现**（>50次）且语义等价
2. 原文底本和标注文件中用法一致
3. 学术界公认为等价变体（繁简对应、通假字、异体字）

**不纳入规范化的异体字**（会报告为实质差异，需人工多版本互校）：
- 低频异体字（<10次）：如 坑/阬、缻/罋
- 可能有语义差别的字
- 底本版本间有明显分歧的字

## 三、修复流程

### 3.1 典型案例：002_夏本纪

**Lint报告**：

```
=== 002_夏本纪 ===  实质:1  标点:19  编码:2
  ■ 实质差异 1 处：
  [1] 替换
      原文  : '作'
      标注  : '之'
      原文上下文 : …五子作歌，太康失政…
      标注上下文 : …五子之歌，太康失政…
```

**分析**：

1. 查看原始底本 `corpus/archive/chapter/002_夏本纪.txt`（**主参照**）：
   - 第32行（正文）：`作五子之歌`
   - 第37行（太史公曰）：`五子作歌，太康失政` ✓

2. 查看标注文件 `chapter_md/002_夏本纪.tagged.md:195`：
   - `〖{五子之歌〗，〖@太康〗失政。` ✗

3. 参照维基文库版本（辅助验证）：
   - `corpus/shiji/wikisource_shiji/002_夏本紀.html`: 正文为`作五子之歌`，但太史公论中无此句（维基版本太史公论不完整）
   - 确认：原始底本正确

**修复**：

```bash
# 在 chapter_md/002_夏本纪.tagged.md:195 修改
修改前:  〖{五子之歌〗，〖@太康〗失政。
修改后:  〖$五〗子作歌，〖@太康〗失政。
```

**验证**：

```bash
python scripts/lint_text_integrity.py 002
# 输出: ✓ 002_夏本纪: 0处实质差异  49处标点  2处编码
```

### 3.2 修复原则

1. **原始底本为准** — lint校验以 `corpus/archive/chapter/*.txt` 为唯一基准
2. **区分规范化与实质差异** — 理解哪些差异被自动规范化（详见2.4节），哪些需要人工修复
3. **异体字疑惑时多版本互校** — 遇到异体字/通假字疑问时，参考多个校对版本（详见3.5节）
4. **发现底本错误时连底本一起更新** — 若多版本互校确认底本有误，先更新底本，再同步所有派生文件
5. **保留标注** — 修复时尽量保留已有的实体标注，只调整文字部分
6. **全章验证** — 修复后立即运行lint验证该章节
7. **记录校勘** — 底本修改通过白名单机制记录，确保可追溯

### 3.3 规范化vs多版本互校的边界

**自动规范化的异体字**（lint不会报告为实质差异）：
- 特征：史记全文中高频出现（>50次）且语义完全等价
- 示例：于/於、后/後、谷/穀、褒/襃等（详见2.4节完整列表）
- 处理：脚本自动将两侧映射到统一形式，差异消失

**需要人工多版本互校的异体字**（lint报告为实质差异）：
- 特征：低频（<10次）或可能有语义差别
- 示例：坑/阬（可能有"地坑"vs"活埋"之别）、缻/罋等
- 处理：执行3.5节的多版本互校流程，确认后修复

**判断流程**：
```
lint报告实质差异
    ↓
查看2.4节规范化表
    ↓
在表中？ → 否 → 该字差异未被自动规范化
    ↓ 是           ↓
    报告有误？    怀疑异体字？
                   ↓ 是
                执行3.5节多版本互校流程
```

**添加新规范化映射的审批流程**：
1. 统计该字在 `corpus/archive/chapter/` 中的出现次数
2. 对比至少3个版本（维基/四库/繁体）确认用法一致
3. 更新 `scripts/lint_text_integrity.py:51-81` 和 SKILL_01a 2.4节
4. 在git commit中记录决策理由

### 3.4 常见错误类型

| 错误类型 | 示例 | 修复方法 |
|---------|------|---------|
| **擅自改字** | `作`→`之` | 恢复原字，删除多余标注 |
| **擅自加字** | 原文`王冠`→标注`王冠礼` | 删除多余的`礼`字 |
| **擅自删字** | 原文`公子婴`→标注`子婴` | 补回缺失的`公`字 |
| **标点转换** | 原文`。`→标注`，` | 恢复原标点（全角句号） |
| **引号添加** | 原文无引号→标注加引号 | 删除引号（或保留，属于标点规范化） |

### 3.5 异体字/通假字疑惑时的多版本互校流程

**触发条件**：
- lint检查发现字符差异
- 怀疑是异体字、通假字、或底本可能有误
- 例如：坑/阬、於/于、後/后、穀/谷等

**标准流程**：

#### 步骤1: 查看原始底本
```bash
# 查看 corpus/archive/chapter/ 中的原文
grep -n "关键词" corpus/archive/chapter/NNN_篇名.txt
```

#### 步骤2: 对照多个校对版本

按优先级依次参考以下版本（详见 [SKILL_01b](SKILL_01b_多版本互校底本.md)）：

| 版本 | 路径 | 用途 |
|------|------|------|
| **简体底本** | `corpus/archive/chapter/*.txt` | 当前基准 |
| **繁体正文** | `corpus/shiji/史記正文.繁体.txt` | 繁体对照 |
| **维基正文** | `corpus/shiji/wikisource_shiji/*.html` | 权威繁体版 |
| **四库版** | `corpus/shiji/史记四库.txt` | 简体对照 |
| **维基三家注** | `corpus/shiji/wikisource_sanjia/*.html` | 含校勘记 |

**示例：007_项羽本纪 "坑/阬" 疑惑**

```bash
# 1. 查看底本
grep "皆.*之" corpus/archive/chapter/007_项羽本纪.txt
# 结果: 皆阬之 ✓

# 2. 查看其他版本
grep "皆.*之" corpus/archive/chapter_improved/007_项羽本纪.txt    # 阬
grep "皆.*之" corpus/archive/chapter_numbered/007_项羽本纪.txt    # 阬
grep "皆.*之" docs/original_text/007_项羽本纪.txt         # 阬

# 3. 结论: 所有底本版本一致使用"阬"，标注文件误写为"坑"
# 4. 操作: 只修复标注文件，底本正确无需修改
```

#### 步骤3: 判定与处理

| 情况 | 判定 | 操作 |
|------|------|------|
| **所有版本一致** | 底本正确，标注错误 | 只修复标注文件 |
| **多数版本一致** | 底本正确/或底本有误 | 按多数版本修改 |
| **版本分歧严重** | 需要人工判断 | 参考权威工具书，记录争议 |
| **底本明显有误** | 底本错误 | 执行步骤4（更新底本） |

#### 步骤4: 底本有误时的完整更新流程

⚠️ **重要**：若确认底本有误，必须同步更新所有派生文件！

**4.1 更新原始底本**
```bash
# 修改 corpus/archive/chapter/NNN_篇名.txt
sed -i 's/错字/正字/g' corpus/archive/chapter/NNN_篇名.txt
```

**4.2 同步更新派生文件系统**

完整流程详见 [SKILL_01b 步骤3](SKILL_01b_多版本互校底本.md#步骤3-同步更新派生文件系统)。

**4.3 记录校勘到白名单**
```bash
# 在 scripts/lint_whitelist.txt 添加
007	错字	正字	多版本互校：错字→正字（维基/四库一致）
```

**4.4 验证完整性**
```bash
# 运行完整性检查
python scripts/lint_text_integrity.py NNN

# 应该输出 0 处实质差异
```

## 四、白名单机制

### 4.1 作用

已确认的**正确校勘**记录到白名单，lint时自动跳过，避免误报。

**场景举例**：
- 原文底本有误（如`？`缺字），标注时已根据权威版本补正
- 原文有衍文，标注时已剔除

### 4.2 格式

文件路径：`scripts/lint_whitelist.txt`

```
# 格式：章节号<TAB>原文片段<TAB>标注片段<TAB>说明
004	？	历	补缺字：季？→季历
005	。	 	删除衍文句号
006	公	 	删除多余"公"字（原文衍文）
```

### 4.3 使用方法

1. 确认差异是**正确的校勘**（而非标注错误）
2. 在 `scripts/lint_whitelist.txt` 添加一行
3. 再次运行lint，该差异标记为"校勘"

```bash
python scripts/lint_text_integrity.py 004
# 输出: ✓ 004_周本纪: 0处实质差异  41处标点  1处编码  校勘:3
```

## 五、参照资源

### 5.1 版本优先级

| 优先级 | 资源 | 路径 | 说明 |
|-------|------|------|-----|
| 1 | **原始底本（corpus/archive/chapter）** | `corpus/archive/chapter/*.txt` | **主参照**：lint校验基准，标注文件必须与之逐字相同 |
| 2 | **维基文库史记** | `corpus/shiji/wikisource_shiji/*.html` | 繁体，基于中华书局本，校勘质量高，用于疑难处参照 |
| 3 | **其他数字化版本** | ctext.org, 国学网 | 在线查阅，辅助参照 |
| 4 | **中华书局点校本** | 实体书/PDF | 权威终极参照 |

⚠️ **发现底本有误时的处理**：
1. 通过多版本互校确认底本确实有误（详见3.4节）
2. 按SKILL_01b流程同步更新底本及所有派生文件
3. 在白名单中记录此次校勘

### 5.2 查找方法

**示例：查找"五子作歌"在各版本中的原文**

```bash
# 查找原始底本（主参照）
grep "五子" corpus/archive/chapter/002_夏本纪.txt

# 查找维基文库版本（辅助参照）
grep -o "五子[^<]*歌" corpus/shiji/wikisource_shiji/002_夏本紀.html

# 查看完整上下文（-C10 = 上下各10行）
grep -C10 "五子" corpus/shiji/wikisource_shiji/002_夏本紀.html
```

### 5.3 HTML提取技巧

维基文库HTML中，文本内容在 `<p>` 标签中，去除HTML标签后即为纯文本：

```bash
# 提取纯文本（去除HTML标签）
sed 's/<[^>]*>//g' corpus/shiji/wikisource_shiji/002_夏本紀.html | grep "五子"
```

```
检查章节: 130 章
有实质差异: 59 章，共 305 处  ← 需要修复
标点规范化: 21,237 处  ← 可忽略
```

### 6.2 优先级

| 优先级 | 章节范围 | 说明 |
|-------|---------|-----|

4. **记录白名单**：遇到正确校勘时添加到白名单
5. **批量验证**：完成一批后运行全章节lint

## 七、Agent提示词模板

### 7.1 单章修复任务

```
任务：修复《史记》{章节号}_{章节名}.tagged.md的标注完整性问题

背景：
- lint报告显示该章有{N}处实质差异
- 需要确保去除全部标注符号后与原始底本txt逐字相同

要求：
1. 读取lint报告中的差异详情
2. 对比 corpus/archive/chapter/{章节号}_{章节名}.txt 原始底本（**主参照**）
3. 参照 corpus/shiji/wikisource_shiji/{章节号}_{章节名}.html 维基版本（辅助参照）
4. 逐处分析差异，判断是标注错误还是正确校勘
5. 修复标注错误（恢复原始底本字符）
6. 正确校勘添加到 scripts/lint_whitelist.txt
7. 修复后运行 python scripts/lint_text_integrity.py {章节号} 验证

输出：
- 修复说明（每处差异的处理方式）
- 最终lint验证结果（0处实质差异）
```

### 7.2 批量修复任务

```
任务：批量修复《史记》本纪（001-012章）的标注完整性问题

背景：
- 当前12章中有{N}章存在实质差异
- 需要系统性清理，确保本纪全部通过完整性检查

要求：
1. 按章节顺序（001→002→...→012）逐章修复
2. 每章完成后立即验证，再进入下一章
3. 优先修复本纪，再世家，最后列传/书/表
4. 累积白名单条目，减少后续误报
5. 每修复5章生成一次进度报告

输出：
- 每章修复日志（差异数量、修复方法、验证结果）
- 最终汇总报告（剩余待修复章节、白名单新增条目）
```

## 八、与父SKILL的关系

本SKILL（01a标注完整性维护）是 [SKILL_01_古籍校勘](SKILL_01_古籍校勘.md) 的子技能：

```
SKILL_01 古籍校勘 (corpus/archive/chapter/*.txt 原始底本校正)
    ├─ 校勘原则
    ├─ 校勘来源（中华书局本/维基文库...）
    ├─ 常见错误类型
    └─ ↓ 衍生子技能
       SKILL_01a 标注完整性维护 (chapter_md/*.tagged.md 标注后验证)
           ├─ 标注铁律
           ├─ 完整性检查
           ├─ 差异分类
           └─ 修复流程
```

**分工**：
- **SKILL_01**：底本阶段，在 `corpus/archive/chapter/` 修正网络来源txt的错误，建立可信原始底本
- **SKILL_01a**：标注阶段，确保标注过程中没有违反铁律，擅自修改原文
- **说明**：`docs/original_text/` 是供读者查看的原文版本，理论上应与 `corpus/archive/chapter/` 保持一致

## 十、附录：差异类型速查表

| lint输出 | 含义 | 常见原因 | 修复方法 |
|---------|------|---------|---------|
| **替换** | 原文字符被改写 | 标注时误改（作→之） | 恢复原字 |
| **插入（标注多字）** | 标注文件多出字符 | 擅自加字（王冠→王冠礼） | 删除多余字符 |
| **删除（标注少字）** | 标注文件少字符 | 标注时丢失（公子婴→子婴） | 补回缺失字符 |
| **标点规范化** | 全角半角转换 | 历史遗留，可忽略 | 不修复（或统一为全角） |
| **编码规范化** | PUA字符差异 | 原文PUA→标注标准Unicode | 不修复（已规范化） |
