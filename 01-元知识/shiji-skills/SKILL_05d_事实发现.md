# SKILL 05d: 事实发现 — 原子事实三元组抽取

> 将《史记》文本分解为最小可验证单元：每个事实是 1–3 个三元组（主–谓–宾），可独立成立、可独立引用、可独立核验。全书约十万规模。

---

## 一、什么是事实

### 1.1 定义

**事实（Fact）** = 一个原子陈述，用 1–3 个三元组（Subject–Predicate–Object）完整表达，来源于《史记》单句或单个从句。

核心特征：
- **原子性**：不可再拆分为更小的有意义单元
- **可引用性**：有明确的来源句，可回溯原文
- **可核验性**：可独立判断真/假/存疑
- **无叙事依赖**：不需要上下文就能理解其含义

### 1.2 与其他层的区别

| 层级 | 颗粒度 | 示例 | 数量级 |
|------|--------|------|--------|
| **实体** | 命名对象 | 〖@秦始皇〗、〖=咸阳〗 | 15,331（18类名词 + 4类动词） |
| **事实** | 原子陈述（1–3个三元组） | 秦始皇于前210年死亡 | **~100,000** |
| **事件** | 叙事单元（多参与者+上下文） | 沙丘之变（政变过程） | 3,185 |
| **关系** | 持久性实体间结构 | 秦始皇↔胡亥（父子） | ~10,000 |

**事实 vs 事件**：
事件是有叙事结构的复合单元（有起因、经过、影响），包含多个事实。
"沙丘之变"是事件；"秦始皇崩于沙丘"、"赵高矫诏"、"扶苏自杀"是三个独立事实。

**事实 vs 关系**：
关系是无时间戳的持久结构（父子、君臣）；事实是有时态的原子陈述（"前210年封X为Y"）。
"秦始皇是胡亥之父"是关系；"秦始皇于前247年即位"是事实。

**事实 vs 姓氏推理（07b）**：
姓氏是事实的一个子类——人物属性事实。

### 1.3 典型示例

| 原文 | 事实 | 三元组 |
|------|------|--------|
| 武王崩 | 周武王死亡 | (周武王, 死亡, ?) |
| 孝公封商鞅商地十五邑 | 秦孝公封商鞅商地 | (秦孝公, 封赏, 商鞅), (商鞅, 封地, 商地十五邑) |
| 赵括代廉颇将 | 赵括接替廉颇为将 | (赵括, 替代, 廉颇), (赵括, 担任, 赵国将领) |
| 坑杀降卒四十余万 | 长平降卒被杀数量 | (白起, 坑杀, 赵降卒), (赵降卒数量, =, 40万) |
| 屈原者，名平，楚之同姓也 | 屈原之名与姓族 | (屈原, 名, 平), (屈原, 姓族, 芈姓楚国) |

## 二、设计原则

**SPO 全用汉字，不设 schema。**

- 主语（S）、谓语（P）、宾语（O）均直接用原文汉字表达
- 不定义谓词枚举、不定义类型代码
- 同一类事实自然会收敛到相似的谓语（"崩"/"薨"/"卒"/"立"/"封"等），但这是归纳出来的，不是预先规定的
- 特殊情况：O 可为 null（无宾语的动词，如"崩"）；多个三元组共享同一来源时，作为同一事实的多条记录

这样做的好处：
- 不需要维护谓词本体；
- 《史记》语言本身就是最好的谓语来源；
- 后期可以从数据中归纳谓语聚类，而不是从设计中强加分类。

**⚠️ 重要规范**：详细的事实抽取规则已记录在 [`skills/references/SKILL_05d1_rules.md`](references/SKILL_05d1_rules.md)，包括：

- **人名规范**：❌ 不规范化（保持原文"黄帝"/"轩辕"并存）
- **谓语归一化**：❌ 不归一化（"崩"/"薨"/"卒"各自保留）
- **抽象描述**：❌ 不拆分（"静渊以有谋"作为整体谓语）
- **复合动作**：✅ 分别记录（"举A、B、C、D"拆分为4条事实）

执行事实抽取时，必须遵循该规范文档。

## 三、数据格式与生成流程

### 3.1 工作流程

**两步法：Markdown表格 → JSON数据**

```
第一步：生成 Markdown 表格
  输入：chapter_md/{章节}.tagged.md（标注文件）
  输出：kg/facts/markdown/{章节}_事实表格.md
  格式：人类可读的表格，易于校对和修正

第二步：转换为 JSON 数据
  输入：kg/facts/markdown/{章节}_事实表格.md
  输出：kg/facts/data/{章节}_事实索引.json
  格式：程序可读的结构化数据，用于知识图谱构建
```

**为什么先生成Markdown表格？**
- 易于人工校对：表格形式一目了然，便于发现遗漏或错误
- 易于修正：直接编辑表格单元格，无需修改JSON语法
- 版本控制友好：Git diff可清晰展示变更
- 最终转换为JSON：一次转换，批量生成标准数据

**⚠️ 数据来源重要说明**：

**输入源文件**：
- **必须使用** `chapter_md/{章节}.tagged.md`（语义标注文件）
- ❌ **不使用** `corpus/shiji/史记.简体.txt`（原始纯文本）

**原因**：
1. 标注文件已完成实体识别（`〖@人名〗`、`〖=地名〗`等），便于抽取S和O
2. 段落编号 `[N]` 已清晰标注，便于建立PN（段落编号）映射
3. 语义标注提供类型信息，提高抽取准确性

**原文字段处理**：
- 事实表格中的"原文"列必须**去除语义标注符号**
- 使用脚本：`scripts/semantic_tags.py` 中的 `remove_semantic_tags()` 函数
- 示例：`〖@黄帝〗者，〖@少典〗之子` → `黄帝者，少典之子`

**段落编号（PN）映射**：
- 每条事实必须记录其来源段落编号（PN）
- PN编号对应标注文件中的 `[N]` 或 `[N.M]` 段落标记
- 格式：`001-1` 表示001章第1段，`001-1.1` 表示第1段的第1个子段落
- 用途：支持按段落查询事实、追溯事实来源、统计段落密度

### 3.2 Markdown表格格式

**文件路径**：`kg/facts/markdown/{章节}_事实表格.md`

**表格结构**：

```markdown
# 001_五帝本纪_事实索引

| ID | PN | 主语(S) | 谓语(P) | 宾语(O) | 时间 | 地点 | 其他背景 | 原文 | 置信度 | 关联事件 |
|----|-------|---------|---------|---------|------|------|----------|------|--------|----------|
| 001-f0001 | 001-1 | 黄帝 | 姓 | 公孙 | | | | 黄帝者，少典之子，姓公孙 | high | |
| 001-f0002 | 001-1 | 黄帝 | 名 | 曰轩辕 | | | | 黄帝者，少典之子，姓公孙，名曰轩辕 | high | |
| 001-f0003 | 001-1.1 | 神农氏 | 世衰 | | | | 轩辕之时 | 轩辕之时，神农氏世衰 | high | |
| 001-f0004 | 001-1.1 | 轩辕 | 修德振兵 | | | | | 轩辕乃修德振兵 | high | |
| 001-f0005 | 001-4.1 | 黄帝 | 居 | 轩辕之丘 | | | | 黄帝居轩辕之丘 | high | |

**合计事实数**：5条（示例）
```

**字段说明**：

| 列名 | 说明 | 必填 | 示例 |
|------|------|------|------|
| ID | `{章节}-f{序号}`，4位补零 | 是 | `001-f0001` |
| PN | 段落编号 `{章节}-{段号}`，对应标注文件中的 `[N]` 或 `[N.M]` | 是 | `001-1`, `001-1.1` |
| 主语(S) | 规范实体名，汉字 | 是 | `黄帝` |
| 谓语(P) | 动词或属性谓语，汉字 | 是 | `姓`、`崩`、`封` |
| 宾语(O) | 受事或属性值，汉字或空 | 否 | `公孙`、`轩辕之丘`、（空） |
| 时间 | 公元纪年或空 | 否 | `-2697`、`前210年`、（空） |
| 地点 | 地点名或空 | 否 | `轩辕之丘`、`咸阳`、（空） |
| 其他背景 | 场合/条件等，汉字或空 | 否 | `因以为名`、`救赵攻秦`、（空） |
| 原文 | 来源句子（已去除标注符号），≤50字 | 是 | `黄帝者，少典之子，姓公孙` |
| 置信度 | exact/high/medium/low | 是 | `high` |
| 关联事件 | 事件ID或空 | 否 | `001-e001`、（空） |

**注意事项**：
- 每行代表一个三元组，如果一个原文句对应多个事实，则分多行记录
- 空单元格用（空）表示，便于与未填写区分
- 时间列可记录原文形式（如"前210年"），转JSON时统一为公元纪年数字
- 表格末尾添加合计行统计事实总数

### 3.3 JSON数据格式

**文件路径**：`kg/facts/data/{章节}_事实索引.json`

**单条事实示例**：

```json
{
  "id": "001-f0001",
  "pn": "001-1",
  "s": "黄帝",
  "p": "姓",
  "o": "公孙",
  "ctx": {
    "time": null,
    "place": null
  },
  "src": "001_五帝本纪",
  "text": "黄帝者，少典之子，姓公孙",
  "conf": "high",
  "event": null
}
```

**更多示例**：

```json
{ "id": "001-f0003", "pn": "001-1.1", "s": "神农氏", "p": "世衰", "o": null,
  "ctx": { "time": null, "place": null, "occasion": "轩辕之时" },
  "src": "001_五帝本纪", "text": "轩辕之时，神农氏世衰" }

{ "id": "005-f0011", "pn": "005-23", "s": "秦孝公", "p": "封",   "o": "商鞅",
  "ctx": { "time": -340, "place": null },
  "src": "005_秦本纪", "text": "孝公封商鞅商地十五邑。" }

{ "id": "073-f0041", "pn": "073-81", "s": "白起",   "p": "坑杀", "o": "赵降卒四十余万",
  "ctx": { "time": -260, "place": "长平" },
  "src": "073_白起王翦列传", "text": "坑杀降卒四十余万。" }

{ "id": "007-f0088", "pn": "007-42.3", "s": "项羽",   "p": "破釜沉舟", "o": null,
  "ctx": { "time": -207, "place": "漳河南", "occasion": "救赵攻秦" },
  "src": "007_项羽本纪", "text": "项羽乃悉引兵渡河，皆沉船，破釜甑。" }
```

**字段说明**：

| 字段 | 说明 | 必填 |
|------|------|------|
| `id` | `{章节}-f{序号}`，全库唯一 | 是 |
| `pn` | 段落编号 `{章节}-{段号}`，对应标注文件中的 `[N]` 或 `[N.M]` | 是 |
| `s` | 主语，汉字规范名 | 是 |
| `p` | 谓语，直接用汉字（崩/封/名/坑杀…） | 是 |
| `o` | 宾语，汉字或 null | 否 |
| `ctx` | 事实成立的背景条件（时间/地点/场合等），可为 `{}` | 是 |
| `ctx.time` | 公元纪年（负数=公元前），整数或 null | 推荐 |
| `ctx.place` | 发生地点，汉字或 null | 推荐 |
| `ctx.*` | 其他背景键值（occasion等），汉字或 null | 可选 |
| `src` | 来源章节 | 是 |
| `text` | 原始句子（已去除标注符号，≤50字） | 是 |
| `conf` | exact / high / medium / low | 是 |
| `event` | 关联事件 ID 或 null | 否 |

**ctx 扩展键值规则**：
- 可自由添加任意汉字键值（occasion、reign、role等）
- 值必须是字符串或 null，不使用数字（时间除外）
- 不预设字段枚举，从数据中自然归纳

```
- 事实粒度是否合适（1-3个三元组）？
```

**阶段一：规则抽取**（待试点成功后展开）

```
高信号动词句式：立/薨/封/拜/以X为Y/名X/字X 等
数字+量词正则：N万/N人/N年/N里
P 直接取原文动词，不映射到枚举
→ 覆盖率约40–60%，精度高
```

**阶段二：句法框架抽取**

```
施事-动词-受事 框架
"以X为Y"结构（任命模式）
"X之Y"结构（归属/属性模式）
→ 覆盖更多隐式事实，精度中
```

**阶段三：LLM批量抽取**

```
输入：段落文本 + 已知实体列表
输出：事实三元组 + 置信度 + 原文依据
质量控制：与阶段一/二结果交叉验证
→ 覆盖语义复杂案例，精度取决于模型
```

**阶段四：验证与去重**

```
同一事实在多章出现 → 保留最高置信度，记录多源
与事件索引、关系索引交叉核验
数字类事实：与 SKILL_07 矛盾检测联动
```
