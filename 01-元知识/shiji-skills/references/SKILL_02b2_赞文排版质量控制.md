# SKILL 02b2: 赞文排版质量控制

> 从《史记》125篇赞文的排版规范化实践中提炼，适用于四言韵文的格式检查与修复。
>
> **前置**: 赞文已用 `::: 赞` 标记，详见 [SKILL_02b_区块与韵文处理](../SKILL_02b_区块与韵文处理.md)。

---

## 一、问题定义

### 1.1 赞文特征

赞是《史记》每篇末尾的四言韵文总结，典型特征：
- **句式整齐**：以四字为一个语义单元
- **对仗工整**：上下句结构对称
- **押韵**：偶数句末字韵脚相同

### 1.2 排版规范

**标准格式**：每行两个四字句（8字+标点）

```markdown
::: 赞
[N] 韩襄遗孽，始从汉中。
剖符南面，徙邑北通。
穨当归国，龙雒有功。
:::
```

**核心要求**：
1. **每行8字**：去除标注符号和Purple Number后，每行恰好8个汉字
2. **全角标点**：使用全角逗号 `，` 和句号 `。`，禁止半角 `,` `.`
3. **硬换行**：每行独立，不得将多句连写
4. **对偶分行**：每行包含一组对偶句（上句+下句）

### 1.3 为什么需要严格排版

1. **HTML渲染需求**：赞文块使用 `white-space: pre-line`，保持硬换行，每行一个 `<br>`
2. **视觉对称性**：对偶句在同一行，便于阅读和理解
3. **数据提取**：韵文提取脚本依赖标准格式
4. **质量保证**：统一格式便于检查和维护

## 二、常见格式问题

### 2.1 连写问题（最常见）

❌ **错误**：多句连写在一行
```markdown
[N] 四字句A，四字句B；四字句C，四字句D。
```

✅ **正确**：每行两个四字句
```markdown
[N] 四字句A，四字句B。
四字句C，四字句D。
```

### 2.2 半角标点问题

❌ **错误**：使用半角逗号、分号
```markdown
[N] 韩襄遗孽,始从汉中;
```

✅ **正确**：使用全角标点
```markdown
[N] 韩襄遗孽，始从汉中。
```

### 2.3 句长不符问题

❌ **错误**：三字句、五字句混排
```markdown
[N] 三字句，四字句甲。
四字句乙，五字句丙丁。
```

✅ **正确**：统一四字句（特殊情况除外）
```markdown
[N] 某某三字，四字句甲。
四字句乙，五字之句。
```

**注意**：《史记》赞文偶有三字句或五字句（如"於戏"、"吾適丰沛"），这是合理的变体，不强制修改。

### 2.4 标注符号干扰

标注符号会影响句长计算：

**原文**：
```markdown
[N] 〖@颇〗、〖@牧〗不用，〖;王迁〗囚虏。
```

**去标注后应为**：
```
颇、牧不用，王迁囚虏。  (8字✓)
```

**检查方法**：lint脚本会自动移除标注符号后再计算字数。

## 三、检查工具

### 3.1 lint_zan_format.py

**功能**：检查赞文是否符合"每行8字"的规范

**检查项**：
1. 每行去除标注符号和Purple Number后，汉字数是否为8
2. 是否使用半角标点
3. 是否有消歧语法处理错误

**用法**：
```bash
# 检查所有章节的赞文格式
python scripts/lint_zan_format.py

# 输出示例：
# 📄 043_赵世家.tagged.md:
#   ❌ 赞文第1段第3行: 应为8字,实际16字
#      原文: [21] 四字句A，四字句B；四字句C，四字句D。
#      去标注后: 四字句A，四字句B；四字句C，四字句D。
#      汉字: 四字句A四字句B四字句C四字句D
```

**核心逻辑**：
```python
# 1. 移除Purple Number
text_no_pn = re.sub(r'\[\d+\]\s*', '', line)

# 2. 处理消歧语法 〖@成|周成王〗 → 成
text_no_disambig = re.sub(r'\|[^〗⟧]+', '', text_no_pn)

# 3. 移除标注符号 〖@少典〗 → 少典
text_no_tags = re.sub(r'[〖⟦][^〗⟧\u4e00-\u9fff]+', '', text_no_disambig)
text_no_tags = re.sub(r'[〗⟧]', '', text_no_tags)

# 4. 只保留汉字
hanzi = re.sub(r'[^\u4e00-\u9fff]', '', text_no_tags)

# 5. 检查长度
if len(hanzi) != 8:
    报告错误
```

### 3.2 lint_zan_in_chapters.py

**功能**：检查赞文是否有多句连写问题

**检查项**：
1. 一行中是否包含多个句号（`>1` 个 `。！？`）
2. 是否有连续四字句未分行（如 `四字，四字；四字，四字。`）
3. 一行汉字数是否 `>=16`（4句或以上）

**用法**：
```bash
# 检查所有章节
python scripts/lint_zan_in_chapters.py

# 输出示例：
# 📄 051_荆燕世家.tagged.md:
#   ⚠️  第1段第2行: 包含3个句子,可能需要分行
#      行: [N] 四字句A。四字句B。四字句C。
#   ⚠️  第1段第5行: 连续四字句(4句)未分行
#      行: [N] 四字句A，四字句B；四字句C，四字句D。
```

**核心逻辑**：
```python
# 1. 检查一行中的句号数量
full_stops = line_clean.count('。') + line_clean.count('！') + line_clean.count('？')
if full_stops > 1:
    警告: 可能需要分行

# 2. 检查连续四字句模式
hanzi = re.sub(r'[^\u4e00-\u9fff]', '', line_clean)
if len(hanzi) >= 16:
    pattern_4char = r'[\u4e00-\u9fff]{4}[，、；]'
    matches_4char = re.findall(pattern_4char, line_clean)
    if len(matches_4char) >= 2:
        警告: 连续四字句未分行
```

## 四、修复流程

### 4.1 标准修复步骤

1. **运行检查**：
   ```bash
   python scripts/lint_zan_format.py
   python scripts/lint_zan_in_chapters.py
   ```

2. **定位问题文件**：
   ```bash
   # 示例输出：
   # 📄 009_吕太后本纪.tagged.md
   # 📄 027_天官书.tagged.md
   # 📄 030_平准书.tagged.md
   ```

3. **手动修复**（推荐）：
   - 打开问题文件
   - 定位到 `::: 赞` 块
   - 按照"每行8字"规范拆分或合并
   - 替换半角标点为全角标点

4. **验证修复**：
   ```bash
   python scripts/lint_zan_format.py
   # 预期输出: ✅ 所有赞文格式检查通过
   ```

5. **⚠️ 重要：更新韵文专项索引**：

   赞文修复后，**必须重新生成韵文数据和HTML**，否则 `docs/special/yunwen.html` 展示的仍是旧格式。

   ```bash
   # 重新提取韵文（更新JSON/MD数据）
   python scripts/extract_yunwen.py

   # 重新生成HTML和PDF
   python scripts/render_yunwen_html.py

   # 验证生成的文件
   ls -lh docs/special/yunwen.*
   # 应显示: yunwen.html, yunwen.json, yunwen.md, yunwen.pdf
   ```

   **检查结果**：
   - 在浏览器中打开 `docs/special/yunwen.html`
   - 验证赞文的硬换行是否正确
   - 确认修复的章节已更新

### 4.2 修复示例

#### 示例1：连写拆分

**修复前**（009_吕太后本纪.tagged.md）：
```markdown
::: 赞
[36] 孝惠皇帝、高后之时，黎民得离战国之苦，君臣俱欲休息乎无为，故惠帝垂拱，高后女主称制，政不出房户，天下晏然。刑罚罕用，罪人是希。民务稼穑，衣食滋殖。
:::
```

**问题分析**：
- 这是散文，不是四言韵文
- 应该不用 `::: 赞` 标记，而是直接放在 `## 太史公论吕太后` 下

**修复后**：
```markdown
## 太史公论吕太后

[36] 孝惠皇帝、高后之时，黎民得离战国之苦，君臣俱欲休息乎无为，故惠帝垂拱，高后女主称制，政不出房户，天下晏然。刑罚罕用，罪人是希。民务稼穑，衣食滋殖。
```

#### 示例2：标点修复

**修复前**（027_天官书.tagged.md）：
```markdown
[109] 汉之兴，五星聚于东井;
平城之围，月晕参毕;
```

**问题分析**：
- 使用了半角分号 `;`
- 应改为全角逗号或句号

**修复后**：
```markdown
[109] 汉之兴，五星聚于东井。
平城之围，月晕参毕。
```

#### 示例3：连续四字句拆分

**修复前**（假设）：
```markdown

```

**修复后**：
```markdown
[N] 韩襄遗孽，始从汉中。
剖符南面，徙邑北通。
```

## 五、特殊情况处理

### 5.1 三字句、五字句

**规则**：《史记》赞文以四字句为主，但偶有变体

**处理方式**：
- **保留原文**：如"於戏"（叹词）、"吾適丰沛"（五字句）
- **不强制修改**：只要总字数合理（如3+5=8）

**示例**（097_郦生陆贾列传）：
```markdown
[37] 於戏，天下之辩士！
```
- "於戏"是2字叹词，后接4字句，总共6字
- 这是历史原文，保留即可

### 5.2 散文误标为赞

**问题**：某些章节的"太史公曰"内容是散文，但被错误标记为 `::: 赞`

**示例**（009_吕太后本纪）：
```markdown
::: 赞
[36] 孝惠皇帝、高后之时，黎民得离战国之苦，...（长篇散文）
:::
```

**修复方式**：
1. 移除 `::: 赞` 标记
2. 将内容直接放在 `## 太史公论XXX` 标题下

### 5.3 褚先生补注

**问题**：褚先生补注部分可能包含韵文，但格式不规范

**处理方式**：
- 单独标注为 `::: 褚先生补` 或直接放在正文
- 不强制执行"每行8字"规范

## 七、工作经验

### 7.1 核心原则

1. **每行8字是铁律**：去除标注符号和Purple Number后，每行必须是8个汉字（偶有6字、10字的变体可接受）
2. **全角标点是规范**：绝对禁止半角标点（`,` `;` `:` 等）
3. **硬换行是需求**：HTML渲染依赖 `<br>` 分行，不能连写
4. **对偶是美学**：上下句在同一行，视觉对称

### 7.2 检查技巧

1. **标注符号清洗**：检查前必须移除所有 `〖...〗` 和 `⟦...⟧`，否则字数计算错误
2. **消歧语法处理**：`〖@成|周成王〗` 显示为"成"，只计1字
3. **Purple Number忽略**：`[N]` 不计入字数
4. **标点不计数**：只计汉字，标点符号不算

### 7.3 修复技巧

1. **分句标记**：先在连写句中用 `/` 标记分句点，再拆行
   ```
   修复前: 四字句A，四字句B；四字句C，四字句D。
   标记:   四字句A，四字句B；/四字句C，四字句D。
   修复后: 四字句A，四字句B。
          四字句C，四字句D。
   ```

2. **标点统一**：用编辑器的"查找替换"批量替换半角标点
   ```
   查找: , (半角逗号)
   替换: ， (全角逗号)
   ```

3. **对称检查**：修复后检查每行的上下句是否对称
   ```
   韩襄遗孽，始从汉中。  ← 上句4字，下句4字 ✓
   剖符南面，徙邑北通。  ← 上句4字，下句4字 ✓
   ```

### 7.4 常见错误

1. **错误删除标点**：
   - ❌ `四字句A四字句B` （缺少逗号）
   - ✅ `四字句A，四字句B。`

2. **过度拆分**：
   - ❌ 每句单独一行（4字/行）
   - ✅ 每行两个四字句（8字/行）

3. **标注符号未清洗**：
   - ❌ `〖@颇〗、〖@牧〗不用，〖;王迁〗囚虏。` 计为20字
   - ✅ 去标注后 `颇、牧不用，王迁囚虏。` 计为8字

## 十、FAQ

### Q1: 为什么必须每行8字？

**A**:
1. **视觉对称**：对偶句在同一行，便于阅读
2. **HTML渲染**：`white-space: pre-line` 保持硬换行，每行一个 `<br>`
3. **数据提取**：韵文提取脚本依赖标准格式
4. **历史传统**：赞文就是四言韵文，每联（上下句）为一行是传统格式

### Q2: 三字句、五字句怎么处理？

**A**:
- **保留原文**：《史记》赞文偶有变体（如"於戏"、"吾適丰沛"）
- **不强制修改**：只要总字数合理（如3+5=8，4+6=10）
- **人工判断**：如果是历史原文，保留；如果是标注错误，修复

### Q3: 散文被误标为赞怎么办？

**A**:
1. 移除 `::: 赞` 标记
2. 将内容直接放在 `## 太史公论XXX` 标题下
3. 更新韵文提取数据

### Q4: 修复后韵文数量变化了怎么办？

**A**:
- **数量减少**：可能是移除了误标的散文，检查是否合理
- **数量增加**：可能是新发现的韵文，检查是否真的是赞
- **验证方法**：对比 `data/yunwen.json` 修复前后的差异

### Q5: 检查脚本报错怎么办？

**A**:
1. **字数不对**：检查是否有标注符号未清洗、消歧语法错误
2. **半角标点**：全局替换半角标点为全角
3. **连写问题**：按照"每行8字"规范拆分
