## 一、核心问题与基础统计

### 1.1 核心挑战

史记拼音标注面临的核心挑战是**上下文相关的多音字**，这类字的读音取决于：
- 词性（动词vs名词）
- 语境（官职vs日常）
- 搭配（固定词组）

仅靠字典无法确定正确读音，需要结合上下文判断。

### 1.2 史记文本统计

**总体情况**：
- **史记总字数**: 510,690 字
- **不重复汉字数**: 4,722 个
- **常见多音字数量**: 63 个
- **多音字出现总次数**: 43,045 次
- **多音字占总字数比例**: 8.43%
- **多音字占不重复汉字比例**: 1.33%

**频率分布**：

| 频率等级 | 出现次数范围 | 数量 | 占比 |
|---------|-------------|------|------|
| 极高频 | >2000次 | 4个 | 6.3% |
| 超高频 | 1000-2000次 | 7个 | 11.1% |
| 高频 | 500-1000次 | 7个 | 11.1% |
| 中高频 | 200-500次 | 23个 | 36.5% |
| 中频 | 100-200次 | 8个 | 12.7% |

**核心发现**：
- 前18个高频多音字（≥500次）仅占多音字总数的28.6%
- 但这18个字的累计出现次数占所有多音字的约**70%**
- **优先处理这18个高频多音字，可以解决绝大部分拼音标注问题**

## 二、典型案例：骑（qí vs jì）

### 问题发现
用户反馈："兒能骑羊"中的"骑"应该读**qí**，而不是jì

### 分析结果
"骑"字在史记中有三种用法：
1. **动词**（骑马、骑羊）→ qí（阳平）
2. **名词**（骑兵、官职）→ jì（去声）
3. **量词**（十骑、百骑）→ jì（去声）

### 解决方案
- ❌ 删除错误的单字条目"骑: jì"
- ✅ 添加15个固定词组（骑都尉、车骑将军、胡骑等）
- ⚠️ 量词用法（十骑、百骑）暂未全覆盖（30+种变化）

### 相关文档
详细案例分析见原 `doc/spec/ANALYSIS_骑字多音规则.md`

## 四、处理策略

### 策略1：穷举固定词组（当前方案）

**适用范围**：
- 官职名（丞相、将军、骑都尉）
- 固定搭配（列传、游说、间谍）
- 专有名词（地名、人名）

**实施方法**：
1. 统计关键词组频率
2. 添加到special-pronunciation.json
3. 按词长排序（长词优先匹配）

**优点**：
- 精确可控
- HTML生成时已确定
- 无性能问题

**缺点**：
- 需要逐个添加
- 量词用法难以穷举

### 策略2：正则模式匹配（未来增强）

**适用范围**：
- 有规律的量词（十骑、百骑、万骑）
- 有规律的数词组合

**技术方案**：
```javascript
// 在 matchSpecialPronunciation() 中添加
function matchPattern(text, startIndex) {
  // 匹配：数词 + [余/馀] + 骑
  const pattern = /^([0-9一二三四五六七八九十百千万]+[余馀]?)骑/;
  const substr = text.substring(startIndex);
  const match = substr.match(pattern);

  if (match) {
    const numText = match[1];
    const numPinyin = pinyinFn(numText, { type: 'array' });
    return {
      text: match[0],
      pinyin: [...numPinyin, 'jì'],
      length: match[0].length
    };
  }
  return null;
}
```

**优点**：
- 一条规则覆盖所有变化
- 减少词表维护成本

**缺点**：
- 需要修改JavaScript代码
- 需要性能测试
- 增加复杂度

### 策略3：AI上下文判断（远期方向）

**构想**：
- 使用简单的词性标注模型
- 基于前后词判断读音
- 集成到前端或后端

**挑战**：
- 需要训练数据
- 性能开销较大
- 维护成本高

|------|---------|------|------|-----------|
| A - 全词组标注 | 官职、地名、人名 | 100%准确 | 需逐个添加 | 高（~40%词条） |
| B - 高频词组 | 常见固定搭配 | 准确率高 | 维护成本中等 | 高（~50%词条） |
| C - 单字标注 | 词性明确单字 | 简洁高效 | 易误判 | 低（~10%词条） |
| D - 依赖默认 | 低频或默认准确 | 无维护成本 | 可能误读 | 中（~40%多音字） |
| E - 正则匹配 | 有规律模式 | 一劳永逸 | 技术复杂 | 未实施 |
| F - AI判断 | 全自动 | 理论最优 | 成本极高 | 研究中 |
