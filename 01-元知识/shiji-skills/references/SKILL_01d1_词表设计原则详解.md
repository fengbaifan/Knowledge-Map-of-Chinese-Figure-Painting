# SKILL_01d1 词表设计原则详解

> **参考文档**：详细的特殊读音词表设计原则、版本演化历史、典型案例分析

---

## 一、核心设计原则（v3.0）

### 1.1 最小化原则

**核心理念**：只收录pinyin-pro无法正确处理的词条。

**实施要点**：
- 能组词就组词，避免单字词条
- 定期Lint检查，清理冗余词条
- 每次添加词条前必须测试pinyin-pro输出

**错误示例**（v3.0前的问题）：
```json
// ❌ 错误：添加了pinyin-pro能正确处理的词条
{
  "text": "立为王",
  "pinyin": ["lì", "wéi", "wáng"],
  "note": "立为王"
}

// 测试结果：pinyin-pro输出 ['lì', 'wèi', 'wáng'] ✓ 正确
// 结论：不应添加此词条
```

### 1.2 固定词组优先

**原则**：优先使用2-4字固定词组，而非单字词条。

**优点**：
- **精准控制**：只在特定词组中生效
- **避免误伤**：不影响其他常用词
- **易于维护**：语义明确，不易混淆

**对比示例**：

| 方案 | 词条 | 优点 | 缺点 |
|------|------|------|------|
| ✅ 固定词组 | `车骑将军` | 精准匹配官职 | 需要列举多个词组 |
| ❌ 单字词条 | `骑` | 覆盖面广 | 误伤"骑马"等常用词 |

**实际案例**：

```json
// ✅ 推荐：官职固定词组
{
  "text": "车骑将军",
  "pinyin": ["chē", "jì", "jiàng", "jūn"],
  "context": "官职",
  "note": "汉代将军号，'骑'读jì（去声）"
},
{
  "text": "骠骑将军",
  "pinyin": ["piào", "jì", "jiàng", "jūn"],
  "context": "官职",
  "note": "汉代高级将军号"
}

// ❌ 避免：单字词条（会误伤"骑马"）
{
  "text": "骑",
  "pinyin": ["jì"]
}
```

### 1.3 查证权威来源

**权威来源优先级**：

1. **Wiktionary** (最高优先级)
   - URL: https://en.wiktionary.org/wiki/
   - 提供词性、读音、例句
   - 必须查证所有不确定的读音

2. **《史记正义》音注**
   - 唐代张守节权威标注
   - 适用于古地名、人名
   - 需要反切转换

3. **原文语境**
   - 结合《史记》上下文判断
   - 区分动词/名词/官职/地名

**案例："称王"的正确读音**

❌ **错误做法**（主观判断）：
```
思路："称王"是"宣布自己为王"，"王"是动词，应读wàng
词条：{"text": "称王", "pinyin": ["chēng", "wàng"]}  // 错误！
```

✅ **正确做法**（查证Wiktionary）：
```
1. 查询 https://en.wiktionary.org/wiki/稱王
2. 发现官方标注：|m=chēngwáng|cat=v
3. 含义：to proclaim oneself king
4. 测试：pinyin-pro输出 ['chēng', 'wáng'] ✓ 正确
5. 结论：不添加此词条
```

**教训**（2026-04-04）：
- 初次标注时主观判断"称王"中的"王"读wàng
- 用户指正后查Wiktionary，发现实际读wáng
- 这个错误导致了词表v3.11→v3.12的修正

## 二、单字词条例外情况

### 2.1 情况1：修正默认库错误

**适用场景**：pinyin-pro的默认拼音库错误标注了某个字。

**判断标准**：
- pinyin-pro输出错误
- 无法通过固定词组规避
- 该字在《史记》中高频出现（≥100次）

**案例1："於"的读音**

```json
{
  "text": "於",
  "pinyin": ["yú"],
  "context": "介词",
  "note": "介词，相当于'在'、'到'、'对于'，读yú（阳平），不读wū。此为例外单字词条，因默认拼音库错误标注为wū"
}
```

**验证过程**：
```javascript
// pinyin-pro默认输出
pinyin('於') → 'wū'  // ❌ 错误

// 《史记》中的实际用法
"於是" → yú shì（于是）
"至於" → zhì yú（至于）

// 结论：必须添加单字词条修正
```

**案例2："燕"的读音**

```json
{
  "text": "燕",
  "pinyin": ["yān"],
  "context": "国名",
  "note": "燕国，周代诸侯国名，读yān（阴平）。在史记中99.1%为国名用法。此为例外单字词条，修正默认拼音库"
}
```

### 2.2 情况2：低风险姓氏单字

**适用场景**：
- 姓氏读音特殊
- 该姓氏在《史记》中只作姓氏用
- 不会误伤其他常用词

**判断标准**：
- 字频分析显示该字99%作为姓氏
- 其他读音的使用场景极少（<1%）
- 添加单字词条不影响其他常用词

**案例："缪"姓**

```json
{
  "text": "缪",
  "pinyin": ["miào"],
  "context": "姓氏",
  "note": "姓缪读miào，不读móu"
}
```

**风险分析**：
```
字频统计（《史记》）：
- 缪姓：15次（100%）
- 其他用法：0次
- 风险评估：✓ 低风险，可添加单字词条
```

**对比："王"不能添加单字词条**

```
字频统计（《史记》）：
- 王（国王，名词wáng）：8170次（99.5%）
- 王（称王，动词wàng）：39次（0.5%）
- 风险评估：❌ 高风险，单字词条会误伤99.5%的用法
- 解决方案：只添加"王天下"、"欲王"等动词词组
```

## 三、禁止使用的方法

### 3.1 常见字单字词条

**为什么禁止**：
- 高频字有多种用法，单字词条会误伤大部分情况
- 违反"最小化原则"
- 难以维护

**典型错误案例**：

```json
// ❌ 绝对禁止：行
{
  "text": "行",
  "pinyin": ["háng"]
}

// 后果：
"银行" → yín háng（❌ 错误，应为yín háng）
"行为" → háng wéi（❌ 错误，应为xíng wéi）
"五行" → wǔ háng（✓ 正确）

// ✅ 正确方案：使用固定词组
{
  "text": "五行",
  "pinyin": ["wǔ", "xíng"],
  "context": "名词",
  "note": "五行学说，'行'读xíng（阳平）"
}
```

### 3.2 上下文相关的模糊标注

**为什么禁止**：
- 无法通过简单匹配实现
- 需要语义分析，超出词表能力
- 容易引入错误

**错误示例**：

```json
// ❌ 禁止：依赖上下文的标注
{
  "text": "为",
  "pinyin": ["wéi"],
  "context": "被动句中",  // ← 无法自动判断
  "note": "在被动句中读wéi"
}

// ✅ 正确方案：固定词组
{
  "text": "是为",
  "pinyin": ["shì", "wéi"],
  "context": "系动词",
  "note": "这就是，'为'读wéi（阳平）"
}
```

## 四、版本演化历史

### v1.0 (2026-03-初)
- **特点**：手工整理，无明确规范
- **问题**：大量单字词条，误伤率高
- **词条数**：约50条

### v2.0 (2026-03-中)
- **改进**：引入《史记正义》提取
- **新增**：反切转拼音工具
- **问题**：未建立Lint机制，冗余词条堆积
- **词条数**：约200条

### v3.0 (2026-03-下)
- **重大重构**：建立三大设计原则
  1. 最小化原则
  2. 固定词组优先
  3. 查证权威来源
- **新增**：单字词条例外规则
- **清理**：删除大量冗余词条
- **词条数**：200 → 52条

### v3.10 (2026-04-04)
- **重大突破**：建立Lint自动化机制
- **工具**：`scripts/lint_pronunciation_dict.js`
- **清理**：自动检测并删除129个冗余词条
- **词条数**：231 → 120条

### v3.11 (2026-04-04)
- **修正**："王"字词条清理
- **删除**：9个"王"相关词条（pinyin-pro已正确）
- **保留**：王天下、称王、欲王（动词wàng）
- **词条数**：129 → 120条

### v3.12 (2026-04-04)
- **修正**："称王"读音错误
- **背景**：用户指正，查Wiktionary确认读wáng
- **删除**："称王"词条（pinyin-pro正确）
- **词条数**：120 → 119条
- **教训**：必须查证权威来源，不能主观判断

## 五、典型案例分析

### 案例1："称王"事件（2026-04-04）

**时间线**：
1. **初次标注**：主观判断"称王"中的"王"读wàng（动词）
2. **用户质疑**：不清楚读音的要查Wiktionary
3. **查证发现**：Wiktionary标注为chēngwáng，"王"读wáng
4. **测试确认**：pinyin-pro输出正确
5. **删除词条**：v3.11→v3.12

**错误根源**：
- 依赖语义推理（"称王"是动词，所以"王"读wàng）
- 未查证权威来源
- 忽视了汉语复杂性（"称王"整体是动词，但"王"本身是名词）

**教训**：
- ✅ 必须查Wiktionary
- ✅ 必须测试pinyin-pro输出
- ❌ 不能依赖主观判断

### 案例2："欲王"近删除事件（2026-04-04）

**时间线**：
1. **初次Lint**：测试"欲王"，pinyin-pro输出`yù wáng`（看似正确）
2. **险些删除**：认为pinyin-pro已正确处理
3. **用户提醒**："欲王是wang4啊"
4. **查证原文**：
   - "沛公欲王关中" - 想在关中称王
   - "欲王"是动词，读yù wàng
5. **保留词条**：pinyin-pro默认wáng不符合语境

**错误根源**：
- 只测试孤立词条，未结合原文语境
- 忽视了"王"作动词时的特殊读音

**教训**：
- ✅ 孤立测试不足，必须验证原文
- ✅ 需要结合语境判断动词/名词
- ✅ Lint工具要支持原文验证

### 案例3："於"字修正（v3.0.2）

**问题**：pinyin-pro默认库将"於"标注为wū，但《史记》中99%读yú。

**解决方案**：
```json
{
  "text": "於",
  "pinyin": ["yú"],
  "context": "介词",
  "note": "介词，相当于'在'、'到'、'对于'，读yú（阳平），不读wū"
}
```

**验证**：
```javascript
// 测试常见用法
"於是" → pinyin-pro默认: wū shì（❌）
"於是" → 修正后: yú shì（✓）

// 统计验证
《史记》中"於"的用法：
- 介词（於是、至於）：99.5%
- 其他：0.5%
```

**成果**：单字词条例外的典型成功案例

## 六、Lint检查规范

### 6.1 自动化检查流程

**工具**：`scripts/lint_pronunciation_dict.js`

**检查项**：
1. 测试每个词条的pinyin-pro输出
2. 对比词条中的pinyin
3. 标记pinyin-pro已正确处理的词条

**示例输出**：
```
=== 拼音词表Lint检查结果 ===

文件: docs/data/special-pronunciation.json
总词条数: 119
✓ 需要保留: 119 (pinyin-pro处理错误)
✗ 应该删除: 0 (pinyin-pro已正确处理)

✓ 所有词条都是必需的（pinyin-pro无法正确处理）
```

### 6.2 手动验证流程

**步骤**：
1. 运行`node scripts/test_pinyin_pro.js`
2. 输入待测词条
3. 对比输出与预期
4. 查Wiktionary确认
5. 决定是否添加

**示例**：
```javascript
const {pinyin} = require('pinyin-pro');

// 测试"称王"
console.log(pinyin('称王', {toneType: 'symbol', type: 'array'}));
// 输出: ['chēng', 'wáng']

// 查Wiktionary: m=chēngwáng
// 结论: pinyin-pro正确，不添加
```

### 6.3 质量保障指标

| 指标 | 目标 | 当前(v3.12) |
|------|------|------------|
| 词条总数 | <150 | 119 ✓ |
| Lint通过率 | 100% | 100% ✓ |
| 必要词条率 | 100% | 100% ✓ |
| 月度清理 | 每月1次 | - |
