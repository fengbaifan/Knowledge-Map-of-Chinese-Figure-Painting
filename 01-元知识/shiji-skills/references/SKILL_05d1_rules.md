# 事实抽取规则（Fact Extraction Rules）

**版本**：v1.0
**日期**：2026-04-03
**来源**：001_五帝本纪试点实验总结
**状态**：正式规范

---

## 一、核心原则

### 1.1 保持原文风貌

**事实抽取的目标是忠实记录《史记》原文，而非改写为标准化知识库。**

- 主语（S）、谓语（P）、宾语（O）直接取自原文
- 不预设 schema，不强制归一化
- 允许同一概念有多种表达（如"黄帝"/"轩辕"，"崩"/"薨"/"卒"）
- 后期可通过实体对齐、谓语聚类来归纳规律，但抽取阶段不强加规则

---

## 二、具体规则

### 2.1 人名规范：❌ 不规范化

**原则**：保持原文用词，不强制统一为规范名。

**示例**：
```json
{ "s": "黄帝", "p": "崩", ... }        // 原文用"黄帝"
{ "s": "轩辕", "p": "修德振兵", ... }  // 原文用"轩辕"
```

**说明**：
- 同一人物在原文中可能有多个称谓（黄帝/轩辕、帝颛顼/高阳）
- 事实抽取阶段保留原文称谓
- 后期通过实体对齐（entity resolution）建立 `轩辕 → 黄帝` 映射
- 优势：
  - 可追溯原文表述习惯（何时用帝号，何时用名）
  - 避免过早规范化导致信息丢失
  - 符合《史记》"互见法"特征（同一人物在不同篇章用不同称谓）

**例外**：
- 如果原文明确定义别名（如"是为黄帝"），则可在事实中记录：
  ```json
  { "id": "001-f0036", "s": "轩辕", "p": "是为", "o": "黄帝", "conf": "exact" }
  ```

---

### 2.2 谓语归一化：❌ 不归一化

**原则**：谓语（P）直接取原文动词或短语，不映射到预定义枚举。

**示例**：
```json
{ "p": "崩", ... }        // 天子之死
{ "p": "薨", ... }        // 诸侯之死
{ "p": "卒", ... }        // 大夫之死
{ "p": "死", ... }        // 庶人之死
```

**说明**：
- 《史记》用词有礼制含义（崩/薨/卒/死区分等级）
- 事实抽取阶段保留原文动词
- 后期可通过谓语聚类（predicate clustering）归纳：
  - `{崩, 薨, 卒, 死} → death`（上位概念）
  - `{立, 即位, 践祚} → accession`
  - `{封, 赐, 锡} → reward`
- 优势：
  - 保留礼制信息
  - 可研究用词分布（"崩"出现频次→天子数量）
  - 灵活支持不同粒度的查询

**例外**：
- 如果原文用复合谓语（如"修德振兵"），整体作为谓语：
  ```json
  { "p": "修德振兵", ... }
  ```

---

### 2.3 抽象描述：❌ 不拆分

**原则**：抽象或修饰性描述作为整体谓语，不拆分为更小单元。

**示例**：
```json
{ "s": "帝颛顼", "p": "静渊以有谋", "o": null, "conf": "medium" }
{ "s": "帝颛顼", "p": "疏通而知事", "o": null, "conf": "medium" }
{ "s": "帝颛顼", "p": "养材以任地", "o": null, "conf": "low" }
```

**说明**：
- 类似描述难以拆分为独立三元组：
  - ❌ 错误：`{ "s": "帝颛顼", "p": "静渊", "o": null }` + `{ "s": "帝颛顼", "p": "有谋", "o": null }`
  - ✅ 正确：`{ "s": "帝颛顼", "p": "静渊以有谋", "o": null }`
- 整体保留原文四字句式或骈文结构
- 置信度标注为 `low` 或 `medium`（抽象描述难以核验）

**拆分标准**：
- ✅ 可拆分：明确的并列事实（"生二子，其後皆有天下" → 2条事实）
- ❌ 不拆分：修饰性短语（"静渊以有谋"）、四字成语（"披山通道"）

---

### 2.4 复合动作：✅ 分别记录

**原则**：涉及多个对象的复合动作，拆分为多条事实。

**示例**：

**原文**：举风后、力牧、常先、大鸿以治民。

**拆分为4条事实**：
```json
{ "id": "001-f0061", "s": "黄帝", "p": "举", "o": "风后", "text": "举风后、力牧、常先、大鸿以治民", "conf": "high" }
{ "id": "001-f0062", "s": "黄帝", "p": "举", "o": "力牧", "text": "举风后、力牧、常先、大鸿以治民", "conf": "high" }
{ "id": "001-f0063", "s": "黄帝", "p": "举", "o": "常先", "text": "举风后、力牧、常先、大鸿以治民", "conf": "high" }
{ "id": "001-f0064", "s": "黄帝", "p": "举", "o": "大鸿", "text": "举风后、力牧、常先、大鸿以治民", "conf": "high" }
```

**说明**：
- 每条事实保留完整原文（`text` 字段相同）
- 便于后期查询："谁举用了风后？"→ 直接匹配 `o = "风后"`
- 避免将宾语记录为列表（不符合三元组定义）

**拆分标准**：
- ✅ 拆分：并列对象（"举A、B、C、D"）
- ✅ 拆分：多个地点（"东至海，西至空桐，南至江，北逐荤粥" → 4条事实）
- ❌ 不拆分：复合谓语（"修德振兵" → 1条事实，不拆为"修德"+"振兵"）
- ❌ 不拆分：复合宾语作为整体概念（"天地之纪" → 整体宾语）

## 三、边界案例

### 3.1 "是为"句式

**原文**：其一曰玄嚣，是为青阳。

**处理方式**：
```json
{ "id": "001-f0084", "s": "嫘祖长子", "p": "曰", "o": "玄嚣", "conf": "exact" }
{ "id": "001-f0085", "s": "玄嚣", "p": "是为", "o": "青阳", "conf": "exact" }
```

**说明**：
- "是为"是定义性句式，置信度为 `exact`
- 建立 `玄嚣 ≡ 青阳` 别名关系，但不在抽取阶段合并

---

### 3.2 隐含主语

**原文**：黄帝崩，葬桥山。

**处理方式**：
```json
{ "id": "001-f0093", "s": "黄帝", "p": "崩", "o": null, "conf": "high" }
{ "id": "001-f0094", "s": "黄帝", "p": "葬", "o": null, "ctx": {"place": "桥山"}, "conf": "high" }
```

**说明**：
- 第二句隐含主语（被动语态），补全为"黄帝"
- 地点信息放入 `ctx.place`

---

### 3.3 数字+量词

**原文**：黄帝二十五子。

**处理方式**：
```json
{ "id": "001-f0076", "s": "黄帝", "p": "子数", "o": "二十五", "conf": "high" }
```

**说明**：
- 谓语自造为"子数"（原文无对应动词）
- 宾语为"二十五"（保留原文数字形式，不转为阿拉伯数字）

---

### 3.4 时间背景

**原文**：轩辕之时，神农氏世衰。

**处理方式**：
```json
{ "id": "001-f0009", "s": "神农氏", "p": "世衰", "o": null,
  "ctx": {"occasion": "轩辕之时"}, "conf": "high" }
```

**说明**：
- 时间背景放入 `ctx.occasion`（而非 `ctx.time`，因为非纪年）
- 如有明确纪年（如"前210年"），则转为数字放入 `ctx.time`

---

## 四、置信度判定

| 置信度 | 定义 | 示例 |
|--------|------|------|
| `exact` | 定义性句式，绝对准确 | "是为黄帝"、"名曰轩辕" |
| `high` | 明确陈述，可独立核验 | "黄帝崩"、"与蚩尤战於涿鹿之野" |
| `medium` | 描述性语句，有原文依据但难以独立核验 | "登丸山"、"静渊以有谋" |
| `low` | 抽象表述，主观性强 | "有圣德"、"迎日推筴" |

## 五、文件组织规范

### 5.1 Markdown 表格

**位置**：`kg/facts/markdown/{章节}_事实表格.md`

**格式**：
```markdown
| ID | 主语(S) | 谓语(P) | 宾语(O) | 时间 | 地点 | 其他背景 | 原文 | 置信度 | 关联事件 |
|----|---------|---------|---------|------|------|----------|------|--------|----------|
| 001-f0001 | 黄帝 | 父 | 少典 | | | | 黄帝者，少典之子 | high | |
```

### 5.2 JSON 数据

**位置**：`kg/facts/data/{章节}_事实索引.json`

**字段**：
```json
{
  "id": "001-f0001",
  "s": "黄帝",
  "p": "父",
  "o": "少典",
  "ctx": {"time": null, "place": null},
  "src": "001_五帝本纪",
  "text": "黄帝者，少典之子",
  "conf": "high",
  "event": null
}
```
