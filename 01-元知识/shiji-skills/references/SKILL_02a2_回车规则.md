# SKILL 02a2: 回车规则 — 标注文件段落内换行标准

> 从《史记》130篇标注文件的可读性优化实践中提炼。
>
> **前置**: 段落编号已完成，详见 [SKILL_02a_章节切分与编号.md](SKILL_02a_章节切分与编号.md)。

---

## 一、核心原则

**一个 Purple Number `[N]` = 一个语义段落**，段落内部可按句号（。）换行以提升可读性，换行不改变段落归属。

### 1.0 引号内不断行（最高优先级）

**引号内的文字不得断行**。一段引语（`"..."` 之间的内容）无论多长，都必须保持在同一行内。断行只能发生在右引号 `"` 之后。

**错误**（引号内断行）：
```
〖@孔子〗曰："臣闻有文事者必有武备。
古者〖#诸侯〗出疆，必具官以从。
请具〖;左右司马〗。"
```

**正确**（引号内保持一行，断在 `"` 后）：
```
〖@孔子〗曰："臣闻有文事者必有武备。古者〖#诸侯〗出疆，必具官以从。请具〖;左右司马〗。"
〖;定公〗曰："诺。"
```

此规则优先级高于所有其他拆行规则——即使段落符合平行结构/纪年等拆行条件，也不得在引号内部断行。

**例外**：带独立PN编号的行（`[N.x]` 或 `- [N.x]`）不得因引号合并而丢失换行。引号合并只针对无PN编号的续行。

### 1.1 换行 vs 不换行

| 类型 | 换行规则 | 说明 |
|------|----------|------|
| 平行世系 | ✅ 每句一行 | "X卒，子Y立。"重复出现的密集世系链 |
| 平行列举 | ✅ 每句一行 | 结构相同的并列短句（如封国概述） |
| 密集命名/短对话 | ✅ 每句一行 | "一曰X，二曰Y"命名列举或密集短对话 |
| 纪年编年 | ✅ 每句一行 | "X年，…。Y年，…。"密集时间序列 |
| 叙事段落 | ❌ 保持单行 | 有情节推进、对话、因果关系的连续叙事 |
| 表格单元格 | ❌ 不回车 | 含 `\|` 的表格行内部不拆行 |
| 韵文/赞 | ✅ 每句一行 | 已在 SKILL_02b 中定义 |

### 1.2 判定标准

脚本通过三条独立路径检测：

**路径A：骨架平行结构**（`parallel`）
1. 提取每句骨架（标注类型+功能词），≥ 3 句骨架相同 → 平行
2. 平均句长 ≤ 80字，骨架长度 ≥ 3

**路径B：密集列举**（`enumeration`）
1. "曰" ≥ 5次，曰密度 ≤ 40字/曰
2. 句数 ≥ 5，平均句长 ≤ 35字
3. 总长 ≤ 500字（排除长叙事对话如"触龙说赵太后"）

**路径C：纪年编年**（`chronicle`）
1. 时间标注 `〖%...〗` ≥ 5个
2. 句数 ≥ 5，平均句长 ≤ 50字
3. "曰" ≤ 2（排除含对话的叙事段落）

**路径D：极短句列举**（`short-list`）
1. 句数 ≥ 6，平均句长 ≤ 15字
2. 典型：龟策列传占卜条目、五脏对应、星象描述等无标注的密集短句

**全局排除**：表格行（含 `|`）不拆行

## 二、需要拆行的典型模式

### 2.1 世系链（最常见）

密集的君主传承记录，每句记录一次即位或死亡：

**拆行前**（难以阅读）：
```
[23] 〖@微子开〗卒，立其弟〖@衍〗，是为〖@微仲〗。〖@微仲〗卒，子〖@宋公〗〖@稽〗立。〖@宋公〗〖@稽〗卒，子〖@丁公〗〖@申〗立。
```

**拆行后**（清晰的世系脉络）：
```
[23] 〖@微子开〗卒，立其弟〖@衍〗，是为〖@微仲〗。
〖@微仲〗卒，子〖@宋公〗〖@稽〗立。
〖@宋公〗〖@稽〗卒，子〖@丁公〗〖@申〗立。
```

### 2.2 侯表世系

年表中的侯位传承：

```
[4] 〖@缪靖侯〗卒，子〖;康侯〗〖@遂成〗立。
〖@遂成〗卒，子〖;怀侯〗〖@世宗〗立。
〖@世宗〗卒，子〖;侯〗〖@终根〗立，为〖;太常〗，〖[坐法〗，国除。
```

### 2.3 并列短事件

多个独立事件按时间排列，彼此无因果关系：

```
[14] 〖@穆公〗〖%五年〗，〖@齐桓公〗卒。
〖%十六年〗，〖@晋文公〗败〖◆楚〗师于〖=城濮〗。
是岁，〖@穆公〗卒，子〖@共公朔〗立。
```

### 2.4 密集战功记录

功臣传中大量"攻X、击X、破X、取X"并列短事件，每个独立战事一行：

**拆行前**（一整段难以阅读）：
```
[2] 将击胡陵、方与，攻秦监公军，大破之。东下薛，击泗水守军薛郭西。复攻胡陵，取之。徙守方与。方与反为魏，击之。丰反为魏，攻之。
```

**拆行后**（每个战事独立一行）：
```
[2] 将击胡陵、方与，攻秦监公军，大破之。
东下薛，击泗水守军薛郭西。
复攻胡陵，取之。
徙守方与。方与反为魏，击之。
丰反为魏，攻之。
```

**适用范围**：曹相国世家、樊郦滕灌列传、傅靳蒯成列传等功臣传的密集军功段落。

### 2.5 短平行结构共享PN

短对话、短列举如果每项很短（均长 ≤ 60字），**不应给每项单独分配子PN**，而应合并为一个段落内的多行：

**错误**（每项独立PN，浪费编号空间）：
```
[12]

- [12.1] 〖@尧〗曰："谁可顺此事？"
- [12.2] 〖@放齐〗曰："〖#嗣子〗〖@丹硃〗开明。"
- [12.3] 〖@尧〗曰："吁！顽凶，不用。"
```

**正确**（合并为一个段落，每句一行）：
```
[12] 〖@尧〗曰："谁可顺此事？"
〖@放齐〗曰："〖#嗣子〗〖@丹硃〗开明。"
〖@尧〗曰："吁！顽凶，不用。"
```

**判定标准**：
- 列表项 ≥ 3个，且每项平均 ≤ 60字 → 合并为一个PN
- 列表项各自较长（均 > 60字）或有独立语义单元 → 保留独立PN
- 合并后，如果列表后面还有后续段落（原来编号较大），需重新编号

**典型合并场景**：
- 短对话（X曰："..."Y曰："..."，每句 ≤ 30字）
- 世系链（X卒，子Y立。每句 ≤ 20字）
- 平行列举（X主Y，Z主W。每项 ≤ 30字）

## 三、过度分段的修复（段落合并）

上述§1-§2讨论的是"段落内换行"。本节讨论相反方向的问题：**一个语义段落被错误拆分为多个PN**，需要合并回一个PN。

### 3.1 引用块合并

短对话不应使用 `> "..."` 引用块格式。引用块会在渲染中产生视觉断裂，且浪费垂直空间。

**错误**（引用块格式）：
```
[11] 公曰：

> "吾欲南之〖=江〗上，何如？"

[12] 对曰：

> "昔〖@祝融〗为〖&高辛氏〗〖;火正〗……"
```

**正确**（合并回前段）：
```
[3] 公曰："吾欲南之〖=江〗上，何如？"
对曰："昔〖@祝融〗为〖&高辛氏〗〖;火正〗……"
```

**修复方法**：`曰：\n\n> "..."` → `曰："..."`

### 3.2 短句过度分段

每个短句独立一个PN是常见的早期标注错误。

**错误**（每句一个PN）：
```
[8] 〖%元年〗，〖;吴王〗〖@阖庐〗闻〖@允常〗死，乃兴师⟦◈伐⟧〖◆越〗。

[9] 〖;越王〗〖@句践〗使〖#死士〗挑⟦◈战⟧。

[10] 〖◆吴〗师观之，〖◆越〗因⟦◈袭⟧⟦◈击⟧〖◆吴〗师。

[11] 〖@阖庐〗且⟦◉死⟧，告其子〖@夫差〗曰："必毋忘〖◆越〗。"
```

**正确**（同一事件合并为一个PN）：
```
[3] 〖%元年〗，〖;吴王〗〖@阖庐〗闻〖@允常〗死，乃兴师⟦◈伐⟧〖◆越〗。
〖;越王〗〖@句践〗使〖#死士〗挑⟦◈战⟧。
〖◆吴〗师观之，〖◆越〗因⟦◈袭⟧⟦◈击⟧〖◆吴〗师。
〖@阖庐〗且⟦◉死⟧，告其子〖@夫差〗曰："必毋忘〖◆越〗。"
```

### 3.3 合并判定标准

**应合并的场景**：

| 场景 | 判定 | 示例 |
|------|------|------|
| 短对话交锋 | 连续X曰/Y曰，每句 ≤ 60字 | 尧与四岳对话 |
| 连续短叙事 | 同一事件的连续动作，每句 ≤ 30字 | 战役过程 |
| 世系链 | "X卒，子Y立"重复 | 越王世系 |
| 引用块对话 | `> "..."` 格式的短引语 | 曰后接引用块 |

**不应合并的场景**：

| 场景 | 判定 | 理由 |
|------|------|------|
| 各段 > 100字 | 各自有独立内容 | 合并后过长 |
| 长篇对话 | 每段引语 > 80字 | 每段引语本身是完整论述 |
| 跨小节段落 | 属于不同小节 | 语义不连续 |
| 独立事件 | 虽然相邻但无因果关系 | 如不同年份的独立事件 |

### 3.4 合并后重新编号

合并段落后必须重新编号，确保PN连续：

```
# 合并前：[8][9][10][11] → 合并后：[3]
# 后续段落 [12][13]... → 重新编号为 [4][5]...
```

**工具**：目前无自动脚本，需手动或编写章节专用脚本处理。典型流程：

1. 合并引用块：`曰：\n\n> "..."` → `曰："..."`
2. 识别连续短段组（小节标题为分断点）
3. 在长段（>150字）处也分断
4. 合并各组，重新分配PN
5. 验证编号连续性

## 四、不应拆行的段落

### 3.1 叙事段落

有情节发展、对话、因果推理的段落保持单行：

```
[30] 〖%九年〗，〖;大司马〗〖@孔父嘉〗妻好，出，道遇〖;太宰〗〖@华督〗，〖@督〗说，目而观之。〖@督〗利〖@孔父〗妻，乃使人宣言国中曰："〖@殇公〗即位〖%十年〗耳，而十一战，〖#民〗苦不堪……"
```

### 3.2 长篇引文/议论

箕子洪范、太史公曰等连续论述：

```
[15] "〖^皇极〗：皇建其有极，敛时〖^五福〗，用傅锡其〖#庶民〗……"
```

## 四、格式规范

### 4.1 换行格式

- `[N]` 编号**只出现在第一行**
- 后续行**顶格**，不缩进
- 段落之间保持空行分隔（与未拆行时一致）

```
[N] 第一句。
第二句。
第三句。

[N+1] 下一段落。
```

### 4.2 禁止事项

- ❌ 不得在后续行重复 `[N]` 编号
- ❌ 不得在换行处添加空行（空行表示段落分隔）
- ❌ 不得为了拆行而改变原文标点

## 五、工具与脚本

### 5.1 自动拆行脚本

```bash
# 预览将要拆分的段落
python scripts/split_lineage_paragraphs.py --dry-run

# 执行拆分
python scripts/split_lineage_paragraphs.py
```

**判定逻辑（骨架相似度算法）**：

1. 按"。"拆分句子
2. 对每句提取"骨架"：标注内容替换为类型标记（`@`→`P`, `%`→`T`, `◆`→`S` 等），只保留功能词（卒/立/生/封等）和标点
3. 归一化骨架：合并连续同类标记（`PP`→`P+`）
4. 统计骨架频次，最大频次 ≥ 3 则判定为平行结构
5. 额外约束：平均句长 ≤ 80字、"曰" ≤ 2次、骨架长度 ≥ 3（排除空骨架误报）

**骨架示例**：
| 原句 | 骨架 | 归一化 |
|------|------|--------|
| `〖@微仲〗卒，子〖@宋公〗〖@稽〗立` | `P卒，子PP立` | `P卒，子P+立` |
| `〖@共公〗〖%八年〗卒，子〖@德公〗立` | `PT卒，子P立` | `PT卒，子P立` |
| `〖!角〗、〖!亢〗、〖!氐〗，〖=兗州〗` | `H、H、H，L` | `H、H、H，L` |

```bash
# 带骨架详情的预览
python scripts/split_lineage_paragraphs.py --dry-run --verbose
```

### 5.2 手动判断流程

对于脚本未覆盖的边界情况：

1. 该段是否有3个以上结构相同的短句？→ 是则拆行
2. 该段是否有对话或因果推理？→ 是则保持单行
3. 拆行后每行是否独立可理解？→ 是则拆行

## 七、检查清单

### 段落内换行（拆行）

- [ ] 平行世系/纪年/列举段落是否已按句号拆行？
- [ ] 拆行后 `[N]` 只出现在首行？
- [ ] 拆行后段落间空行未被破坏？
- [ ] 叙事段落未被误拆？

### 过度分段修复（合并）

- [ ] 是否存在 `> "..."` 引用块格式的短对话？→ 合并回前段
- [ ] 是否存在连续极短段落（< 30字）？→ 考虑合并
- [ ] 合并后PN编号是否连续？
- [ ] 合并后段落粒度是否合理（每段30-300字）？

### 段落粒度参考

| 指标 | 参考值 | 说明 |
|------|--------|------|
| 每章段落数 | 30-80 | 过少说明段落太大，过多说明过度分段 |
| 平均段落长度 | 50-200字 | < 30字通常需要合并 |
| < 30字段落占比 | < 20% | 占比过高说明过度分段 |
