# SKILL 02b1: 韵文识别规则 — 自动检测、分类与标题提取

> 从《史记》130篇的韵文提取实践中提炼，适用于赞、诗歌、赋的自动识别与结构化。
>
> **前置**: 区块已用 `::: 标签` 标注完成，详见 [SKILL_02b_区块与韵文处理.md](SKILL_02b_区块与韵文处理.md)。

## 一、问题定义

《史记》中的韵文散布在130篇中，包括：

1. **赞**（110篇）— 章末的韵文评价，四字为主
2. **诗歌**（11篇）— 历史人物作品或刻石铭文
3. **赋**（5篇）— 长篇韵文，如屈原赋、司马相如赋

**核心挑战**：
- 某些章节的赞嵌入在"太史公曰"散文块中，需分离
- 诗歌和赋的标题需从上下文提取，不能用通用格式"章节名+诗/赋"
- 需要内容特征匹配来确定具体作品名称

## 二、韵文类型识别

### 2.1 赞的识别模式

#### 文本特征

**典型特征**：
- 4-6个汉字为一句（主要是4字句）
- 每句末尾带标点符号：`，。：；！？`
- 至少连续3行符合上述模式
- 对仗工整，押韵明显

**示例**：
```
韩襄遗孽，始从汉中。
剖符南面，徙邑北通。
穨当归国，龙雒有功。
```

#### 检测算法（verse_score函数）

**核心逻辑**：
1. 移除标注符号，保留纯文本内容
2. 按标点符号拆分成句子
3. 统计句子长度分布
4. 计算韵文得分（0-1）

**评分规则**：
- 4字句占比 > 50%：得分0.95（强韵文）
- 3-5字句占比 > 70%：得分0.8（中强韵文）
- 4-7字句占比 > 70%：得分0.7（诗歌可能）
- 平均句长6-8字且方差小：得分0.75（五言七言诗）
- 平均句长 > 10字：得分0.1（散文）

**阈值**：得分 > 0.6 视为韵文

### 2.2 赞与散文的边界识别

#### 典型场景

某些"太史公曰"块包含：
1. 前半部分：散文评论（长句、议论性）
2. 后半部分：赞体韵文（短句、四字为主）

**分离规则**：
1. 检测起点：在"太史公曰"块内，找到第一个连续3行符合赞特征的位置
2. 切分点：该位置前一行的末尾
3. 编号处理：分离后的赞块获得新的段落编号（`[N+1]`）
4. 人工确认：自动分离后需人工检查，确保语义完整

### 2.3 诗歌与赋的识别

**诗歌特征**：
- 句长4-7字，五言或七言为主
- 历史人物作品，出现在叙事中
- 标注方式：`::: 诗歌`

**赋特征**：
- 篇幅较长（通常100字以上）
- 有明确的作品名称
- 标注方式：`::: 赋`

## 三、标题提取规则

### 3.1 基本原则

**禁止使用通用格式**：
- ❌ 错误：秦始皇本纪诗、屈原贾生列传赋
- ✅ 正确：泰山刻石、怀沙赋

**标题来源优先级**：
1. 文中明确提到的作品名（最高优先级）
2. 历史事件名称（如"垓下歌"）
3. 地点相关名称（如"泰山刻石"）
4. 特征性内容关键词（如"大人赋"）

## 五、HTML渲染优化

### 5.1 唯一ID系统

**问题**：同一章节的多个韵文共用章节号作为锚点ID，导致链接错误

**解决方案**：为每个韵文分配唯一ID（`yunwen-0`, `yunwen-1`, ...）

```python
# 在 render_yunwen_html.py 中
for idx, item in enumerate(yunwen_data):
    item['_unique_id'] = f"yunwen-{idx}"

# TOC链接
html += f'<a href="#{unique_id}">{title}</a>'

# 内容锚点
html += f'<div id="{unique_id}">'
```

### 5.2 标题显示优先级

1. 具体标题（如"泰山刻石"） — 优先显示
2. 章节标题 + 韵文类型（如"秦始皇本纪 · 诗歌"）
3. 仅章节标题（如"秦始皇本纪"）

## 七、关键经验

1. **标题必须具体化** — "秦始皇本纪诗"不是有效标题，必须提取到"泰山刻石"这样的具体名称

2. **内容特征优先** — 当文中明确提到作品名时，优先使用；否则根据地点、事件命名

3. **检查顺序很重要** — 识别赋标题时，先检查更具体的特征（如"大人赋"），再检查通用特征（如"子虚"）

4. **唯一ID解决重复锚点** — 同一章节的多个韵文不能共用锚点ID，必须用序号区分

5. **散文/韵文边界需人工确认** — 自动分离算法可能误判，特别是古文散文中出现四字节奏时

6. **verse_score阈值**：0.6是韵文/散文分界线，>0.9为强韵文（主要是4字赞）

7. **深度挖掘发现隐藏赞** — 2026-04-06通过verse_score算法在33篇"无赞"章节中发现了17篇隐藏的赞，将赞总数从97篇提升到110篇
