# Purple Numbers (PN) 编号规范

## 概述

Purple Numbers (PN) 是为《史记》每个段落分配的唯一标识符，格式可以是 `[N]`、`[N.M]` 或 `[N.M.K]`，用于精确引用和链接文本。

## 基本格式

### 多级编号系统

Purple Numbers支持最多三级的层级编号：

- **一级编号**: `[N]` - 顶级section编号
- **二级编号**: `[N.M]` - section内的子section或段落编号
- **三级编号**: `[N.M.K]` - 子section内的段落编号（可选）

**二级结构示例**（简单章节）:
```
[1]           ← 顶级section的父编号
[1.1] 段落1   ← 第一段
[1.2] 段落2   ← 第二段

[2] 单段内容  ← 只有一段的section

[3]           ← 多段section的父编号
[3.1] 段落1
[3.2] 段落2
```

**三级结构示例**（复杂章节）:
```
[1]                    ← 顶级section父编号
[1.1]                  ← 子section父编号
[1.1.1] 段落1          ← 子section的第一段
[1.1.2] 段落2          ← 子section的第二段

[1]                    ← 同一顶级section（可能对应不同markdown标题）
[1.2]                  ← 另一个子section父编号
[1.2.1] 段落1          ← 另一个子section的段落

[2]                    ← 下一个顶级section
[2.1]
[2.1.1] 段落1
```

### 选择二级还是三级

根据章节的复杂度和逻辑结构选择：

**使用二级编号** `[N.M]`:
- 章节结构简单，每个markdown标题独立成section
- 不需要对段落进行二级逻辑分组
- 示例：列传、本纪等叙事性章节

**使用三级编号** `[N.M.K]`:
- 章节结构复杂，多个markdown标题需要归属同一个逻辑大section
- 需要对段落进行二级分组
- 示例：八书（礼书、历书等）、十表等结构性章节

## 编号分配规则

### 1. 顶级编号分配策略

有两种策略：

**策略A：每个标题一个顶级编号**（二级结构）
- 每个 `###` 或 `####` 标题开始一个新的顶级section
- 适用于结构简单的章节

```markdown
### 历法起源与意义          ← 分配 [1]
[1]
[1.1] 昔自在古...
[1.2] 於时冰泮...

### 改正朔的重要性          ← 分配 [2]
[2] 王者易姓受命...         ← 单段

### 黄帝建历               ← 分配 [3]
[3]
[3.1] 太史公曰...
```

**策略B：逻辑分组的顶级编号**（三级结构）
- 多个相关的 `###` 或 `####` 标题共用一个顶级section编号
- 每个标题对应一个二级section `[N.M]`
- 适用于结构复杂、有明显逻辑分组的章节

```markdown
### 历法起源与意义          ← 第一个二级section [1.1]
[1]
[1.1]
[1.1.1] 昔自在古...
[1.1.2] 於时冰泮...

### 改正朔的重要性          ← 第二个二级section [1.2]
[1]
[1.2]
[1.2.1] 王者易姓受命...

### 黄帝建历               ← 新的顶级section，第一个二级section [2.1]
[2.1]
[2.1.1] 太史公曰...
```

### 2. 单段与多段区分

#### 二级结构中

**单段section**: 只有1个段落
- 编号格式: `[N]` (内容直接跟在编号后)
- 示例: `[2] 王者易姓受命，必慎始初...`

**多段section**: 有2个或更多段落
- 编号格式: `[N]` (父编号，独立一行) + `[N.1]`, `[N.2]`, ... (子编号)
- 示例:
  ```
  [1]
  [1.1] 昔自在古...
  [1.2] 於时冰泮...
  [1.3] 物乃岁具...
  ```

#### 三级结构中

**单段子section**: 只有1个段落
- 编号格式: `[N.M.1]` (可以省略 `[N]` 和 `[N.M]` 父编号行)
- 示例: `[1.2.1] 王者易姓受命...`

**多段子section**: 有2个或更多段落
- 编号格式:
  - 必须有 `[N]` 父编号（顶级section父编号）
  - 必须有 `[N.M]` 父编号（子section父编号）
  - 然后是 `[N.M.1]`, `[N.M.2]`, ... (段落编号)
- 示例:
  ```
  [1]           ← 顶级section父编号
  [1.1]         ← 子section父编号
  [1.1.1] 昔自在古...
  [1.1.2] 於时冰泮...
  [1.1.3] 物乃岁具...
  ```

### 3. 父编号规则

#### 二级结构

对于多段section，必须有父编号 `[N]`:
- 父编号独立一行
- 紧接着第一个子编号 `[N.1]`
- 父编号和第一个子编号之间没有空行

```
[3]          ← 父编号
[3.1] 内容1  ← 第一个子编号
[3.2] 内容2
[3.3] 内容3
```

#### 三级结构

对于多段子section，必须有两级父编号 `[N]` 和 `[N.M]`:
- 顶级父编号 `[N]` 独立一行
- 子section父编号 `[N.M]` 独立一行
- 然后是段落编号 `[N.M.1]`, `[N.M.2]`, ...
- 父编号之间和父编号与第一个段落之间都没有空行

```
[1]              ← 顶级父编号
[1.1]            ← 子section父编号
[1.1.1] 内容1    ← 第一个段落
[1.1.2] 内容2
[1.1.3] 内容3

[1]              ← 同一顶级section的另一个子section
[1.2]            ← 另一个子section父编号
[1.2.1] 内容1
```

**注意**: 同一顶级section `[N]` 的不同子section会重复出现 `[N]` 父编号行。

### 4. 无内容标题不编号

如果某个标题下面没有直接的内容段落，只有下级标题，根据使用的策略：

**二级结构**: 不分配编号
```markdown
### 上古历法传统（黄帝至尧舜）  ← 无直接内容，不编号
#### 黄帝建历                    ← 有内容，分配 [3]
[3]
[3.1] 太史公曰...
```

**三级结构**: 作为逻辑分组的容器，不需要独立编号
```markdown
### 上古历法传统（黄帝至尧舜）  ← 逻辑分组，对应顶级 [2]
#### 黄帝建历                    ← 对应 [2.1]
[2.1]
[2.1.1] 太史公曰...
```

## 连续性要求

### 顶级编号连续性

顶级编号必须从 `[1]` 开始连续递增：
- ✅ `[1]`, `[2]`, `[3]`, `[4]`, ...
- ❌ `[1]`, `[2]`, `[5]`, `[7]`, ... (跳号)

### 子编号连续性

同一父编号下的子编号必须从 `[N.1]` 开始连续递增：
- ✅ `[3.1]`, `[3.2]`, `[3.3]`, ...
- ❌ `[3.1]`, `[3.3]`, `[3.5]`, ... (跳号)

## 特殊情况处理

### 1. 章节标记符号

原文中的章节标记（如 `◎历术甲子篇`）保留但不分配PN编号。

```markdown
## 历术甲子篇       ← markdown导航标题
◎历术甲子篇        ← 原文标记，不编号
### 历法术语说明    ← 下一个section开始编号
[16] 太初元年...
```

### 2. 太史公赞

太史公在篇末的韵文赞语作为独立section，通常是单段：

```markdown
## 太史公赞
::: 赞
[38] 历数之兴，其来尚矣。
重黎是司，容成斯纪。
...
```

### 3. 术语解释段

"右XXX书"格式的术语解释段作为独立section：

```markdown
## 历法术语释义
[37]
[37.1] 右历书：大馀者，日也。小馀者，月也。
[37.2] 端蒙者，年名也。
```

## 验证方法

使用 `scripts/lint_purple_numbers.py` 检查编号是否符合规范：

```bash
python3 scripts/lint_purple_numbers.py 026
```

检查项目：
- ✓ 格式正确性（`[N]` 或 `[N.M]`）
- ✓ 连续性（无跳号）
- ✓ 父子关系（子编号有对应父编号）
- ✓ 无重复编号

## 实例：026_历书

### 最终结构

- **共38个顶级section**
- Section 1: 历法起源与意义（10段）→ `[1]` + `[1.1]`~`[1.10]`
- Section 2: 改正朔的重要性（1段）→ `[2]`
- Section 3: 黄帝建历（5段）→ `[3]` + `[3.1]`~`[3.5]`
- ...
- Section 36: 建始纪年（12段）→ `[36]` + `[36.1]`~`[36.12]`
- Section 37: 历法术语释义（5段）→ `[37]` + `[37.1]`~`[37.5]`
- Section 38: 太史公赞（1段）→ `[38]`

### 转换过程

**原始问题**:
```
[1.1.1] 昔自在古...     ← 三级编号
[1.1.2] 於时冰泮...
...
[4.2.228] ...           ← 大量三级编号
```

**修复后**:
```
[1]                     ← 添加父编号
[1.1] 昔自在古...      ← 转换为二级
[1.2] 於时冰泮...
...
[36.12] ...             ← 按标题重新编号
[37]                    ← 新section
[37.1] 右历书...
[38] 历数之兴...        ← 太史公赞
```

## 注意事项

1. **不要删除原文字符**: PN编号只能添加标记符号 `[N]` 或 `[N.M]`，不得修改原文内容
2. **保持格式一致**: 所有PN编号格式必须统一为 `[N]` 或 `[N.M]`
3. **段落定义**: 前后有空行的独立文本块视为一个段落
4. **章节标记**: 特殊符号（如 `◎`）标记的章节分隔符不编号
5. **韵文处理**: 太史公赞等韵文作为整体编一个号，不按行拆分
