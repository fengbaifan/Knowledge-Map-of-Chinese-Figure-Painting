# 变更日志索引

本目录存储项目的所有数据修订变更日志。

## 目录结构

```
99-日志文档/
├── INDEX.md              # 本文件
└── [年月]/               # 按年月组织
    └── 变更日志_[日期].md
```

## 日志组织规则

- 按年月创建子目录（格式：`YYYY-MM`）
- 每天一个日志文件（格式：`变更日志_YYYY-MM-DD.md`）
- 同一天的多次修订追加到同一文件

## 日志内容

每条变更记录包含：
- 修改人（Git 用户名 + 邮箱）
- 修改时间（精确到秒）
- 修改类型（自动修复 / 手动修订）
- 影响范围（文件路径、修改行数、字段数）
- 修改内容详情（字段级对比，使用 git diff）
- 修改原因
- 相关文件（审查报告、问题清单、Git commit）

## 自动生成

变更日志由 `change-logger` 技能自动生成：
- extraction-reviewer 完成数据修订后自动触发
- 用户手动修改文件后可手动触发

## 查看日志

```bash
# 查看今天的日志
cat 99-日志文档/$(date +%Y-%m)/变更日志_$(date +%Y-%m-%d).md

# 查看某个月的所有日志
ls -lh 99-日志文档/2026-04/

# 搜索特定文件的修改记录
grep -r "M2.1_历史人物" 99-日志文档/
```

## 日志归档

建议定期归档旧日志：
- 保留最近 3 个月的日志在此目录
- 更早的日志可以压缩归档到 `99-日志文档/archive/`
