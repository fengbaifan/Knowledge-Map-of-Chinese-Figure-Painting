# 中国人物画知识图谱

基于古代画史文献的结构化知识库项目，采用 M3 三层框架（宏观-中观-微观）构建中国人物画历史的系统性知识表达。

## 项目概述

本项目将非结构化的古籍画史文本转化为结构化知识图谱，实现对中国人物画历史的学术溯源、关系网络构建和知识推理。

### 核心特性

- **学术溯源**：所有数据可追溯到原始文献（书名、章节、原文）
- **M3 三层架构**：
  - **M1 宏观层**：时序本体、空间地理本体、图像志题材分类
  - **M2 中观层**：历史人物核心表、履历与时空轨迹、社会关系实例
  - **M3 微观层**：作品实体表、文献著录与品评
- **混合范式**：规则 + LLM + 人工抽检的数据生产机制

## 数据结构规范

### M3 三层框架（v1.6）

#### M1 宏观层：时空与分类本体
- **M1.1 时序本体**：朝代、年号、纪年换算
- **M1.2 空间地理本体**：地名、行政区划、地理坐标
- **M1.3 图像志题材分类**：人物、山水、花鸟等题材分类体系

#### M2 中观层：人物与关系网络
- **M2.1 历史人物核心表**：姓名、字号、籍贯、生卒年、身份
- **M2.2 履历与时空轨迹**：任职、活动、迁徙等时空事件
- **M2.3 社会关系实例**：师承、交游、亲属、鉴藏等关系

#### M3 微观层：作品与文献
- **M3.1 作品实体表**：作品名称、尺寸、材质、藏所
- **M3.2 文献著录与品评**：著录信息、品评文本、风格描述

### 溯源字段规范

所有表格均包含统一的溯源字段：
- `source_book`：来源典籍（如《历代名画记》）
- `source_chapter`：章节卷次（如"卷一·顾野王"）
- `source_text`：原文引用
- `source_page`：页码版本信息（可选）

## 技术架构

### 双层 Skill 体系

#### 元技能层（14个）
通用方法论技能，包括：
- 反思闭环
- 迭代工作流
- 冷启动方法
- 柳叶刀方法（问题分层拆解）
- 知识压缩
- 标注体系设计
- 质量控制
- 数据融合
- Agent提示词工程
- 可读性优化
- 数据体感培养
- 技能迁移

#### 管线技能层（10个阶段）

| 阶段 | 目标 | 核心任务 |
|------|------|----------|
| 00 总览 | 定义流程与依赖 | 明确"文献 → M1/M2/M3"全流程 |
| 01 校勘 | 文本可信 | 统一书名、卷次、标点、异体字 |
| 02 结构 | 结构可抽取 | 卷-条-段切分，评语/著录分区 |
| 03 实体 | 识别人、地、题材 | 人物、地名、题材、机构、术语 |
| 04 事件 | 提取生平轨迹 | 任职、交游、师承、活动、创作 |
| 05 关系 | 构建网络 | 人物关系、作品关系、文献互证 |
| 06 本体 | 统一语义模型 | 映射到 M1/M2/M3 字段与约束 |
| 07 推理 | 发现隐含知识 | 断代推断、同名消歧、矛盾检测 |
| 08 管驭 | 量化质量与覆盖 | 覆盖度、置信度、成本、版本控制 |
| 09 应用 | 形成可用产品 | 检索、时间线、图谱浏览、可视化 |

### 数据处理流程

```
原始文献（.txt）
    ↓
文本校勘与规范化
    ↓
结构切分（卷-条-段）
    ↓
实体抽取（人物、地名、作品）
    ↓
事件识别（履历、创作、交游）
    ↓
关系构建（师承、鉴藏、著录）
    ↓
本体映射（M1/M2/M3表格）
    ↓
质量控制与推理验证
    ↓
结构化数据输出（.csv）
```
