# 中国人物画知识图谱

基于古代画史文献的结构化知识库项目，采用 M3 三层框架（宏观-中观-微观）构建中国人物画历史的系统性知识表达。

## 项目概述

本项目旨在将非结构化的古籍画史文本转化为结构化的知识图谱，实现对中国人物画历史的学术溯源、关系网络构建和知识推理。

### 核心特性

- **学术溯源**：所有数据可追溯到原始文献（书名、章节、原文）
- **三层架构**：
  - **M1 宏观层**：时序本体、空间本体、图像志分类
  - **M2 中观层**：历史人物、社会关系、履历轨迹
  - **M3 微观层**：作品实体、文献著录、风格技法
- **混合范式**：规则 + LLM + 人工抽检的数据生产机制

### 数据来源

主要文献包括：
- 《历代名画记》（唐·张彦远）
- 《宣和画谱》（宋）
- 《图绘宝鉴》（元·夏文彦）
- 《画史》（宋·米芾）
- 其他画史、画论文献

## 技术架构

### Skill 体系

项目采用**双层 Skill 架构**：

#### 元技能层（Meta-Skills）
14 个通用方法论，包括：
- 反思闭环
- 迭代工作流
- 冷启动方法
- 柳叶刀方法（问题分层拆解）
- 知识压缩
- 标注体系设计
- 质量控制
- 数据融合

#### 管线技能层（Pipeline Skills）
按数据处理流程组织的 10 个阶段：

| 阶段 | 目标 | 任务 |
|------|------|------|
| 00 总览 | 定义流程与依赖 | 明确"文献 → M1/M2/M3"全流程 |
| 01 校勘 | 文本可信 | 统一书名、卷次、标点、异体字 |
| 02 结构 | 结构可抽取 | 卷-条-段切分，评语/著录分区 |
| 03 实体 | 识别人、地、题材 | 人物、地名、题材、机构、术语 |
| 04 事件 | 提取生平轨迹 | 任职、交游、师承、活动、创作 |
| 05 关系 | 构建网络 | 人物关系、作品关系、文献互证 |
| 06 本体 | 统一语义模型 | 映射到 M1/M2/M3 字段与约束 |
| 07 推理 | 发现隐含知识 | 断代推断、同名消歧、矛盾检测 |
| 08 管驭 | 量化质量与覆盖 | 覆盖度、置信度、成本、版本控制 |
| 09 应用 | 形成可用产品 | 检索、时间线、图谱浏览、可视化 |
| 10 管理 | 工程化协同 | Issue/TODO/日志/提交规范 |

## 数据处理流程

```
原始文献 → 文本识别 → 校勘规范化 → 结构切分 
    ↓
实体抽取 → 事件识别 → 关系构建 → 本体映射
    ↓
逻辑推理 → 质量控制 → 知识库管理 → 应用构造
```

## 开发指南

### 迁移策略

- **可直接复用（60%-70%）**：元技能体系、质量控制框架、项目管理框架
- **需适配（20%-30%）**：实体类型、关系类型、章节结构规则、抽取模板
- **需新建（10%-20%）**：作品著录规则、品评语义分层、图像志判别规则

### 质量原则

- 可溯源、可复核、可迭代
- 规则型 lint 与语义型反思分层协同
- 人工抽样观察不可替代
- MVP 先行，迭代提质
