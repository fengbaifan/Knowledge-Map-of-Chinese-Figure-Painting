# 中国人物画知识图谱

基于古代画史文献的结构化知识库项目，采用 M3 三层框架（宏观-中观-微观）构建中国人物画历史的系统性知识表达。

## 项目概述

本项目将非结构化的古籍画史文本转化为结构化知识图谱，实现对中国人物画历史的学术溯源、关系网络构建和知识推理。

### 核心特性

- **学术溯源**：所有数据可追溯到原始文献（书名、章节、原文）
- **M3 三层架构**：
  - **M1 宏观层**：时序本体、空间地理本体、图像志题材分类
  - **M2 中观层**：历史人物核心表、履历与时空轨迹、社会关系实例
  - **M3 微观层**：作品实体表、文献著录与品评
- **混合范式**：规则 + LLM + 人工抽检的数据生产机制

## 数据结构规范

### M3 三层框架（v1.6）

#### M1 宏观层：时空与分类本体
- **M1.1 时序本体**：朝代、年号、纪年换算
- **M1.2 空间地理本体**：地名、行政区划、地理坐标
- **M1.3 图像志题材分类**：人物、山水、花鸟等题材分类体系

#### M2 中观层：人物与关系网络
- **M2.1 历史人物核心表**：姓名、字号、籍贯、生卒年、身份
- **M2.2 履历与时空轨迹**：任职、活动、迁徙等时空事件
- **M2.3 社会关系实例**：师承、交游、亲属、鉴藏等关系

#### M3 微观层：作品与文献
- **M3.1 作品实体表**：作品名称、尺寸、材质、藏所
- **M3.2 文献著录与品评**：著录信息、品评文本、风格描述

### 溯源字段规范

所有表格均包含统一的溯源字段：
- `source_book`：来源典籍（如《历代名画记》）
- `source_chapter`：章节卷次（如"卷一·顾野王"）
- `source_text`：原文引用
- `source_page`：页码版本信息（可选）

## 技术架构

### Claude Code 技能系统

项目采用 Claude Code 标准技能格式，技能存储在 `.claude/skills/` 目录。

#### extraction-reviewer（数据质量审查）
**功能**：自动审查抽取数据的质量，检测并修复常见问题
- 三层检查：完整性、准确性、一致性
- 自动修复：字段名错误、格式不规范
- 生成报告：审查报告、问题清单、质量统计

**使用方法**：
```bash
# 审查单个文件
python .claude/skills/extraction-reviewer/scripts/reviewer.py \
  --file 03-初步抽取/古画品录/M2.1_历史人物核心表.csv \
  --output-dir 04-审查与修订

# 审查整个文献目录
python .claude/skills/extraction-reviewer/scripts/reviewer.py \
  --dir 03-初步抽取/古画品录 \
  --output-dir 04-审查与修订
```

#### change-logger（变更日志记录）
**功能**：自动记录数据修订操作，追踪所有字段级变更
- 字段级变更检测：使用 git diff 对比修改前后
- 自动触发：extraction-reviewer 完成修复后自动调用
- 详细记录：修改人、时间、影响范围、字段变更详情

**输出示例**：
```markdown
## [18:29:19] 数据修订记录 #5

**修改人**：Baifan Feng <fengbaifan@ahpu.edu.cn>
**修改时间**：2026-04-19 18:29:19
**修改类型**：数据修订（自动修复）
**触发来源**：extraction-reviewer

### 影响范围
- 修改行数：18 行
- 修改字段数：4 个字段

### 修改内容详情
**第 1 行**：
- 字段：`source_chapter`
- 修改前：`source.chapter`
- 修改后：`source_chapter`
- 原因：字段名格式修正
```

### 数据处理流程

```
02-原始文档/*.txt
    ↓
文本校勘与规范化
    ↓
结构切分（卷-条-段）
    ↓
实体抽取（人物、地名、作品）
    ↓
事件识别（履历、创作、交游）
    ↓
关系构建（师承、鉴藏、著录）
    ↓
本体映射（M1/M2/M3表格）
    ↓
03-初步抽取/[文献名]/*.csv
    ↓
数据审查与修订（extraction-reviewer）
    ↓
04-审查与修订/[文献名]/修订数据/*.csv
    ↓
变更日志记录（change-logger）
    ↓
99-日志文档/[年月]/变更日志.md
```
